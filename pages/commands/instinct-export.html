Instinct导出命令

导出Instinct规则为YAML格式

类型: Command类

实现效果

导出 Claude Code Instinct 规则为可共享的 YAML 格式：

  • 多范围导出
    支持导出项目、全局或全部范围的 Instinct
  • 过滤器支持
    按领域和置信度过滤导出的 Instinct
  • YAML输出格式
    标准 YAML 格式，易于共享和版本控制

    • 核心原理 

    
    Instinct 导出流程
    

    
导出流程:

Step 1: 确定项目上下文
├─ 检测当前工作目录
├─ 确定项目范围
└─ 识别 project_id 和 project_name

Step 2: 加载 Instinct
├─ 加载项目范围的 Instinct
├─ 加载全局 Instinct
└─ 根据 scope ('project'|'global'|'all') 选择

Step 3: 应用过滤器
├─ --domain <name>     : 按领域过滤
├─ --min-confidence <n>: 按置信度过滤
└─ 同时应用多个过滤器

Step 4: 生成 YAML 输出
├─ 格式化为标准 YAML
├─ 包含所有元数据
└─ 输出到文件或 stdout

YAML 结构:
id: instinct-name
trigger: "触发场景"
confidence: 0.8
domain: code-style
source: session-observation
scope: project
project_id: abc123
project_name: my-app
message: "建议使用函数式风格"

脚本:
node scripts/instinct-export.js [options]

    使用方法

    触发场景与调用 
    # 导出 Instinct 规则时触发

"/instinct-export [options]"
格式: 导出 Instinct 规则为 YAML
要求: 运行 /instinct-export 开始导出

示例用法：
1. /instinct-export                         # 导出所有个人Instinct
2. /instinct-export --domain testing        # 只导出测试领域的Instinct
3. /instinct-export --min-confidence 0.7    # 只导出高置信度的Instinct
4. /instinct-export --output team.yaml      # 输出到指定文件
5. /instinct-export --scope project         # 只导出项目Instinct

可用的选项:
--domain <name>      : 只导出指定领域的 Instinct
--min-confidence <N> : 最小置信度阈值 (0-1)
--output <file>      : 输出文件路径 (默认 output stdout)
--scope <type>       : 导出范围 (project|global|all)

范围说明:
- project : 只导出当前项目的 Instinct
- global  : 只导出全局 Instinct
- all     : 导出项目 + 全部合并 (默认)

    使用技巧

    1
    使用 `--output` 选项将 Instinct 保存到版本控制系统
    2
    团队共享时，使用 `--min-confidence 0.7` 只导出高质量规则
    3
    使用 `--domain code-style` 导出代码风格相关的 Instinct
    4
    对比导入前的 YAML 可以看到规则是否匹配
    5
    导出后检查 YAML 格式，确保 project_id 正确
    6
    使用 /instinct-import 导入导出的 YAML 文件

可用命令:
- /instinct-export [options]  - 导出 Instinct
- /instinct-import [options]  - 导入 Instinct
- /instinct-status            - 显示 Instinct 状态

YAML 格式说明:
- id: Instinct 唯一标识符
- trigger: 触发场景描述
- confidence: 置信度 (0-1)
- domain: 领域 (code-style, testing, error-handling 等)
- source: 来源 (session-observation, manual, etc.)
- scope: 范围 (project, global)
- project_id: 项目标识符
- project_name: 项目名称
- message: 建议的具体内容
