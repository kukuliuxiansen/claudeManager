检查点命令

创建和验证工作检查点，追踪项目进度和状态变化

类型: Command类

实现效果

检查点命令允许您在开发过程中创建时间点快照，并在后续验证工作状态的变化。这对于追踪进度、验证修复效果和回溯问题非常有用。

  • 创建检查点
    保存当前项目状态、git SHA 和相关元数据
  • 验证检查点
    对比当前状态与检查点，显示文件、测试和覆盖率变化
  • 列出检查点
    显示所有已保存检查点及其状态

    • 核心原理

    检查点系统使用 git stash/commit 和日志文件来追踪项目状态：

    1. 创建检查点 - 验证当前状态，创建 git stash/commit，将检查点信息记录到 .claude/checkpoints.log
    2. 验证检查点 - 读取检查点日志，对比文件变更、测试结果和覆盖率变化
    3. 列出检查点 - 读取日志文件，显示所有检查点及其 git SHA 和状态

    检查点数据存储在：

    • 日志文件: .claude/checkpoints.log
    • 格式: timestamp | checkpoint_name | git_sha

    使用方法

    触发方式 
    /checkpoint <create|verify|list> [name]

    子命令说明：

    可用子命令 
    /checkpoint create "feature-start"    # 创建新检查点
/checkpoint verify "feature-start"     # 验证相对于检查点的变更
/checkpoint list                       # 列出所有检查点
/checkpoint clear                      # 清除旧检查点（保留最后5个）

    使用技巧

    1
    命名规范
    使用描述性名称如 "core-done"、"refactor-start"
    2
    创建时机
    在完成重要阶段后创建检查点
    3
    验证时机
    在回溯问题或对比效果时验证检查点
    4
    清理策略
    定期运行 /checkpoint clear 保持日志整洁
    5
    状态对比
    检查点比较包含文件、测试和覆盖率三方面
    6
    工作流串联
    创建 → 实现 → 验证 → 修复确立标准工作流
