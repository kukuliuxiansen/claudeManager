PM2命令

基于PM2进程管理器的项目服务自动化配置工具

类型: Command类

实现效果

自动检测项目结构并生成完整的PM2配置和命令集合

  • 服务检测
    扫描项目识别前端/后端/数据库服务类型
  • 配置生成
    自动生成 ecosystem.config.cjs 等配置文件
  • 命令脚本
    生成单服务/全部服务的启停重启命令
  • CLAUDE.MD更新
    自动在项目文档中添加PM2使用说明

    • 核心原理

    PM2命令通过三阶段自动化配置流程工作：

    1. 阶段1：服务扫描 - 检测框架类型（Vite/Next/Nuxt/Express/FastAPI/Flask/Go）和对应端口
    2. 阶段2：配置生成 - 为每个服务生成专属配置文件和start.cjs包装器（Python服务需要）
    3. 阶段3：命令创建 - 生成一组便捷的命令脚本文件和PowerShell脚本

    对Windows系统特别优化，强制使用.cjs扩展名并正确配置Node.js和Python服务的运行路径。

    使用方法

    触发场景与调用 
    /pm2
# 自动检测项目服务并生成配置
# 默认端口优先级：用户指定 > .env > 配置文件 > 默认值

    使用技巧

    1
    首次运行后需执行 `pm2 save` 以保留进程列表
    2
    Python服务自动使用Node.js包装器（start.cjs）运行
    3
    所有命令文件内容简洁，仅包含1-2行描述+bash代码块
    4
    可通过 `/pm2-all-stop` 快速停止所有服务
    5
    使用 `/pm2-logs` 或 `/pm2-{port}` 查看实时日志
    6
    Windows用户注意：配置文件必须使用 .cjs 扩展名

已更新项目 CLAUDE.md

Claude命令：/pm2-all, /pm2-all-stop, /pm2-3000, /pm2-3000-stop, /pm2-logs, /pm2-status

终端命令（首次）：
pm2 start ecosystem.config.cjs && pm2 save

终端命令（后续）：
pm2 start all          # 启动全部
pm2 stop all           # 停止全部
pm2 restart all        # 重启全部
pm2 start {name}       # 启动单服务
pm2 stop {name}        # 停止单服务
pm2 logs               # 查看日志
pm2 monit              # 监控面板
