保存会话命令

在会话结束前保存工作状态到会话文件

类型: Command类

实现效果

保存会话命令捕获当前工作会话的完整上下文，包括已完成的工作、未成功尝试的记录和下一步计划，然后保存为带时间戳的会话文件供以后恢复。

  • 上下文收集
    汇总所有修改的文件、讨论的决策和遇到的错误
  • 结构化保存
    使用标准格式的会话文件，包含所有必要部分
  • 批准应用
    显示拟议内容并等待用户批准后才保存

    • 核心原理

    该命令通过以下步骤保存会话：

    1. 收集上下文：读取会话期间修改的所有文件，回顾讨论和决策
    2. 创建文件：在 ~/.claude/session-data/ 目录中创建 YYYY-MM-DD-shortid-session.tmp 文件
    3. 填充内容：按标准格式写入 Started、Last Updated、Project、What We Are Building 等所有必需部分
    4. 显示确认：向用户展示完整内容并请求批准

    使用方法

    触发场景与调用 
    /save-session

# 自动触发场景：
# - 在关闭 Claude Code 之前结束一个工作会话
# - 在达到上下文限制之前（先运行此命令，然后开始新会话）
# - 解决复杂问题后需要记住的任何时候
# - 需要将上下文交接给未来会话的任何时候

    使用技巧

    1
    不要跳过部分
    即使某些部分没有内容也要写入 "Nothing yet" 或 "N/A"
    2
    记录失败原因
    \"What Did NOT Work\" 部分对于未来会话至关重要
    3
    使用短 ID 格式
    新会话文件应使用 YYYY-MM-DD-shortid-session.tmp 格式
    4
    使用别名
    为频繁引用的会话创建别名以简化加载

    输出示例

    Session File Format 
    ```markdown
**Started:** ~2pm
**Last Updated:** 5:30pm
**Project:** my-app
**Topic:** Building JWT authentication with httpOnly cookies

## What We Are Building
User authentication with JWT tokens stored in httpOnly cookies.

## What Worked (With Evidence)
- Register endpoint - confirmed by Postman
- JWT generation - confirmed by unit test

## What Did Not Work (And Why)
- Next-Auth - conflicts with custom Prisma adapter
- localStorage for JWT - causes SSR hydration mismatch

## What Has Not Been Tried Yet
- Store JWT as httpOnly cookie
- Use cookies() from next/headers

## Current State Of Files
| File | Status | Notes |
|------|--------|-------|
| register/route.ts | Complete | Works |
| login/route.ts | In Progress | Token works, cookie not set |

## Decisions Made
- httpOnly cookie over localStorage
- Custom auth over Next-Auth

## Blockers & Open Questions
- Does cookies().set() work in Route Handler?

## Exact Next Step
Set JWT as httpOnly cookie in login/route.ts
```
