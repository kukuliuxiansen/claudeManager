提交命令

基于当前变更创建 Glasgow 提交

类型: Command类

实现效果

提交命令基于当前 git 变更创建单个提交，自动推断合适的提交消息。

  • 上下文感知
    基于当前 git 状态、差异和分支分析
  • 智能消息
    根据变更内容生成合适的提交消息
  • 单次提交
    将所有变更打包成单个提交

    • 核心原理

    提交命令使用上下文驱动的提交消息生成：

    1. 收集上下文 - 获取 git status、git diff 和当前分支信息
    2. 分析变更 - 解析变更内容，理解功能或修复类型
    3. 生成消息 - 根据变更类型生成符合约定的提交消息
    4. 执行提交 - staging 并创建提交

    提交消息格式：

    • feat: 新功能
    • fix: Bug 修复
    • docs: 文档更新
    • refactor: 代码重构
    • test: 测试相关

    使用方法

    触发方式 
    /commit

    当有需要提交的变更时运行此命令：

    执行流程 
    1. 检查 git status 和 git diff
2. 分析变更内容和类型
3. 生成合适的提交消息
4. staging 所有变更
5. 创建提交

    使用技巧

    1
    语义化提交
    提交消息遵循语义化约定
    2
    一次性提交
    所有相关变更应在一个提交中
    3
    明确消息
    提交消息应清晰描述变更
    4
    检查变更
    提交前确认变更正确性
    5
    避免大提交
    保持提交聚焦于单一功能
    6
    阶段性提交
    重要功能可分阶段提交

    输出示例

    上下文输入 
    - Current git status: !`git status`
- Current git diff (staged and unstaged changes): !`git diff HEAD`
- Current branch: !`git branch --show-current`
- Recent commits: !`git log --oneline -10`
    Your Task 
    Based on the above changes, create a single git commit.
You have the capability to call multiple tools in a single response.
Stage and create the commit using a single message. Do not use any
other tools or do anything else. Do not send any other text or
messages besides these tool calls.
    提交输出示例 
    // Tool calls (single response):
// 1. git add -A
// 2. git commit -m "feat: add user authentication"

// Example output:
Created commit abc1234:
  "feat: add user authentication"

  - Added auth middleware
  - Implemented JWT token generation
