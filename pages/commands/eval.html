评估命令

功能特性评估、回归测试和上线准备验证命令

类型: Command类

实现效果

为功能特性定义评估标准，运行评估检查并生成全面报告以验证上线准备状态。

  • 评估定义创建
    在.claude/evals/目录创建评估定义文件
  • 能力评估检查
    验证特性是否按预期工作（pass@3 > 90%）
  • 回归评估检查
    验证现有功能仍正常工作（pass^3 = 100%）
  • 全面评估报告
    生成包含指标和建议的综合报告
  • 评估列表查看
    查看所有评估定义及其状态

    • 核心原理

    评估命令基于可重复的验证框架，支持特性能力和回归保护：

    1. 定义评估：创建评估文件定义成功标准
    2. 运行检查：尝试验证每个标准并记录PASS/FAIL
    3. 生成报告：汇总结果、指标和推荐
    4. 上线决策：基于评估结果决定是否上线

    能力评估要求pass@3 > 90%（3次尝试中至少2次通过），回归评估要求pass^3 = 100%（3次尝试必须全部通过）。

    使用方法

    触发场景与调用 
    触发场景：
- 功能开发完成后验证
- 上线前的质量门控
- 集成测试完成后的确认
- 定期回归验证

调用方式：
在Claude Code中输入：
/eval define feature-name         # 定义新评估
/eval check feature-name          # 运行评估检查
/eval report feature-name         # 生成全面报告
/eval list                        # 查看所有评估
/eval clean                       # 清理旧评估日志

    使用技巧

    1
    先定义后检查：始终先定义评估标准再运行检查
    2
    能力与回归分离：明确区分特性能力和回归测试
    3
    记录成功标准：在评估文件中明确通过阈值
    4
    使用报告进行决策：基于报告的RECOMMENDATION字段决定
    5
    定期清理日志：使用eval clean保持评估目录整洁

    输出示例

    评估命令输出 
    ### 定义评估
/eval define feature-auth

创建评估文件 .claude/evals/feature-auth.md：
```markdown
Eval: feature-auth
Created: 2026-03-29
### Capability Evals
- [ ] 用户可以使用OAuth提供商登录
- [ ] 会话令牌在30分钟后过期
- [ ] 无效凭据返回401状态码
- [ ] 登录错误被正确处理且不泄露内部信息
- [ ] 两次连续失败后锁定账户
### Regression Evals
- [ ] 现有用户仍使用密码登录
- [ ] 注销功能正常清除服务器状态
### Success Criteria
- pass@3 > 90% for capability evals
- pass^3 = 100% for regression evals
```

### 检查评估
/eval check feature-auth

EVAL CHECK: feature-auth
========================
能力: 3/5 通过
回归: 2/2 通过
状态: IN PROGRESS

### 生成报告
/eval report feature-auth

EVAL REPORT: feature-auth
=========================
Generated: 2026-03-29

CAPABILITY EVALS
[oauth-login]: PASS (pass@2) - required retry
[session-expire]: PASS (pass@1)
[invalid-credentials]: PASS (pass@1)
[error-handling]: FAIL - 输入验证不足
[account-lockout]: PASS (pass@3)

REGRESSION EVALS
[password-login]: PASS
[logout-clears-state]: PASS

METRICS
Capability pass@1: 60%
Capability pass@3: 80%
Regression pass^3: 100%

NOTES
- oauth-login需要重试2次，表明可能有竞态条件
- error-handling评估显示输入验证不足，需要添加规范化

RECOMMENDATION
NEEDS WORK - 修复error-handling评估

### 上线决策矩阵
| 评估结果 | 推荐 | 动作 |
|---------|------|------|
| ALL PASS | SHIP | 准备上线 |
| SOME FAIL | NEEDS WORK | 修复失败的评估 |
| CRITICAL FAIL | BLOCKED | 不要上线 |
| NOT STARTED | NOT STARTED | 开始定义评估
