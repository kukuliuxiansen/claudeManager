多工作流命令

结构化 6 阶段工作流的统一开发命令

类型: Command类
everything-claude-code

实现效果

multi-workflow 命令提供结构化的 6 阶段工作流，协调 Codex（后端）、Gemini（前端）和 Claude（编排）多模型协作。适用于全栈功能开发、需要严格质量控制的项目、涉及多个领域的复杂任务。每个阶段包含质量检查，不符合标准则停止。

  • 6阶段工作流

    Structure: Research → Ideation → Plan → Execute → Optimize → Review

  • 多模型协作

    Codex（后端）+ Gemini（前端）+ Claude（编排）

  • 质量门禁

    每个阶段包含质量检查（评分 0-10），低于 7 分则停止

  • MCP 集成

    支持 ace-tool MCP 用于增强提示和上下文检索

    • 核心原理

    multi-workflow 命令的核心原理是结构化多阶段协作工作流。通过定义清晰的 6 个阶段和每个阶段的职责分配，实现系统化的开发流程。

    阶段详解:

    1. Phase 1: Research & Analysis - 需求理解、上下文收集、完整性评分
    2. Phase 2: Solution Ideation - Codex（后端）与 Gemini（前端）并行分析
    3. Phase 3: Detailed Planning - 多模型协同规划，生成计划文件
    4. Phase 4: Implementation - Claude 编码实现，遵循计划
    5. Phase 5: Code Optimization - Codex 与 Gemini 并行审查优化
    6. Phase 6: Quality Review - 最终评估、测试验证、交付确认

    全局规则:

    • 阶段顺序不可跳过（除非用户明确指示）
    • 外部模型零文件写权限，所有修改由 Claude 执行
    • 分数 < 7 时强制停止，需补充需求
    • 每个阶段完成后需要用户确认

    外部编排:

    • 对于需要独立 git 状态的并行工作器，使用 tmux/worktree
    • 轻量分析使用内置子代理
    • 参考 `node scripts/orchestrate-worktrees.js`

    使用方法

    当用户需要进行全栈开发或复杂功能开发时触发此命令。

    触发场景与调用 
    // 触发场景
1. "develop a new feature"
2. "build a complete module"
3. "implement this complex requirement"

// 命令调用
/workflow <任务描述>

// 示例：全栈功能
/workflow 实现用户注册和登录功能

// 示例：复杂模块
/workflow 构建订单管理系统，包含添加、编辑、删除、列表

// 示例：项目初始化
/workflow 初始化微服务项目架构

    使用技巧

    1

    完整描述 - 详细的任务描述确保每个阶段有足够信息

    2

    质量门禁 - 关注阶段评分，低于 7 分需补充需求

    3

    用户确认 - 每个阶段完成等待用户确认再继续

    4

    MCP 配置 - 确保 ace-tool MCP 可用以获得增强能力
