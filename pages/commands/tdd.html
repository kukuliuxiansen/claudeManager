TDD命令

驱动开发流程：编写测试 - 实现 - 重构

类型: Command类

实现效果

TDD 命令通过 tdd-guide agent 执行测试驱动开发流程，首先编写测试（RED），然后实现代码使其通过（GREEN），最后重构（REFACTOR）。

  • 接口沙箱

    首先定义类型和接口

  • 编写测试优先

    编写失败的测试（RED）

  • 最小化实现

    编写刚好通过测试的代码（GREEN）

  • 重构优化

    改进代码结构，保持测试通过（REFACTOR）

    • 核心原理

    TDD 命令的核心原理是测试驱动开发循环：编写测试 → 实现代码 → 重构优化。tdd-guide agent 遵循严格的 TDD 流程，确保代码质量和可维护性。

    三个阶段:

    • RED: 编写失败的测试，验证测试框架工作正常
    • GREEN: 编写最小化代码使测试通过
    • REFACTOR: 改进代码结构，保持所有测试通过

    质量目标:

    • 80%+ 测试覆盖
    • 测试独立且可重复
    • 测试命名清晰描述行为

    使用方法

    当需要实现新功能或修复 bug 时使用 TDD 命令。

    触发场景与调用 
    // 启动 TDD 流程
/tdd

// 示例：实现市场流动性分数计算器
/tdd 我需要一个计算市场流动性分数的函数

    使用技巧

    1

    先写测试 - 永远从编写失败的测试开始

    2

    最小化实现 - 只写刚好通过测试的代码

    3

    测试命名 - 使用描述性测试名称说明行为

    4

    覆盖边界 - 添加边界值和错误处理测试
