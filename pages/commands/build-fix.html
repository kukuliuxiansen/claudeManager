构建修复命令

自动检测构建系统并逐个修复编译错误

类型: Command类

实现效果

构建修复命令自动识别项目构建系统，运行构建命令，识别并修复所有错误，然后验证修复结果。

  • 智能识别构建系统
    自动检测 package.json、Cargo.toml、pom.xml 等配置文件
  • 分组修复错误
    按文件分组错误，按依赖顺序修复
  • 一次修复一个错误
    确保每次修复的最小化和可验证性

    • 核心原理

    构建修复采用五步工作流程：

    1. 步骤 1: 检测构建系统 - 根据项目文件（package.json、Cargo.toml、pom.xml 等）确定构建命令
    2. 步骤 2: 分析错误 - 运行构建命令，按文件分组错误，按依赖顺序排序
    3. 步骤 3: 逐个修复 - 读取文件 → 诊断问题 → 最小修复 → 重新构建 → 验证成功
    4. 步骤 4: 安全防护 - 如果修复引入更多错误或超过3次尝试失败，停止并询问用户
    5. 步骤 5: 输出摘要 - 显示修复数量、剩余错误、新引入的错误和下一步建议

    支持的构建系统：

    • JavaScript/TypeScript - npm run build / npx tsc --noEmit
    • Rust - cargo build
    • Java - mvn compile / ./gradlew compileJava
    • Go - go build ./...
    • Python - python -m compileall / mypy

    使用方法

    触发方式 
    /build-fix

    当构建失败时运行该命令，Claude 将自动：

    自动执行流程 
    1. 根据项目结构识别构建系统
2. 运行对应的构建命令
3. 解析错误输出并按文件分组
4. 逐个读取文件并诊断问题
5. 使用最小修改修复每个错误
6. 重新运行构建验证修复
7. 输出修复摘要报告

    使用技巧

    1
    先运行构建
    确保你能在终端看到相同的错误输出
    2
    修复依赖顺序
    先修导入和类型错误，再修逻辑错误
    3
    避免重构
    优先使用最小差异修复，而不是大范围重构
    4
    缺失依赖处理
    如果是缺少依赖，需要先安装再构建
    5
    循环错误检查
    同一个错误反复出现3次可能是设计问题
    6
    询问确认
    修复引入更多错误时停止并询问用户

    输出示例

    修复类型对照表 
    | 问题类型 | 诊断方法 | 修复策略 |
|---------|---------|---------|
| 缺少模块/导入 | 检查文件顶部的 import 语句 | 验证包已安装，建议安装命令 |
| 类型不匹配 | 读取类型定义 | 用更宽的类型修复，或更精确地使用 |
| 循环依赖 | 用 import 图识别循环 | 建议提取公共模块 |
| 版本冲突 | 检查 package.json/Cargo.toml | 查看版本约束，建议兼容版本 |
| 构建工具配置错误 | 读取配置文件，对比默认值 | 更新配置以匹配项目需求
