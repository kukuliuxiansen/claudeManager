更新配置命令

配置 Claude Code harness 的 settings.json

类型: Command类

实现效果

更新配置命令用于配置 Claude Code harness 的 settings.json 文件。自动化行为需要在 settings.json 中配置 hooks - harness 会执行这些操作，而不是 Claude。

  • 自动化行为配置

    配置 hooks 实现自动化触发行为

  • Settings.json 管理

    直接修改 Claude Code 的配置文件

  • Hook 配置

    配置 PreToolUse、PostToolUse、Stop 等 hook 类型

  • 权限控制

    配置 allowedTools、危险操作权限等安全设置

    • 核心原理

    更新配置命令的核心原理是直接修改 settings.json 配置文件。Claude Code harness 会读取这个配置文件来决定自动化行为、hooks 执行、工具权限等。

    配置文件位置:

    • 全局配置: ~/.claude/settings.json
    • 项目配置: .claude/settings.json

    Hook 类型:

    • PreToolUse: 工具执行前触发（验证、参数修改）
    • PostToolUse: 工具执行后触发（自动格式化、检查）
    • Stop: 会话结束时触发（最终验证）

    自动化行为触发词:

    • "from now on when X"
    • "each time X"
    • "whenever X happens"
    • "before/after X"

    使用方法

    当用户需要配置自动化行为或修改 settings.json 时触发此命令。

    触发场景与调用 
    // 触发场景
1. 用户说 "from now on when I commit, run tests"
2. 用户说 "each time I create a file, add header"
3. 用户说 "whenever X happens, do Y"
4. 用户说 "before/after X, run Y"

// 通过命令调用
/update-config

// 示例配置 (settings.json)
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Bash",
        "hooks": ["validate-command.sh"]
      }
    ],
    "PostToolUse": [
      {
        "matcher": "Write",
        "hooks": ["format-file.sh"]
      }
    ]
  },
  "allowedTools": ["Read", "Write", "Bash"],
  "autoApprove": false
}

    使用技巧

    1

    区分项目/全局 - 项目配置优先于全局配置

    2

    谨慎 autoApprove - 只对可信计划启用自动批准

    3

    Hook 调试 - 使用日志验证 hook 执行情况

    4

    权限细化 - 按需配置 allowedTools 而非全部开放
