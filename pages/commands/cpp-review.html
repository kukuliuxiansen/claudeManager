C++代码审查命令

自动化C++代码审查，检查内存安全、并发安全和现代C++规范

类型: Command类

实现效果

自动执行C++代码审查，识别内存安全、并发安全和代码规范问题，并生成分类的审查报告。

  • 修改文件识别
    通过git diff查找.mcpp、.hpp、.cc、.h文件的修改
  • 静态分析运行
    执行clang-tidy和cppcheck进行静态代码分析
  • 内存安全扫描
    检查原生new/delete、缓冲区溢出、use-after-free等问题
  • 并发安全分析
    分析线程安全、互斥锁使用、数据竞争等问题
  • 现代C++检查
    验证代码是否符合C++17/20规范和最佳实践

    • 核心原理

    该命令采用多层审查策略，结合静态分析工具和C++知识库进行综合评估：

    1. 静态分析：使用clang-tidy和cppcheck识别代码模式问题
    2. 内存安全检查：扫描原生new/delete使用、缓冲区边界、指针生命周期
    3. 并发安全分析：检查互斥锁使用、线程同步、数据竞争模式
    4. 现代C++规则：基于C++17/20规范检查代码风格和最佳实践
    5. 严重性分类：将问题分为CRITICAL、HIGH、MEDIUM三个等级

    审查结果用于指导合并决策，CRITICAL和HIGH级别的问题应修复后才能合并。

    使用方法

    触发场景与调用 
    触发场景：
- C++代码编写或修改完成后
- 提交代码前进行质量检查
- 代码审查环节
- 新成员加入C++项目时
- 怀疑有内存安全问题时

调用方式：
在Claude Code中输入：
/cpp-review

审查类别：
- CRITICAL (必须修复)
- HIGH (应当修复)
- MEDIUM (考虑修复)

    使用技巧

    1
    结合/cpp-test使用：先运行cpp-test确保测试通过
    2
    结合/cpp-build使用：如果构建出错，先用cpp-build修复
    3
    在PR前运行：代码审查应在合并前执行
    4
    关注CRITICAL问题：内存安全问题必须优先修复
    5
    定期运行：定期审查代码质量，防止技术债务积累

### 审查的文件
- src/handler/user.cpp (已修改)
- src/service/auth.cpp (已修改)

### 静态分析结果
✓ clang-tidy: 2个警告
✓ cppcheck: 无问题

### 发现的问题

[CRITICAL] 内存泄漏
文件: src/service/auth.cpp:45
问题: 原生new未匹配delete
```cpp
auto* session = new Session(userId);  // 内存泄漏！
cache[userId] = session;
```
修复: 使用std::unique_ptr
```cpp
auto session = std::make_unique(userId);
cache[userId] = std::move(session);
```

[HIGH] 缺少const引用
文件: src/handler/user.cpp:28
问题: 大对象按值传递
```cpp
void processUser(User user) {  // 不必要的拷贝
```
修复: 使用const引用传递
```cpp
void processUser(const User& user) {
```

| ❌ 阻止 | 发现CRITICAL或HIGH问题 |
