清理已删除分支命令

删除远程已删除的本地分支及其关联的工作树

类型: Command类

实现效果

清理已删除分支命令用于同步本地和远程分支状态，删除远程已删除但本地仍存在的分支，以及清理相关的工作树。

  • 列出过期分支
    识别标记为 [gone] 的本地分支
  • 工作树清理
    识别并删除关联的工作树
  • 分支删除
    安全删除所有过期分支

    • 核心原理

    清理命令使用三步流程同步本地和远程状态：

    1. 列出现有分支 - 运行 `git branch -v` 识别状态
    2. 检查工作树 - 运行 `git worktree list` 找出关联的工作树
    3. 清理操作 - 对于标记为 [gone] 的分支，先删除工作树再删除分支

    关键处理逻辑：

    • 带有 [gone] 状态的分支表示远程已删除
    • 带有 + 前缀的分支有关联工作树，必须先删除工作树
    • 使用 `git branch -D` 强制删除过期分支

    使用方法

    触发方式 
    /clean-gone

    运行清理命令后，将自动执行：

    清理流程 
    1. 运行 git branch -v 列出所有分支
2. 运行 git worktree list 获取工作树信息
3. 查找标记为 [gone] 的分支
4. 删除关联的工作树（如有需要）
5. 删除分支
6. 输出清理摘要报告

    使用技巧

    1
    定期清理
    定期运行清理命令保持分支列表整洁
    2
    检查工作树
    特别注意带有 + 前缀的分支
    3
    主动拉取
    运行 git fetch --prune 后再清理
    4
    确认无未推送工作
    清理前确保没有未推送的重要工作
    5
    备份工作树
    如有重要工作树，先手动备份
    6
    只在主目录运行
    确保在 git 仓库根目录运行
