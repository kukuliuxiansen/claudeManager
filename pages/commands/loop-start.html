循环启动命令

启动自动化循环工作流，支持多种循环模式

类型: Command类
everything-claude-code

实现效果

配置和启动自动化循环工作流，支持多种循环模式和质量门禁设置。

  • 多种循环模式
    支持 sequential、continuous-pr、rfc-dag、infinite 模式
  • 安全与快速模式
    通过 --mode 参数选择严格 gates 或快速模式
  • 自动配置钩子
    为所选模式启用相应钩子和profile
  • 生成运行手册
    在 .claude/plans/ 下创建循环计划

核心原理

循环启动命令配置自动化工作流：

命令格式 
/loop-start [pattern] [--mode safe|fast]

处理流程：

  1. 确认仓库状态和分支策略
  2. 选择循环模式和模型层级策略
  3. 为所选模式启用所需钩子/profile
  4. 创建循环计划并在 .claude/plans/ 下写入运行手册
  5. 打印启动和监控循环的命令

使用方法

启动循环 
# 使用默认安全模式的 sequential 循环
/loop-start sequential

# 使用快速模式的 continuous-pr 循环
/loop-start continuous-pr --mode fast

# 使用默认模式（安全）
/loop-start

循环模式说明：

模式 描述
sequential 顺序执行任务，每个任务完成后开始下一个
continuous-pr 持续PR模式，自动创建和审查PR
rfc-dag RFC决策图模式，用于架构决策流程
infinite 无限循环模式，持续运行直到手动停止

使用技巧

1
安全第一：生产环境优先使用安全模式，确保质量门禁严格
2
指定模式：明确指定模式以避免混淆，不要依赖默认值
3
检查准备：启动前确保测试通过且无紧急错误
4
停止条件：始终确保循环具有明确的停止条件

输出示例

循环计划生成 
$ /loop-start continuous-pr --mode fast

计划生成中...
配置：continuous-pr 模式 (fast 模式)

✅ 仓库状态检查：通过
✅ 分支策略确认：feature/automation-branch
✅ 必需钩子已启用：ECC_HOOK_PROFILE

运行手册已创建：.claude/plans/continuous-pr-fast.md

启动循环命令：
```bash
# 运行循环
claude --loop "continuous-pr" --mode fast

# 监控循环状态
claude --loop-status

# 停止循环
claude --loop-stop
```

注意事项：
- Fast模式跳过部分质量门禁以提高速度
- 确保循环有明确的停止条件
- 检查每日限额以避免超出API预算
所需安全检查 
启动循环前必须验证：
- [ ] 测试通过：./gradlew test
- [ ] 静态分析通过：./gradlew detekt
- [ ] 安全扫描通过：./gradlew securityCheck
- [ ] ECC_HOOK_PROFILE 未全局禁用
- [ ] 循环具有明确的停止条件
