提交推送 PR 命令

完整 PR 工作流：新分支、提交、推送、创建 Pull Request

类型: Command类

实现效果

提交推送 PR 命令自动执行完整的 Pull Request 工作流，从创建新分支到推送代码再到创建 PR。

  • 分支创建
    如果在 main/master 分支则创建新功能分支
  • 自动提交
    根据变更内容生成合适的提交消息
  • 推送远程
    将分支推送到 origin 远程仓库
  • 创建 PR
    使用 gh pr create 创建 Pull Request

    • 核心原理

    完整 PR 工作流采用顺序执行的四个步骤：

    1. 分支检查 - 检测当前是否在 main/master 分支，如果是则创建新分支
    2. 提交变更 - 基于 git diff 生成合适的提交消息并创建提交
    3. 推送分支 - 使用 git push -u 将分支推送到远程
    4. 创建 PR - 调用 `gh pr create` 创建 Pull Request

    关键约束：

    • 所有操作必须在单个响应中完成
    • 不得使用其他工具或执行其他操作
    • 仅执行这五个操作后停止

    使用方法

    触发方式 
    /commit-push-pr

    当有 변경 코드 ready 时运行此命令：

    执行流程 
    1. If on main/master: create new feature branch
2. Create commit with appropriate message
3. Push branch to origin
4. Create PR using `gh pr create`
5. Report PR URL and status

    使用技巧

    1
    分支命名
    分支名称应反映功能或修复内容
    2
    提交消息
    提交消息应清晰描述变更内容
    3
    PR 标题
    PR 标题应简洁描述功能
    4
    PR 描述
    在 PR 中添加详细的变更说明
    5
    reviewers
    PR 中指定合适的 reviewers
    6
    关联问题
    PR 中关联相关 issue
