多计划命令

多模型协作规划与计划生成命令

类型: Command类
everything-claude-code

实现效果

multi-plan 命令通过多模型协作生成开发计划。并行调用 Codex 和 Gemini 进行分析与规划，结合双方优势生成全面的实现方案。适用于复杂功能规划、跨领域任务规划、需要多视角评审的规划场景。

  • 并行分析

    Codex（后端）与 Gemini（前端）并行分析任务

  • 跨领域规划

    同时考虑后端架构与前端实现的完整方案

  • 计划持久化

    自动保存到 .claude/plan/ 目录，便于后续执行

  • SESSION_ID 交换

    包含 CODEX_SESSION 和 GEMINI_SESSION 供后续 /ccg:execute 使用

    • 核心原理

    multi-plan 命令的核心原理是并行模型分析与计划综合。通过同时调用 Codex 和 Gemini 获取多视角的规划方案，再由 Claude 综合生成最终计划。

    关键特性:

    • 并行调用 -Codex 和 Gemini 同时分析，减少等待时间
    • 信任规则 - 后端遵循 Codex，前端遵循 Gemini
    • 零写权限 - 模型不直接修改文件，只输出规划方案
    • 计划保存 - 自动生成 `.claude/plan/xxx.md` 文件

    生成内容:

    • 技术方案 - 最优解决方案与理由
    • 实施步骤 - 详细的分步实现计划
    • 关键文件 - 需要修改的文件列表与修改描述
    • SESSION_ID - CODEX_SESSION 和 GEMINI_SESSION
    • 风险评估 - 潜在风险与缓解措施

    后续使用:

    • 用户确认后，使用 `/ccg:execute .claude/plan/xxx.md` 执行
    • 系统自动复用保存的 SESSION_ID 恢复上下文

    使用方法

    当用户需要为复杂任务生成多视角的开发计划时触发此命令。

    触发场景与调用 
    // 触发场景
1. "create a plan for user authentication"
2. "plan the implementation of this feature"
3. "what's the best approach for this task?"

// 命令调用
/ccg:plan [task-description]

// 示例：简单功能规划
/ccg:plan 实现用户登录功能

// 示例：复杂功能规划
/ccg:plan 构建用户管理系统，包含注册、登录、权限管理

// 示例：现有对话的上下文
/ccg:plan related to our previous discussion about API design

    使用技巧

    1

    描述清晰 - 完整的任务描述帮助生成更准确的计划

    2

    等待批准 - 计划生成后等待用户批准再执行

    3

    保存计划 - 计划会自动保存到 .claude/plan/ 目录

    4

    ETHER 风格 - 计划使用英文与模型通信，中文与用户沟通
