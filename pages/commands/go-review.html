Go代码审查报告

多维度审查Go代码质量与安全性

类型: Command类

实现效果

对Go代码进行全面审查，覆盖安全、并发、规范等多个维度：

  • 静态分析工具扫描
    go vet、staticcheck、golangci-lint 自动检测
  • 安全性漏洞检测
    SQL注入、命令注入、竞态条件、硬编码凭据
  • Go规范性检查
    函数命名、错误处理、接口设计、包结构

    • 核心原理 

    
    审查维度与流程
    

    
Step 1: 查找Go变更
├─ git diff # 查找修改的 .go 文件
└─ 确定审查范围

Step 2: 运行静态分析
├─ go vet ./...            # 编译器级检查
├─ staticcheck ./...       # 高级静态分析
├─ golangci-lint run       # Lint 检查
├─ go build -race ./...    # 竞态检测
└─ govulncheck ./...       # 安全漏洞扫描

Step 3: 代码审查 (人类检查)
├─ 安全性审查
│  ├─ SQL/Command 注入
│  ├─ 竞态条件
│  ├─ 硬编码凭据
│  └─ 不安全的指针使用
├─ 并发性审查
│  ├─ Goroutine 泄漏
│  ├─ 通道使用模式
│  ├─ 互斥锁保护
│  └─ 上下文传播
├─ Go风格审查
│  ├─ 函数命名约定
│  ├─ 错误处理模式
│  ├─ 接口设计
│  └─ 文件组织
└─ 生成审查报告

Step 4: 按严重性分类
├─ CRITICAL (必须修复)
├─ HIGH (应该修复)
├─ MEDIUM (考虑修复)
└─ LOW (可选优化)

    使用方法

    触发场景与调用 
    # 在编写或修改 Go 代码后触发

"/go-review"
格式: 审查 Go 代码质量与安全性
要求: 运行 /go-review 开始审查

示例用法：
1. 提交代码前运行审查
2. MR/PR 审查阶段
3. 重构后验证代码质量

关键点：
- 审查所有修改的 .go 文件
- 并行运行多种分析工具
- 生成按严重性排序的报告
- 提供修复建议

    使用技巧

    1
    先运行 `/go-test` 确保测试通过，再进行代码审查
    2
    先运行 `/go-build` 修复编译错误，再审查代码
    3
    重点关注 CRITICAL 级别问题，特别是安全性问题
    4
    审查并发代码时，特别注意互斥锁的正确使用
    5
    确保所有公共函数都有 godoc 注释
    6
    错误处理应该使用 fmt.Errorf 包装上下文

 静态分析结果
✅ go vet: 未发现问题

 发现的问题

[CRITICAL] 竞态条件
文件：internal/service/auth.go:45
问题：共享 map 未同步访问
```go
var cache = map[string]*Session{}  // 并发访问！
func GetSession(id string) *Session {
    return cache[id]  // 竞态条件
}
```
修复：使用 sync.RWMutex 或 sync.Map
```go
var (
    cache   = map[string]*Session{}
    cacheMu sync.RWMutex
)
func GetSession(id string) *Session {
    cacheMu.RLock()
    defer cacheMu.RUnlock()
    return cache[id]
}
```

[HIGH] 缺失错误上下文
文件：internal/handler/user.go:28
问题：错误返回时缺少上下文
```go
return err  // 无上下文
```
修复：使用 fmt.Errorf 包装
```go
return fmt.Errorf("获取用户 %s: %w", userID, err)
```

[LOW] 缺少 godoc 注释
文件：internal/service/user.go
问题：导出函数缺少文档注释
```go
// GetUser 根据ID获取用户信息
func GetUser(id string) (*User, error) {
    // ...
}
```
