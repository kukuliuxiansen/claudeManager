更新文档命令

从代码源生成和更新文档

类型: Command类

实现效果

更新文档命令从项目中的代码源文件自动生成文档。它识别 source of truth 并生成或更新开发者文档。

  • 识别源文件

    从 package.json、.env.example 等文件提取信息

  • 生成脚本参考

    从 package.json 提取 npm 脚本描述

  • 环境变量文档

    从 .env.example 生成环境变量参考

  • 更新文档文件

    生成 CONTRIBUTING.md 和 RUNBOOK.md

    • 核心原理

    更新文档命令的核心原理是从代码源文件提取信息并生成文档。它读取 package.json、.env.example、openapi.yaml 等文件，自动填充开发者文档。

    源文件:

    • package.json → scripts 参考
    • .env.example → 环境变量文档
    • openapi.yaml → API 端点参考
    • 源代码导出 → 公共 API 文档
    • Dockerfile/docker-compose.yml → 基础设施设置文档

    生成的文档:

    • CONTRIBUTING.md → 开发设置和流程
    • ENV.md → 环境变量参考
    • RUNBOOK.md → 部署和故障排除

    使用方法

    当需要更新项目文档或在重大变更后保持文档同步时使用此命令。

    触发场景与调用 
    // 更新所有文档
/update-docs

// 用于以下场景：
- 添加新 npm 脚本后
- 修改环境变量后
- API 更改后

    使用技巧

    1

    源即真理 - 总是从代码而不是手动编辑生成的区域

    2

    保留手动部分 - 只更新生成部分；保留手动编写的 prose

    3

    标记生成内容 - 使用 AUTO-GENERATED 标记

    4

    检查陈旧度 - 标记超过 90 天未更新的文档
