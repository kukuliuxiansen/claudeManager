Rust 审查命令

全面审查 Rust 代码的安全性、所有权模式和最佳实践

类型: Command类

实现效果

Rust 审查命令在代码变更上运行静态分析工具，识别安全问题、所有权模式问题和编码最佳实践违规，并按严重程度分类结果。

  • 自动化静态分析
    运行 cargo check、cargo clippy 和 cargo fmt 验证自动化检查
  • 安全审查
    检查不安全代码、命令注入和硬编码凭证
  • 所有权分析
    检查不必要的克隆、生命周期和借用模式

    • 核心原理

    该命令遵循以下审查流程：

    1. 自动化检查：运行 cargo check、cargo clippy、cargo fmt 和 cargo test
    2. 识别变更：使用 git diff HEAD~1 找到修改的 .rs 文件
    3. 安全扫描：检查不安全代码、命令注入、硬编码凭据
    4. 所有权审查：分析克隆模式、生命周期和借用模式
    5. 生成报告：按严重程度（关键/高/中）分类结果

    使用方法

    触发场景与调用 
    /rust-review

# 自动触发场景：
# - 编写或修改 Rust 代码后
# - 提交 Rust 更改前
# - 审查包含 Rust 代码的拉取请求时
# - 新加入 Rust 代码库时
# - 学习惯用 Rust 模式时

    使用技巧

    1
    先运行测试
    在审查前运行 /rust-test 确保测试通过
    2
    修复关键问题
    在合并前必须修复关键问题
    3
    使用 /rust-build
    如果有构建错误，先运行 /rust-build
    4
    合并前重新审查
    修复后重新运行审查验证

    输出示例

    Rust Review Report 
    ### CRITICAL (Must Fix)
- Unchecked `unwrap()`/`expect()` in production code paths
- `unsafe` without `// SAFETY:` comment
- SQL injection via string interpolation
- Command injection via unvalidated input
- Hardcoded credentials
- Use-after-free via raw pointers

### HIGH (Should Fix)
- Unnecessary `.clone()` to satisfy borrow checker
- `String` parameter where `&str` suffices
- Blocking in async context
- Missing `Send`/`Sync` bounds
- Wildcard `_ =>` match on critical enums
- Large functions (>50 lines)

### MEDIUM (Consider)
- Unnecessary allocation in hot paths
- Missing `with_capacity` when size known
- Suppressed clippy warnings
- Public API without `///` documentation

## Summary
- CRITICAL: 1
- HIGH: 1
- MEDIUM: 0
Recommendation: Block merge until CRITICAL issue is fixed

## Approval Criteria
| Status | Condition |
|--------|-----------|
| Approve | No CRITICAL or HIGH issues |
| Warning | Only MEDIUM issues |
| Block | CRITICAL or HIGH issues found |
