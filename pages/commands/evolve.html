演进命令

分析立驱动指令并将其演进为更高阶的复合命令、技能或代理

类型: Command类

实现效果

分析用户指令并自动聚类相关模式，将其演进为结构化的命令、技能或代理组件。

  • 指令读取
    读取项目和全局instincts文件，项目scope优先处理命名冲突
  • 按模式分组
    按触发器/领域模式聚类instincts，识别重复模式
  • 演进分类
    将聚类分类为：Command（用户调用）、Skill（自动触发）、Agent（复杂过程）
  • 文件生成
    执行--generate时，在evolved/目录生成结构化组件文件
  • 提升建议
    当适用时，显示project-to-global提升候选

    • 核心原理

    演进命令识别模式并将其提升为更高阶的抽象组件：

    1. 资料收集：读取instinct文件并按 scope (project/global) 分组
    2. 聚类分析：基于触发器/域模式匹配将related instincts分组
    3. 分类决策：
      • Command: 多个user-invoked actions
      • Skill: trigger-based automatic behaviors
      • Agent: 复杂的multi-step workflows
    4. 文件生成：--generate标志时创建evolved/目录文件

    此流程将低级instinct提升为可复用、可维护的组件。

    使用方法

    触发场景与调用 
    触发场景：
- 识别多个相关的instinct模式
- 将工作流提升为共享组件
- 创建可复用的命令、技能或代理
- 分析instinct质量并生成文档

调用方式：
在Claude Code中输入：
/evolve                    # 仅分析instincts
/evolve --generate         # 分析并生成文件

输出内容：
- 分析结果：聚类详情、confidence分数
- 建议：Command/Skill/Agent分类
- 文件：生成的Markdown文件（若启用--generate）

    使用技巧

    1
    先分析后生成：先运行/evolve不带标志查看建议，确认后再用--generate
    2
    关注置信度：仅考虑≥80%置信度的instincts进行提升
    3
    检查聚类大小：Skill候选需要2+个instincts聚类
    4
    使用--generate谨慎：文件生成后可能需要手动调整
    5
    查看生成内容：生成文件后审阅并根据项目标准调整

文件格式示例：

### Command
```markdown
name: new-table
description: Create a new database table with migration, schema update, and type generation
command: /new-table
evolved_from:
  - new-table-migration
  - update-schema
  - regenerate-types
---

1. Run migration script
2. Update database schema
3. Regenerate types
```

### Skill
```markdown
name: functional-patterns
description: Enforce functional programming patterns
evolved_from:
  - prefer-functional
  - use-immutable
  - avoid-classes
---

- Use pure functions when possible
- Avoid mutable state
- Prefer composition over inheritance
```

### Agent
```markdown
name: debugger
description: Systematic debugging agent
model: sonnet
evolved_from:
  - debug-check-logs
  - debug-isolate
  - debug-reproduce
  - debug-verify
---

1. Check application logs
2. Isolate the failing component
3. Create minimal reproduction
4. Verify fix with test
```

### 成长路径
从 instinct → Command/Skill/Agent:

**Command (用户调用)**
- 多个关于"user asks to..."的instincts
- 明确的repeatable sequence
- Example: new-table-step1, step2, step3 → /new-table

**Skill (自动触发)**
- Pattern-matching triggers
- Error handling responses
- Code style enforcement
- Example: prefer-functional, use-immutable → functional-patterns

**Agent (需要深度/隔离)**
- 复杂的multi-step workflows
- Debugging workflows
- Research tasks
- Example: debug-step1-4 → debugger agent
