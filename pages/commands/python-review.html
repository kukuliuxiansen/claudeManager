Python审查命令

静态分析Python代码质量的安全和风格审查工具

类型: Command类

实现效果

审查Python代码变更并生成多维度分析报告

  • 变更检测
    通过git diff识别修改的Python文件
  • 静态分析
    运行ruff/mypy/pylint/black检查
  • 安全扫描
    检查SQL注入、命令注入、不安全反序列化
  • 结果分类
    按严重程度分类（CRITICAL/HIGH/MEDIUM）

    • 核心原理

    Python审查命令通过多工具组合进行代码分析：

    1. 变更检测 - 使用git diff识别已修改的.py文件
    2. 格式检查 - 运行black --check/isort --check-only
    3. 类型检查 - 运行mypy进行静态类型分析
    4. 代码质量 - 运行ruff和pylint进行lint检查
    5. 安全扫描 - 运行bandit检测常见安全漏洞

    仅审查变更文件，确保快速反馈。

    使用方法

    触发场景与调用 
    /python-review
# 审查当前项目中已修改的Python代码

    使用技巧

    1
    在提交前运行以确保代码质量
    2
    结合 /tdd 确保测试通过后再审查
    3
    CRITICAL问题必须修复才能合并
    4
    CLK（common language keyword）错误用法可通过black自动修复
    5
    Django/FastAPI/Flask项目有框架特定检查规则
    6
    运行black地区进行格式化处理

=== 发现问题 ===

[CRITICAL] SQL注入漏洞
文件: app/routes/user.py:42
问题: 用户输入直接拼接到SQL查询中
```python
query = f"SELECT * FROM users WHERE id = {user_id}"  # 错误
```
修复: 使用参数化查询
```python
query = "SELECT * FROM users WHERE id = %s"
cursor.execute(query, (user_id,))
```

[HIGH] 可变默认参数
文件: app/services/auth.py:18
问题: 可变默认参数导致状态共享
```python
def process_items(items=[]):  # 错误
    items.append("new")
    return items
```
修复: 使用None作为默认值
```python
def process_items(items=None):  # 正确
    if items is None:
        items = []
    items.append("new")
    return items
```

[MEDIUM] 缺少类型注解
文件: app/services/auth.py:25
问题: 公共函数缺少类型注解
```python
def get_user(user_id):  # 错误
    return db.find(user_id)
```
修复: 添加类型注解
```python
def get_user(user_id: str) -> Optional[User]:  # 正确
    return db.find(user_id)
```
