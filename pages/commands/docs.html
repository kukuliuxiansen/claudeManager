文档查询命令

查找库、框架或API的最新文档并返回摘要答案

类型: Command类

实现效果

通过Context7 MCP工具获取库、框架或API的最新文档，返回包含相关代码片段的摘要答案。

  • 库ID解析
    调用resolve-library-id获取Context7兼容的库ID
  • 文档查询
    使用query-docs工具以库ID和问题查询文档
  • 摘要生成
    返回简明答案并包含相关代码示例
  • 出处标注
    如果Context7不可用，说明并从训练数据回答

    • 核心原理

    该命令使用Context7 MCP工具获取实时文档，确保答案反映当前文档而非训练数据：

    1. 解析库ID：调用resolve-library-id获取Context7兼容的库标识符
    2. 查询文档：使用query-docs工具获取相关文档内容
    3. 生成摘要：基于获取的文档内容生成简明答案
    4. 代码示例：包含相关代码片段帮助用户理解

    如果Context7 MCP不可用，命令会明确说明并从训练数据回答，同时标注可能过时。

    使用方法

    触发场景与调用 
    触发场景：
- 查询库、框架或API的最新文档
- 需要特定功能的使用方法
- 查看版本特定的API用法
- 查找代码示例

调用方式：
在Claude Code中输入：
/docs [库名] [问题]

示例：
/docs "Next.js" "如何配置中间件？"
/docs React "useEffect依赖数组最佳实践"
/docs "Prisma ORM" "如何执行原生SQL查询"

注意：对于多词参数，请使用引号将它们解析为单个令牌。

    使用技巧

    1
    使用引号：多词参数用引号包裹确保正确解析
    2
    指定版本：如果相关，包含版本号以获取准确文档
    3
    具体问题：问得越具体，返回的答案越精确
    4
    比搜索更快：对于熟悉库，/docs比手动搜索文档更快
    5
    检查过时提示：如果Context7不可用，注意标注的过时信息

### 总结答案
Next.js 中间件配置（v13+）

Next.js 13+ 引入了基于文件系统的中间件系统。创建 middleware.ts 文件：

```typescript
// middleware.ts
import { NextResponse } from 'next/server'
import type { NextRequest } from 'next/server'

export function middleware(request: NextRequest) {
  // 获取cookie
  const token = request.cookies.get('token')?.value

  // 保护路由
  if (request.nextUrl.pathname.startsWith('/admin') && !token) {
    return NextResponse.redirect(new URL('/login', request.url))
  }

  return NextResponse.next()
}

// 配置匹配器
export const config = {
  matcher: ['/admin/:path*', '/api/auth/:path*']
}
```

### 相关配置

中间件匹配器选项：
- `matcher` - 数组或字符串路径
- `skip` - 跳过路径数组
- `only` - 仅匹配路径数组

版本: Next.js 13.0+
文档来源: https://nextjs.org/docs middleware

### 总结答案
React useEffect依赖数组最佳实践

### 空依赖数组
```javascript
useEffect(() => {
  // 仅在挂载时运行一次
  const subscription = fetchOrCreateSubscription()
  return () => subscription.unsubscribe()
}, []) // ← 空数组 = 仅挂载/卸载
```

### 依赖数组包含状态
```javascript
useEffect(() => {
  // 在count更改时运行
  document.title = `Count: ${count}`
}, [count]) // ← 包含count
```

### 不要立即调用函数
```javascript
// ❌ 错误：每次渲染时都会调用 fetchData()
useEffect(() => {
  fetchData()
}, [fetchData()])

// ✅ 正确：引用函数
useEffect(() => {
  fetchData()
}, [fetchData])

// 或使用 useCallback
const fetchData = useCallback(() => {
  // ...
}, [])
```

### React Hooks规则
- 总是将依赖数组作为第二个参数传递
- 依赖数组中包含_EFFECT_中使用的所有值
- 使用useCallback/useMemo创建稳定的引用
- 开启eslint-plugin-react-hooks规则
