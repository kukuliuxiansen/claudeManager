PR审查命令

并行运行多个审查代理，对拉取请求进行全面代码审查

类型: Command类

实现效果

PR审查命令通过并行运行多个专业审查代理，对代码变更进行全面分析，覆盖代码质量、测试、错误处理等多个维度。该命令识别变更文件并自动选择适用的审查类型。

  • 智能审查范围
    自动检测 git diff 变更文件并选择适用的审查类型
  • 多维度审查
    支持注释、测试、错误处理、类型、代码质量和简化等多个审查维度
  • 结果聚合
    按严重程度分类审查结果（关键/重要/建议）

    • 核心原理

    该命令通过以下流程进行 PR 审查：

    1. 确定审查范围：检查 git 状态识别变更文件，解析用户参数确定审查维度
  • 识别审查类型：根据变更文件类型自动选择适用的审查（代码/测试/注释/错误处理/类型）
  • 并行启动代理：一次性启动所有适用的审查代理或按顺序逐个审查
  • 聚合结果：整合所有代理的报告，按严重程度分类并生成操作计划

    • 使用方法

    触发场景与调用 
    # 完整审查（默认）
/pr-review-toolkit:review-pr

# 特定审查维度
/pr-review-toolkit:review-pr tests errors
/pr-review-toolkit:review-pr comments
/pr-review-toolkit:review-pr simplify

# 并行审查
/pr-review-toolkit:review-pr all parallel

    使用技巧

    1
    尽早运行
    在创建 PR 之前运行，而非之后
    2
    关注变更
    代理默认分析 git diff 的变更内容
    3
    分优先级修复
    先处理关键问题，再处理重要问题
    4
    修复后重运行
    修复问题后重新运行审查验证
