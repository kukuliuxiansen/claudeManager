推广命令

将项目特定本能提升为全局可用本能的命令工具

类型: Command类

实现效果

将跨项目的高质量本能提升为全局可用本能

  • 项目检测
    自动识别当前工作项目
  • 自动提升
    提升符合跨项目条件的本能（出现≥2个项目）
  • 单个提升
    指定instinct-id提升特定本能
  • 全局存储
    写入 ~/.claude/homunculus/instincts/personal/ 作用域为 global

    • 核心原理

    推广命令基于连续学习v2技能的instinct-cli.py脚本，通过以下逻辑工作：

    1. 依赖分析 - 识别出现在至少2个项目的本能作为跨项目候选
    2. 置信度检查 - 验证本能是否达到最低置信度阈值
    3. 作用域转换 - 将 personal 作用域改为 global 并写入全局目录
    4. dry-run模式 - 允许预览将要提升的本能而不实际修改

    使用方法

    触发场景与调用 
    /promote                      # 自动检测并提升候选本能
/promote --dry-run            # 预览自动提升候选
/promote --force              # 提升所有合格候选，无需提示
/promote grep-before-edit     # 提升特定本能（当前项目）

    使用技巧

    1
    使用 /promote --dry-run 查看哪些本能可提升
    2
    --force 模式适合批量提升已验证的本能
    3
    单个提升时指定完整 instinct-id
    4
    提升前检查本能是否已在全局可用
    5
    全局本能对所有项目生效，谨慎提升
    6
    结合 /projects 查看当前项目配置

    输出示例

    推广命令 Output 
    检测到当前项目：claudeManager

提升候选（跨项目 >= 2）：
1. grep-before-edit (2个项目)
   置信度: 0.87
2. python-testing-automation (3个项目)
   置信度: 0.92

运行升降级命令以提升这些本能：
/promote --force

或提升单个：
/promote grep-before-edit

预览模式 (--dry-run)：
将提升以下本能到全局：
- grep-before-edit -> ~/.claude/homunculus/instincts/personal/
- python-testing-automation -> ~/.claude/homunculus/instincts/personal/

作用域变更：personal(当前项目) -> global(所有项目)
