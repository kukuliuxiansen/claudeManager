旁注命令

在任务执行过程中快速提问，而不中断当前工作流

类型: Command类

实现效果

旁注命令允许您在Claude执行任务过程中快速提问，而不会丢失当前任务状态。提问后，Claude会直接回答，然后立即恢复到原来的任务。

  • 不中断任务
    提问时当前任务会被冻结，回答后立即恢复
  • 简洁回答
    直接回答问题核心，不展开长篇大论
  • _contextaware
    可以访问当前文件和代码上下文

    • 核心原理

    旁注命令的工作流程分为三个步骤：

    1. 冻结任务状态：记录当前正在处理的任务、文件和下一步计划
    2. 回答问题：直接、简洁地回答用户问题，可读取当前文件上下文
    3. 恢复任务：立即回到原来的任务状态，从暂停点继续执行

    关键规则：

    • 回答必须简短，只给核心答案
    • 回答后必须说明"回到任务"的状态
    • 绝对不能修改任何文件（只读权限）
    • 如果问题涉及任务方向改变，必须先询问用户

    使用方法

    触发方式 
    /aside <你的问题>

    示例用法：

    使用示例 
    /aside what does this function actually return?
/aside is this pattern thread-safe?
/aside why are we using X instead of Y here?
/aside what's the difference between foo() and bar()?
/aside should we be worried about the N+1 query we just added?

    使用技巧

    1
    回答格式
    使用 "ASIDE: [问题] + [答案] + — Back to task: [原任务]" 格式
    2
    直接回答
    开盘就说答案，不要铺垫，如果需要详细说明可以问是否post-task展开
    3
    文件访问
    可以读取当前正在编辑的文件来回答，但不能写入
    4
    方向检查
    如果问题暗示任务方向改变，必须先确认用户意图
    5
    长答案处理
    先给简短版，然后问是否需要任务后的深入解释
    6
    无任务处理
    如果没有活动任务，仍然使用标准格式，显示"No active task"

ASIDE: is the current approach thread-safe?
No — the shared cache object in src/cache/store.ts:34 is mutated without locking.
Under concurrent requests this is a race condition. It's low risk in a single-process
Node.js server but would be a real problem with worker threads or clustering.
