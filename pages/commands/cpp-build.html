C++构建命令

自动诊断和修复C++构建错误的Slash命令

类型: Command类

实现效果

自动执行C++构建诊断并逐步修复错误的slash命令，通过增量方式帮助开发者解决编译问题。

  • 编译错误诊断
    执行cmake构建并自动解析错误输出，按文件分组并按严重性排序
  • 静态分析集成
    运行clang-tidy和cppcheck进行代码静态分析
  • 增量修复模式
    逐个修复错误，每次修改后重新验证构建结果
  • 完整修复报告
    显示修复的错误数量、修改的文件以及剩余问题

    • 核心原理

    该命令基于增量修复原则设计，通过以下流程确保构建问题被系统性解决：

    1. 诊断阶段：执行cmake构建和clang-tidy/cppcheck静态分析，捕获所有错误输出
    2. 分类阶段：将错误按文件分组，根据错误类型（编译、链接、警告）排序
    3. 修复阶段：逐个处理错误，使用minimal fix原则修改代码
    4. 验证阶段：每次修改后立即重新构建验证修复效果
    5. 报告阶段：汇总修复结果，显示成功率和剩余问题

    如果同一错误尝试3次后仍未解决，或修复引入新错误，命令将停止并报告问题。

    使用方法

    触发场景与调用 
    触发场景：
- cmake --build build 执行失败时
- 链接错误（未定义引用、多重定义）
- 模板实例化失败
- include/依赖问题
- 拉取新代码后构建失败

调用方式：
在Claude Code中输入：
/cpp-build

或在Claude Code中自动触发C++构建错误诊断

    使用技巧

    1
    保持最小修改：只修复错误，不要重构代码
    2
    优先处理编译错误：先解决编译问题，再处理链接错误
    3
    单次验证：每次修改后立即重新构建验证
    4
    结合cpp-review：构建成功后使用cpp-review检查代码质量
    5
    使用cpp-test验证：修复后运行cpp-test确保测试通过

$ cmake --build build

### 修复 1: 未声明的标识符
文件: src/service/user.cpp:25
错误: use of undeclared identifier 'UserRepository'
原因: 缺少include
```cpp
// 添加的include
#include "repository/user_repository.hpp"
```
$ cmake --build build

### 修复 2: 无匹配函数
文件: src/handler/api.cpp:42
错误: no matching function for call to 'process'
```cpp
// 修改前
process(params.get("count"));
// 修改后
process(std::stoi(params.get("count")));
```
$ cmake --build build

### 修复 3: 缺少返回值
文件: src/handler/api.cpp:58
错误: non-void function does not return a value
```cpp
std::optional getUser(const std::string& id) {
    if (id.empty()) {
        return std::nullopt;
    }
    auto user = findUser(id);
    // 添加缺失的返回
    return user;
}
```
$ cmake --build build
