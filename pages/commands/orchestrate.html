编排命令

多 Agent 协作编排与工作流管理命令

类型: Command类
everything-claude-code

实现效果

orchestrate 命令通过编排多个专业 Agent（planner、tdd-guide、code-reviewer、security-reviewer 等）协作完成复杂任务。适用于功能开发、Bug 修复、代码重构、安全审查等工作流。支持预定义工作流类型、自定义 Agent 序列，以及外部 tmux/worktree 编排。

  • 预定义工作流

    feature、bugfix、refactor、security 等工作流类型

  • Agent 链编排

    按顺序执行 Agent，前一个输出作为后一个输入

  • 并行执行

    独立检查可并行执行（code-reviewer ∥ security-reviewer）

  • 控制平面

    支持外部 tmux/worktree 编排与会话管理

    • 核心原理

    orchestrate 命令的核心原理是多 Agent 工作流编排。通过定义 Agent 序列，将复杂任务分解为多个专业 Agent 的协作流程。

    工作流类型:

    • feature - planner → tdd-guide → code-reviewer → security-reviewer
    • bugfix - planner → tdd-guide → code-reviewer
    • refactor - architect → code-reviewer → tdd-guide
    • security - security-reviewer → code-reviewer → architect

    Handoff 文档:

    • Context - 上一个 Agent 完成的工作总结
    • Findings - 关键发现或决策
    • Files Modified - 修改的文件列表
    • Open Questions - 待解决的问题
    • Recommendations - 下一步建议

    外部编排:

    • 对于需要独立 git 状态的长运行工作，使用 tmux/worktree
    • 一键式工作流推荐：`node scripts/orchestrate-worktrees.js plan.json --execute`
    • 脚本路径可作为 seedPaths 配置注入到每个工作器

    使用方法

    当用户需要执行复杂任务工作流时触发此命令。

    触发场景与调用 
    // 触发场景
1. "orchestrate a feature development"
2. "run the bug fix workflow"
3. "refactor with code review"

// 命令调用
/orchestrate [workflow-type] [task-description]

// 示例：功能开发
/orchestrate feature "Add user authentication with OAuth2"

// 示例：Bug 修复
/orchestrate bugfix "Fix login timeout issue"

// 示例：重构
/orchestrate refactor "Migrate to TypeScript"

// 示例：安全审查
/orchestrate security "Review payment processing code"

// 示例：自定义工作流
/orchestrate custom "architect,tdd-guide,code-reviewer" "Redesign caching layer"

    使用技巧

    1

    选择合适工作流 - 根据任务类型选择对应预定义工作流

    2

    安全审查 - 涉及认证/支付/敏感数据时必须使用 security 工作流

    3

    并行执行 - 独立检查可并行执行以提高效率

    4

    外部编排 - 长运行任务使用 tmux/worktree 外部编排
