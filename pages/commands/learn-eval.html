学习评估命令

评估从对话中提取的学习模式的价值和适用性

类型: Command类
everything-claude-code

实现效果

评估从对话中提取的学习模式的重复使用价值，通过系统性检查列表和整体判断。

  • 系统性检查清单
    检查技能重叠、MEMORY.md重叠和可复用性
  • 整体判断 verdict
    Save / Improve then Save / Absorb / Drop
  • 具体改进建议
    当需要改进时，提供明确的改进建议列表
  • 改进建议合并
    当 verdict 为 Absorb 时，建议追加到现有技能

核心原理

评估命令使用清单式判断替代数值评分：

判断流程 
1. 检查 skills/ 目录中是否有内容重叠
2. 检查 MEMORY.md 中是否有重叠
3. 考虑追加到现有技能是否足够
4. 确认这是可复用模式而非一次性修复

5. 综合所有检查结果做出整体判断：
   - Save          : 唯一、具体、范围明确
   - Improve then Save: 有价值但需要优化
   - Absorb into [X]: 应追加到现有技能
   - Drop          : 微不足道、冗余或过于抽象

设计原理：

现代前沿模型（Opus 4.6+）具有强大的上下文判断能力。将丰富的定性信号强制转化为数值分数会损失细微差别，并可能导致误导性的总数。整体判断方法让模型自然权衡所有因素，产生更准确的保存/丢弃决策，而明确的检查列表确保不会跳过任何关键检查。

使用方法

评估触发 
/learn-eval

何时使用：

  • 从对话中提取模式后，评估是否值得保存为技能
  • 创建learned/技能文件之前进行质量验证
  • 不确定模式是否独特或过于抽象时

使用技巧

1
具体性优先：包含可立即使用的代码示例或命令的模式更有价值
2
范围聚焦：单个技能应专注于一个模式，不要过度泛化
3
检查重叠：总是先检查 skills/ 和 MEMORY.md 中的现有内容
4
可复用性：只保存在未来会重复使用的真实模式
