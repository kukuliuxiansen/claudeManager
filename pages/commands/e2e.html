E2E测试命令

端到端测试生成和执行命令，使用Playwright进行用户流程测试

类型: Command类

实现效果

生成Playwright E2E测试，执行跨浏览器测试，捕获失败工件，并生成HTML报告。

  • 测试流程生成
    基于用户流创建Playwright测试，使用Page Object Model模式
  • 跨浏览器执行
    在Chrome、Firefox、Safari等多个浏览器上运行测试
  • 失败工件捕获
    截图、视频、跟踪文件在失败时自动捕获
  • HTML报告生成
    包含结果和工件的HTML报告，JUnit XML用于CI集成
  • 不稳定测试检测
    检测并隔离不稳定的测试，推荐修复方案

    • 核心原理

    E2E测试命令使用e2e-runner Agent执行完整的测试生命周期：

    1. 流程分析：分析用户流并识别测试场景
    2. 测试生成：使用Page Object Model模式生成Playwright测试
    3. 多浏览器执行：在Chrome、Firefox、Safari上并行运行
    4. 失败工件：截图、视频、跟踪文件自动捕获
    5. 报告生成：HTML报告包含结果和工件链接
    6. 稳定性检测：多次运行检测不稳定测试

    工件仅在测试失败时捕获，节省存储空间并提高CI效率。

    使用方法

    触发场景与调用 
    触发场景：
- 测试关键用户流程（登录、交易、付款）
- 验证多步骤流程端到端工作
- 测试UI交互和导航
- 验证前端和后端集成
- 准备生产部署

调用方式：
在Claude Code中输入：
/e2e [功能描述]

示例：
/e2e 测试市场搜索和查看流程
/e2e 验证用户连接钱包和浏览市场
/e2e 测试交易下单和确认流程

    使用技巧

    1
    使用Page Object Model：为可维护性创建POC类
    2
    使用data-testid：为选择器使用数据属性而不是CSS类
    3
    等待API响应：使用waitForResponse而非任意超时
    4
    审查失败工件：在CI失败时始终查看Screenshot和Trace
    5
    移除 Flake 测试：不稳定的测试会误导开发，应隔离或修复

### 测试场景识别
**用户流程**：市场搜索 → 查看结果 → 点击市场 → 查看详情
**测试覆盖**：
1. 导航到市场页面
2. 执行语义搜索
3. 验证搜索结果
4. 点击第一个结果
5. 验证市场详情页面加载
6. 验证图表渲染

### 生成的测试代码
```typescript
// tests/e2e/markets/search-and-view.spec.ts
import { test, expect } from '@playwright/test'
import { MarketsPage } from '../../pages/MarketsPage'
import { MarketDetailsPage } from '../../pages/MarketDetailsPage'

test.describe('Market Search and View Flow', () => {
  test('user can search markets and view details', async ({ page }) => {
    // 1. 导航到市场页面
    const marketsPage = new MarketsPage(page)
    await marketsPage.goto()
    await expect(page).toHaveTitle(/Markets/)

    // 2. 执行语义搜索
    await marketsPage.searchMarkets('election')
    await page.waitForResponse(resp =>
      resp.url().includes('/api/markets/search') && resp.status() === 200
    )

    // 3. 验证搜索结果
    const marketCards = marketsPage.marketCards
    await expect(marketCards.first()).toBeVisible()
    expect(await marketCards.count()).toBeGreaterThan(0)
    await page.screenshot({ path: 'artifacts/search-results.png' })

    // 4-6. 点击并验证详情
    const firstMarketTitle = await marketCards.first().textContent()
    await marketCards.first().click()
    await expect(page).toHaveURL(/\/markets\/[a-z0-9-]+/)
    const detailsPage = new MarketDetailsPage(page)
    await expect(detailsPage.marketName).toBeVisible()
    await expect(detailsPage.priceChart).toBeVisible()
  })
})
```

### CI/CD集成
```yaml
- name: Install Playwright
  run: npx playwright install --with-deps
- name: Run E2E tests
  run: npx playwright test
- name: Upload artifacts
  if: always()
  uses: actions/upload-artifact@v3
  with:
    name: playwright-report
    path: playwright-report/
```

### 最佳实践
**DO:**
- ✅ 使用Page Object Model
- ✅ 使用data-testid属性
- ✅ 等待API响应而非超时
- ✅ 测试端到端用户旅程
- ✅ 在合并前运行测试
- ✅ 查看失败工件

**DON'T:**
- ❌ 使用脆弱的选择器
- ❌ 测试实现细节
- ❌ 在生产环境运行
- ❌ 忽略不稳定测试
- ❌ 跳过失败审查
- ❌ 使用E2E测试每个边缘情况（用单元测试）
