重构清理命令

查找未引用导出并安全删除死代码的自动化工具

类型: Command类

实现效果

检测并删除项目中的死代码，同时保持代码健康

  • 死代码检测
    运行专业工具识别未使用的导出、文件和依赖
  • 安全分级
    将发现分为SAFE/CAUTION/DANGER安全级别
  • 安全删除循环
    删除前运行测试，失败立即回滚
  • 重复代码合并
    识别并合并相似函数和冗余类型定义

    • 核心原理

    重构清理命令通过六个步骤工作：

    1. 步骤1：死代码检测 - 运行 knip/depcheck/ts-prune/vulture/deadcode/cargo-udeps 等工具
    2. 步骤2：发现分级 - 将发现按风险分为SAFE/CAUTION/DANGER安全级别
    3. 步骤3：安全删除循环 - 每个SAFE项：运行测试→删除→重跑测试→失败则回滚
    4. 步骤4：谨慎项目处理 - 对CAUTION项：检查动态导入、字符串引用、外部消费者
    5. 步骤5：重复合并 - 合并相似函数（80%+相似）、冗余类型定义、无价值的包装函数
    6. 步骤6：总结报告 - 输出删除统计和代码行节省

    使用方法

    触发场景与调用 
    /refactor-clean
# 自动检测并安全删除死代码
# 也可手动使用语言特定工具：
npx knip        # TypeScript/JavaScript
vulture src/    # Python
deadcode ./...  # Go
npx ts-prune    # TypeScript专用

    使用技巧

    1
    删除前总是运行完整测试套件作为基线
    2
    一次删除一个项，原子变更便于回滚
    3
    不确定时跳过，保留死代码优于破坏生产
    4
    清理时不要重构，分离关注点（先清理后重构）
    5
    在清理前后运行测试以验证完整性
    6
    结合 /python-review /rust-review 进行语言特定审查
