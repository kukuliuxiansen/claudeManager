循环状态命令

查看和监控循环任务的当前状态与进度

类型: Command类
everything-claude-code

实现效果

loop-status 命令用于查看和监控循环任务的当前状态与进度。适用于检查定时任务运行情况、诊断循环.failure原因、监控长期运行的轮询任务等场景。

  • 实时状态监控

    显示循环任务的当前阶段、执行次数、上次成功检查点

  • 失败原因诊断

    显示失败的检查项、错误信息和推荐的干预措施

  • 时间成本估算

    显示预计剩余时间与成本漂移，帮助决策

  • 智能干预建议

    根据状态自动建议继续/暂停/停止

    • 核心原理

    loop-status 命令的核心原理是状态查询与报告。通过读取循环任务的执行上下文，提取关键状态信息并格式化输出。

    监控的关键指标:

    • active loop pattern - 当前循环模式（固定间隔/条件触发）
    • current phase - 当前执行阶段（Research/Plan/Execute/Optimize/Review）
    • last checkpoint - 上次成功完成的检查点
    • execution count - 已执行次数与重试次数
    • estimated drift - 预计时间与_TOKEN_消耗漂移

    Watch 模式:

    • 指定 --watch 参数后，系统会定期刷新状态
    • 状态发生显著变化时自动输出通知
    • 适用于长期运行的监控任务

    使用方法

    当用户需要检查循环任务的当前状态或诊断问题时触发此命令。

    触发场景与调用 
    // 触发场景
1. "check my loop status"
2. "what's the current state of the monitoring task?"
3. "show me the progress of the polling loop"

// 命令调用
/loop-status

// -watch 模式持续监控
/loop-status --watch

// 备用调用（直接查询）
检查循环状态
查看轮询任务进度
查询执行状态

    使用技巧

    1

    定期检查 - 长时间运行的任务建议每 15-30 分钟检查状态

    2

    Watch 模式 - 对于关键任务使用 --watch 实时监控

    3

    关注失败原因 - 当显示 failing checks 时优先处理

    4

    成本监控 - 注意 estimated time/cost drift 提示
