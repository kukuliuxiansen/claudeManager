Gradle构建修复

诊断并修复Gradle/KMP构建问题

类型: Command类

实现效果

系统化诊断和修复Gradle/KMP/Android构建错误：

  • 构建配置检测
    自动识别KMP、Android或普通Gradle项目类型
  • 错误分组排序
    分离Kotlin编译错误与Gradle配置错误
  • Gradle Sync
    自动运行 Gradle sync 和 clean 修复缓存问题

    • 核心原理 

    
    Gradle构建诊断流程
    

    
Step 1: 确定构建配置
├─ 检测 build.gradle.kts (KMP)
├─ 检测 app/ 目录 (Android)
├─ 检测 settings.gradle.kts (多模块)
└─ 查询 gradle.properties 配置

Step 2: 运行适当构建命令
├─ KMP: ./gradlew compileKotlinMetadata 2>&1
├─ Android: ./gradlew compileDebugKotlin 2>&1
├─ 普通: ./gradlew assemble 2>&1
└─ Detekt: ./gradlew detekt 2>&1

Step 3: 解析并分组错误
├─ 分离 Kotlin 编译错误
├─ 分离 Gradle 配置错误
├─ 按模块分组
└─ 按依赖顺序排序

Step 4: 修复循环
├─ 读取文件上下文
├─ 分析错误类型（import、类型、依赖）
├─ 最小化修改
├─ 重新运行构建
└─ 继续下一个错误

Step 5: 验证
├─ go test ./... (如果适用)
├─ ./gradlew test 2>&1
└─ 最终成功确认

    使用方法

    触发场景与调用 
    # Gradle/Kotlin/Android/KMP构建失败时触发

"/gradle-build"
格式: 修复 Gradle 构建错误
要求: 运行 /gradle-build 开始诊断和修复

示例用法：
1. 构建失败后运行
2. ./gradlew assemble 失败
3. Kotlin 编译错误

关键点：
- 命令会自动检测项目类型
- 运行正确的构建命令
- 逐个修复错误并验证
- 如果 3 次尝试后仍失败，会询问用户

    使用技巧

    1
    KMP项目：确保 expect/actual 声明配对完整
    2
    依赖问题：运行 `./gradlew dependencies` 查看依赖树
    3
    重复类错误：检查 Gradle 依赖是否存在冲突
    4
    配置缓存问题：避免使用非序列化任务输入
    5
    KSP/Room生成代码：运行对应的 KSP 任务重新生成
    6
    Compose编译错误：检查 Kotlin 和 Compose 编译器版本匹配

 Step 1: 检测构建配置
文件检测结果：
├─ build.gradle.kts (KMP 项目)
├─ composeApp/ (KMP 共享模块)
└─ androidApp/ (Android 端应用)

选择构建命令：
$ ./gradlew composeApp:compileKotlinMetadata 2>&1

 Step 2: 运行构建 - 发现错误

internal/compose/shared/Network.kt:25:15: Unresolved reference: Retrofit
internal/compose/shared/Database.kt:42:9: Expect declaration has no actual
composables/ui/Theme.kt:58:2: Compose compiler version mismatch

错误总数：3

 Step 3: 修复中

#[1/3] 未解析引用
文件：internal/compose/shared/Network.kt:25
错误：Unresolved reference: Retrofit
原因：依赖未在 commonMain 中声明
修复：在 commonMain.dependencies {} 中添加 Retrofit 依赖
```kotlin
commonMain.dependencies {
    implementation("com.squareup.retrofit2:retrofit:2.9.0")
}
```
结果：通过

#[2/3] Expect 声明无实际实现
文件：internal/compose/shared/Database.kt:42
错误：Expect declaration has no actual
原因：缺少 actual 实现
修复：在 androidMain 中添加 actual 实现
```kotlin
actual object Database {
    actual fun create(): Db = AndroidDatabase()
}
```
结果：通过

#[3/3] Compose 编译器版本不匹配
文件：compose/composables/ui/Theme.kt:58
错误：Compose compiler version mismatch
原因：Kotlin 和 Compose 编译器版本不一致
修复：在 libs.versions.toml 中对齐版本
```toml
[versions]
kotlin = "1.9.20"
compose-compiler = "1.5.6"
```
结果：通过
