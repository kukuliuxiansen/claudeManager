验证命令

运行完整的验证检查套件

类型: Command类

实现效果

验证命令按顺序执行完整的验证检查套件，包括构建、类型检查、lint、测试和安全扫描，以确保代码符合发布标准。

  • 构建检查

    运行项目构建命令

  • 类型检查

    运行 TypeScript 或类型检查器

  • Lint 检查

    运行代码 linter

  • 测试套件

    运行所有测试并报告覆盖率

  • 控制台日志审计

    搜索源文件中的 console.log

    • 核心原理

    验证命令的核心原理是按顺序运行多个验证检查。它按依赖顺序执行检查，如果早期检查失败，会停止后续检查并报告错误。

    检查顺序:

    1. 构建 - 确保代码可以编译和打包
    2. 类型 - 确保类型正确且没有类型错误
    3. Lint - 确保代码风格一致
    4. 测试 - 确保所有测试通过
    5. 审计 - 查找潜在问题（console.log、Secrets）

    验证模式:

    • quick - 构建 + 类型检查
    • pre-commit - 构建 + 类型 + lint
    • full - 所有检查（默认）
    • pre-pr - 完整检查 + 安全扫描

    使用方法

    当需要验证代码是否可以提交或打包时使用此命令。

    触发场景与调用 
    // 运行完整验证
/verify

// 快速验证（仅构建+类型）
/verify quick

// 提交前验证
/verify pre-commit

// PR 前验证
/verify pre-pr

    使用技巧

    1

    提交前运行 - 使用 /verify pre-commit 在提交前快速检查

    2

    PR 前完整验证 - 使用 /verify pre-pr 进行完整检查

    3

    快速迭代 - 修改时使用 /verify quick 节省时间

    4

    检查输出 - 仔细阅读验证报告以找出问题
