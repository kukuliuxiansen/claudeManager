模型路由命令

根据任务特性自动选择最优模型的智能路由命令

类型: Command类
everything-claude-code

实现效果

model-route 命令根据任务特性和复杂度自动选择最优模型（Haiku/Sonnet/Opus），实现成本与质量的平衡。适用于需要根据任务类型自动选择合适模型的场景。

  • 智能模型选择

    基于任务复杂度自动推荐最合适的模型

  • 置信度评估

    提供推荐模型的置信度和匹配理由

  • 备选方案

    提供失败时的备选模型策略

  • 预算感知

    支持预算参数，平衡成本与质量

    • 核心原理

    model-route 命令的核心原理是任务特征分析与模型匹配。通过分析任务需求的复杂度、确定性要求和成本约束，自动选择最优模型。

    模型特性:

    • Haiku - 快速、低成本、确定性高，适合机械性变更
    • Sonnet - 默认模型，平衡速度与能力，适合实现与重构
    • Opus - 最强大，适合架构设计、深度分析和模糊需求

    匹配逻辑:

    • 机械性任务 → Haiku（确定性高、成本低）
    • 实现与重构 → Sonnet（默认推荐）
    • 复杂分析 → Opus（需要深度推理）
    • 预算敏感 → 优先 Haiku/Sonnet

    使用方法

    当用户需要根据任务特性选择合适模型时触发此命令。

    触发场景与调用 
    // 触发场景
1. "what model should I use for this task?"
2. "recommend a model for refactoring"
3. "which model is best for architecture planning"

// 基本调用
/model-route [task-description]

// 带预算参数
/model-route [task-description] --budget low
/model-route [task-description] --budget med
/model-route [task-description] --budget high

// 示例：机械性变更
/model-route "rename variable from x to y"

// 示例：复杂重构
/model-route "refactor login module with multiple patterns"

// 示例：架构设计
/model-route --budget high "design microservices architecture for e-commerce"

    使用技巧

    1

    预算优先 - 成本敏感任务使用 --budget low

    2

    复杂度评估 - 简单任务不需要高预算模型

    3

    容错思考 - 关键任务可考虑 Opus 作为首选

    4

    混合策略 - 简单任务 Haiku，复杂任务 Opus
