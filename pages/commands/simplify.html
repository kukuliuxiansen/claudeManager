simplify

代码简化 Skill，审查变更代码的复用性、质量和效率

类型: Code Quality Skill

实现效果

simplify Skill 用于审查已变更的代码，检查复用性、代码质量和执行效率，并修复发现的问题。确保代码保持简洁、高效和可维护。

  • 代码复用检查

    识别重复代码，建议提取为公共函数或模块

  • 质量评估

    检查代码是否符合编码标准和最佳实践

  • 效率优化

    识别低效算法和冗余操作，优化执行性能

  • 自动修复

    发现问题后自动应用修复，保持代码简洁

    • 核心原理

    simplify Skill 的核心原理是审查变更并简化。通过分析 git diff 识别新增和修改的代码，然后检查复用机会、质量问题和效率瓶颈。

    检查维度:

    • 复用性: 是否存在重复代码可以提取
    • 可读性: 函数是否过长、命名是否清晰
    • 效率: 是否有不必要的循环或复杂度
    • 一致性: 是否符合项目编码风格

    简化原则:

    • 小函数优于大函数（<50 行）
    • 单一职责，避免混合逻辑
    • 消除不必要的条件分支
    • 使用早返回减少嵌套

    使用方法

    simplify Skill 在代码编写或修改后主动触发，审查变更代码。

    触发场景与调用 
    // 触发场景
1. 完成代码编写后 - 自动审查质量
2. 修改已有代码后 - 检查变更影响
3. 重构过程中 - 确保简化方向正确

// 通过 Skill 工具调用
/simplify

// 示例：审查后的简化建议
// Before (复杂嵌套)
function process(data) {
  if (data) {
    if (data.items) {
      if (data.items.length > 0) {
        return data.items.map(i => i.value)
      }
    }
  }
  return []
}

// After (简化)
function process(data) {
  const items = data?.items ?? []
  return items.map(i => i.value)
}

    使用技巧

    1

    早返回 - 使用 guard clauses 减少嵌套

    2

    提取函数 - 重复代码提取为独立函数

    3

    利用语法 - 使用可选链、空值合并等现代语法

    4

    保持可读 - 简化不应牺牲代码可读性
