多执行命令

直接执行已讨论计划的高效开发命令

类型: Command类
everything-claude-code

实现效果

multi-execute 命令用于直接执行已经讨论和批准的开发计划。跳过规划阶段，直接进行代码实现，适用于已有完整计划的场景。通过 Codex/Gemini 获取代码变更方案，Claude 执行实际修改并进行质量审查。

  • 计划复用

    读取计划文件，复用已讨论的实现方案和 SESSION_ID

  • 代码复原

    将 Codex/Gemini 的 Unified Diff 转换为高质量代码

  • 自动审核

    代码修改后自动并行调用 Codex/Gemini 进行审查

  • 代码主权

    所有文件修改由 Claude 执行，外部模型零写权限

    • 核心原理

    multi-execute 命令的核心原理是计划驱动的代码交付。通过读取已批准的计划文件，复用之前的会话上下文，执行代码修改并进行质量审查。

    执行流程:

    1. Phase 0: 读取计划 - 解析计划文件，提取关键信息
    2. Phase 1: 上下文检索 - 获取目标文件的上下文
    3. Phase 3: 原型获取 - 并行调用 Codex/Gemini 获取代码变更
    4. Phase 4: 实现 - Claude 将 Diff 转换为高质量代码
    5. Phase 5: 审核 - 并行审查 + 修复 + 交付

    路由规则:

    • 前端任务 → Gemini（前端权威）
    • 后端任务 → Codex（后端权威）
    • 全栈任务 → Codex ∥ Gemini 并行

    信任规则:

    • 后端修改遵循 Codex 方案
    • 前端修改遵循 Gemini 方案
    • Claude 所有文件修改，模型零写权限

    使用方法

    当用户需要执行已经讨论和批准的开发计划时触发此命令。

    触发场景与调用 
    // 触发场景
1. "execute the plan we just discussed"
2. "run the implementation for user-auth feature"
3. "start working on this plan file"

// 命令调用
/ccg:execute [plan-file-path]
/ccg:execute [plan-description]

// 示例：执行计划文件
/ccg:execute .claude/plan/user-auth.md

// 示例：直接描述任务
/ccg:execute implement user authentication feature based on previous plan

    使用技巧

    1

    计划完整 - 确保计划文件包含完整的 SESSION_ID 和关键文件

    2

    上下文充分 - 执行前确保已获取目标文件的完整上下文

    3

    审核通过 - 等待 Codex/Gemini 审核完成后再交付

    4

    问题修复 - 根据审核反馈修复代码后重新审查
