Harness审计命令

自动评估工作流质量并生成改进建议

类型: Command类

实现效果

使用确定性引擎评估Claude工作流质量，提供可复现的审计结果：

  • 7个固定评估维度
    工具覆盖、上下文效率、质量门禁、内存持久化、评测覆盖、安全门禁、成本效率
  • 确定性评分引擎
    基于文件/规则检查，同一次提交可复现相同结果
  • Top 3改进建议
    自动识别最高优先级的改进项

    • 核心原理 

    
    审计评估维度 (0-10分，共70分)
    

    
1. 工具覆盖 (Tool Coverage)
├─ Bash工具使用
├─ 文件操作工具
├─ 搜索与研究工具
└─ 特定技能调用

2. 上下文效率 (Context Efficiency)
├─ 读取必要文件
├─ 避免过度读取
├─ 精确查询
└─ 摘要使用

3. 质量门禁 (Quality Gates)
├─ TDD流程遵循
├─ 测试覆盖率检查
├─ 代码审查
└─ 构建验证

4. 内存持久化 (Memory Persistence)
├─ TodoWrite使用
├─ 上下文管理
├─ 状态跟踪
└─ 历史记录引用

5. 评测覆盖 (Eval Coverage)
├─ 单元测试
├─ 集成测试
├─ E2E测试
└─ 覆盖率报告

6. 安全门禁 (Security Guardrails)
├─ 敏感信息检测
├─ 安全扫描
├─ 权限检查
└─ 风险评估

7. 成本效率 (Cost Efficiency)
├─ 模型选择
├─ 上下文窗口优化
├─ 并行度控制
└─ 避免重复工作

运行脚本：
node scripts/harness-audit.js <scope> --format <text|json>

    使用方法

    触发场景与调用 
    # 评估工作流质量时触发

"/harness-audit"
格式: 审计工作流质量
要求: 运行 /harness-audit 开始评估

示例用法：
1. /harness-audit                    # 自动项目审计
2. /harness-audit hooks              # 只审计 hooks
3. /harness-audit skills --format json  # JSON格式输出

审计范围 (scope):
- repo (默认) : 完整项目审计
- hooks       : 只审计 hooks 配置
- skills      : 只审计 skills 配置
- commands    : 只审计 commands 配置
- agents      : 只审计 agents 配置

输出格式:
- text (默认) : 人类可读格式
- json        : 机器可读格式

    使用技巧

    1
    使用 `--format json` 可以在CI/CD中自动处理审计结果
    2
    审计结果包含 exact file path，可直接定位到问题文件
    3
    关注 "Top 3 Actions" - 这些是优先级最高的改进项
    4
    scores < 50 表示需要重大改进；> 60 表示良好；> 70 表示优秀
    5
    可以比较不同提交的审计结果，跟踪质量改进趋势
    6
    聚焦于失败的检查项，这些是明确的改进机会

失败的检查项:

1) [安全门禁] 需要在 hooks/hooks.json 中添加预检查安全防护
   文件: hooks/hooks.json

2) [工具覆盖] 需要同步 commands/harness-audit.md 和 .opencode/commands/harness-audit.md
   文件: .opencode/commands/harness-audit.md

3) [评测覆盖] 需要增加 scripts/hooks/lib 的自动化测试覆盖
   文件: tests/

Top 3 改进建议:

1) [安全门禁] 在 hooks/hooks.json 中添加 prompt/tool 预检安全防护
2) [工具覆盖] 同步 commands/harness-audit.md 和 .opencode/commands/harness-audit.md
3) [评测覆盖] 增加 scripts/hooks/lib 的自动化测试覆盖
