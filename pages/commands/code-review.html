代码审查命令

使用 code-review 技能审查和改进代码质量

类型: Command类

实现效果

代码审查命令调用 code-review 技能提供专业的代码审查服务，覆盖代码规范、质量和效率等多个维度。

  • 规范审查
    检查代码是否符合项目命名、结构和风格规范
  • 质量评估
    评估代码复杂度、可读性和可维护性
  • 最佳实践
    验证代码是否遵循语言和框架最佳实践

    • 核心原理

    代码审查技能采用系统化审查流程：

    1. 代码阅读 - 读取相关文件，理解上下文和功能意图
    2. 模式识别 - 识别代码重复、过长函数、深层嵌套等反模式
    3. 规则检查 - 验证命名规范、错误处理、输入验证等
    4. 改进建议 - 提供具体、可操作的改进建议

    审查覆盖维度：

    • 代码规范：命名、格式、注释
    • 函数设计：长度、复杂度、单一职责
    • 错误处理：异常处理、边界条件
    • 性能：算法效率、资源使用

    使用方法

    触发方式 
    /code-review

    代码审查技能将在以下场景自动触发：

    触发场景 
    1. 代码修改后主动运行此命令
2. PR 提交前进行最终审查
3. 代码完成时要求审查

    使用技巧

    1
    小范围审查
    每次审查 1-2 个文件以获得最佳质量
    2
    主要修改后
    在完成主要功能后立即审查
    3
    结合 TDD
    在编写测试代码时运行审查
    4
    PR 前必做
    提交 PR 前必须通过代码审查
    5
    关注 HIGH 问题
    优先处理 HIGH 严重性问题
    6
    定期回顾
    定期回顾历史审查记录学习改进
