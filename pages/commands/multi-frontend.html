多前端命令

COCO 协议引导的多模型协作前端开发命令

类型: Command类
everything-claude-code

实现效果

multi-frontend 命令通过 COCO 协议实现多模型协作的前端开发流程。使用 Gemini 作为前端专家主导 UI/UX 开发，Codex 辅助参考后端视角，Claude 进行协调与执行。适用于组件设计、响应式布局、UI 动画、样式优化等前端任务。

  • 多模型协作

    Gemini 主导 + Codex 辅助 + Claude 协调的三模型协同

  • 6阶段工作流

    Research → Ideation → Plan → Execute → Optimize → Review

  • 会话复用

    通过 SESSION_ID 复用 Gemini 上下文，提高效率

  • 质量保障

    每个阶段包含质量检查，不符合标准则停止

    • 核心原理

    multi-frontend 命令的核心原理是基于 COCO 协议的多模型责任分离。通过将不同职责分配给不同模型，发挥各模型优势，实现最佳前端开发效果。

    角色分工:

    • Gemini - 前端 UI/UX 专家（权威信任）：组件、布局、样式、动画
    • Codex - 辅助参考：从后端视角提供补充意见
    • Claude - 协调与执行：统筹、规划、编码、交付

    工作流程:

    1. Phase 0: 提示增强（可选）
    2. Phase 1: Research - 需求理解与上下文收集
    3. Phase 2: Ideation - Gemini 主导分析，输出方案
    4. Phase 3: Plan - Gemini 主导规划，输出架构
    5. Phase 4: Execute - Claude 编码实现
    6. Phase 5: Optimize - Gemini 主导优化与审查
    7. Phase 6: Review - 最终质量评估

    会话管理:

    • 每个 Gemini 调用返回 SESSION_ID
    • 后续阶段使用 resume <SESSION_ID> 复用上下文
    • 减少重复输入，提高效率

    使用方法

    当用户需要进行前端开发任务（UI、组件、样式等）时触发此命令。

    触发场景与调用 
    // 触发场景
1. "help me design a new component"
2. "create a responsive dashboard layout"
3. "optimize the button animation"

// 命令调用
/frontend <前端任务描述>

// 示例：组件设计
/frontend 设计一个用户信息展示卡片组件

// 示例：响应式布局
/frontend 创建手机端和桌面端适配的导航栏

// 示例：样式优化
/frontend 优化主题颜色和按钮样式，遵循设计系统

    使用技巧

    1

    描述具体 - 清晰的任务描述帮助模型理解需求

    2

    质量检查 - 关注每个阶段的评分，低于 7 分需补充

    3

    会话复用 - 保存 GEMINI_SESSION 用于后续阶段

    4

    阶段确认 - 每个阶段完成后等待用户批准再继续
