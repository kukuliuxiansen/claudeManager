更新代码地图命令

扫描项目结构并生成系统架构地图

类型: Command类

实现效果

更新代码地图命令扫描项目结构，生成系统架构地图文件集合，包括后端、前端、数据和依赖关系的可视化描述。

  • 扫描项目结构

    识别项目类型和源目录

  • 生成架构地图

    创建架构基准地图文件

  • 检测变更

    计算与之前地图的差异

  • 保存分析报告

    记录文件变更和新鲜度状态

    • 核心原理

    更新代码地图命令的核心原理是扫描源代码文件结构并输出可视化地图。它分析项目中的关键文件、路由、数据流和依赖关系，生成可供 AI 消耗的架构文档。

    地图类型:

    • architecture.md: 高级系统图和服务边界
    • backend.md: API 路由和服务到存储库映射
    • frontend.md: 页面树和组件层次结构
    • data.md: 数据库表和关系
    • dependencies.md: 外部服务和第三方集成

    输出特点:

    • Token 友好，优化 AI 上下文使用
    • 使用 ASCII 图表代替冗长描述
    • 记录新鲜度和变更状态

    使用方法

    当需要更新系统架构地图或在重大变更后重新扫描项目时使用此命令。

    触发场景与调用 
    // 更新所有地图
/update-codemaps

// 用于重大功能添加或重构会话后

    使用技巧

    1

    高阶结构优先 - 关注文件路径和函数签名而非完整代码块

    2

    保持简洁 - 每个地图保持在 1000 token 以内

    3

    ASCII 图表 - 使用 ASCII 图表表示数据流

    4

    定期更新 - 关键重构或功能添加后运行

    输出示例

    update-codemaps Output
