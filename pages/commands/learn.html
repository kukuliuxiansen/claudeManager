学习命令

从对话中提取可复用的学习模式并保存为技能

类型: Command类
everything-claude-code

实现效果

在会话过程中自动或手动触发，从复杂问题解决中提取可复用的模式并保存为技能文件。

  • 多维度模式识别
    识别错误解决、调试技术、工作区和项目特定模式
  • 智能位置决策
    分析并选择保存到全局 or 项目特定位置
  • 确认保存流程
    在保存前展示详细草稿以供用户确认
  • 避免重复
    检查现有技能和MEMORY.md以避免重复内容

核心原理

学习命令通过系统性流程提取和保存知识：

模式识别流程 
1. 检查会话中的可提取模式
2. 识别最有价值/可复用的见解
3. 确定保存位置：
   - 全局 (~/.claude/skills/learned/)
     适用于 2+ 项目通用的模式
   - 项目特定 (.claude/skills/learned/)
     仅适用于当前项目的特定知识
4. 草拟技能文件
5. 用户确认后保存

保存策略：

全局模式更通用，更容易移动到项目级别（反之则更困难）。当不确定时优先选择全局存储。

使用方法

触发场景 
/learn

何时使用：

  • 会话期间解决非平凡问题时
  • 发现有用的调试技术或工作区时
  • 解决库/API怪癖时
  • 识别项目特定架构决策时

使用技巧

1
关注重要问题：不提取微不足道的修复（拼写错误、简单语法错误）
2
避免一次性问题：不提取特定API中断等临时问题
3
聚焦模式：保持技能聚焦 - 每个技能一个模式
4
未来时钟：问自己"这个模式会为将来节省时间吗？"

输出示例

提取的模式示例 
**Extracted:** 2026-03-29
**Context:** 修复Gradle项目中Kotlin类型推断错误

### Problem
Gradient中使用call.parameters["count"]返回String类型，但代码期望Int类型，导致类型不匹配错误。

### Solution
使用安全调用和空值处理：
```kotlin
val count = call.parameters["count"]?.toIntOrNull()
    ?: return@get call.respond(HttpStatusCode.BadRequest, "Invalid count")
```

### When To Use
- 从Ktor调用参数读取时
- Kotlin类型推断与预期不符时
- API返回可空但需要非空值时
保存位置决策 
保存到全局 (~/.claude/skills/learned/):
适用：通用调试技术、跨项目可复用的模式
示例：
- Gradle构建命令调试
- Kotlin空安全处理技巧
- 协程错误处理模式

保存到项目 (.claude/skills/learned/):
适用：项目特定知识、配置诀窍
示例：
- 项目特定配置文件位置
- 团队命名约定
- 自定义构建脚本模式
避免保存 
以下情况不应保存：
-拼写错误修正
-简单语法错误修复
-特定版本API中断（临时性）
-仅适用于单次会话的一次性修复

原因：这些模式无法在未来会话中重复使用
