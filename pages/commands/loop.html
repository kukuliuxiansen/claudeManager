loop

循环执行 Skill，按设定间隔重复运行命令或提示

类型: Automation Skill

实现效果

loop Skill 用于按设定间隔重复运行命令或提示。适用于轮询状态、定期检查部署、重复执行任务等场景。默认间隔为 10 分钟。

  • 定时轮询

    按设定间隔自动执行命令，如每 5 分钟检查部署状态

  • Slash 命令支持

    可以循环执行任何 slash 命令或自定义 prompt

  • 灵活间隔

    支持分钟、小时等时间单位，可自定义执行频率

  • 状态监控

    持续监控任务状态，直到满足条件或手动停止

    • 核心原理

    loop Skill 的核心原理是定时任务调度。通过配置时间间隔，系统会自动重复执行指定的命令或 prompt，无需人工干预。

    时间格式:

    • /loop 5m /foo - 每 5 分钟执行
    • /loop 1h /bar - 每 1 小时执行
    • /loop /baz - 默认每 10 分钟执行

    典型应用场景:

    • "check the deploy every 5 minutes"
    • "keep polling until the server is ready"
    • "run health check every 10 minutes"
    • "monitor the queue status periodically"

    停止条件:

    • 手动取消
    • 满足预设条件时自动停止
    • 达到最大执行次数

    使用方法

    当用户需要设置重复任务或轮询状态时触发此 Skill。

    触发场景与调用 
    // 触发场景
1. "check the deploy every 5 minutes"
2. "keep polling until the server is ready"
3. "run /health-check every 10 minutes"
4. "monitor the queue status periodically"

// 通过 Skill 工具调用
/loop [interval] [command]

// 示例：每 5 分钟检查部署状态
/loop 5m /check-deploy-status

// 示例：每 1 小时运行健康检查
/loop 1h /health-check

// 示例：默认 10 分钟间隔
/loop /my-custom-command

// 示例：轮询直到条件满足
/loop 2m "check if server is ready, stop when status=healthy"

    使用技巧

    1

    设置停止条件 - 明确何时停止轮询，避免无限循环

    2

    合理间隔 - 根据任务性质设置间隔，避免资源浪费

    3

    日志记录 - 记录每次执行结果便于追踪

    4

    组合命令 - 可与其他 Skill 组合使用
