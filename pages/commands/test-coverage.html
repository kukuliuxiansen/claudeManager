测试覆盖命令

分析测试覆盖率并生成缺失的测试

类型: Command类

实现效果

测试覆盖命令分析项目的测试覆盖率，识别覆盖率低于 80% 的文件，并为这些文件生成缺失的测试。

  • 检测测试框架

    识别 Jest、Vitest、pytest 等框架

  • 分析覆盖率报告

    解析覆盖率 JSON 或终端输出

  • 生成缺失测试

    为低覆盖率文件生成测试

  • 验证改进

    重新运行覆盖率确认改进

    • 核心原理

    测试覆盖命令的核心原理是运行框架特定的覆盖率命令并解析输出。它根据项目配置识别测试框架，然后生成缺失的测试以提高覆盖率。

    支持的框架:

    • Jest: `npx jest --coverage`
    • Vitest: `npx vitest run --coverage`
    • Pytest: `pytest --cov=src --cov-report=json`
    • Rust: `cargo llvm-cov --json`
    • Go: `go test -coverprofile=coverage.out`

    覆盖率目标:

    • 80% - 一般代码最低要求
    • 100% - 关键业务逻辑、认证、安全代码

    使用方法

    当需要分析或提高测试覆盖率时使用此命令。

    触发场景与调用 
    // 启动测试覆盖分析
/test-coverage

// 自动检测测试框架并生成覆盖率报告

    使用技巧

    1

    关注分支覆盖 - 确保测试覆盖所有 if/else 分支

    2

    测试错误处理 - 包含错误路径和边缘情况

    3

    优先关键模块 - 先提高核心业务逻辑覆盖率

    4

    保持测试独立 - 每个测试应独立运行且可重复
