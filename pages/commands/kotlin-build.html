Kotlin构建命令

诊断并修复Kotlin项目构建错误

类型: Command类
everything-claude-code

实现效果

自动诊断并修复Kotlin项目构建错误，支持逐步修复和验证。

  • 多诊断命令
    运行Gradle构建、detekt、ktlintCheck等诊断命令
  • 错误分类排序
    按文件和严重程度对错误进行分组排序
  • 逐步修复验证
    一次修复一个错误，每个变更后重新运行构建
  • 修复报告
    显示修复摘要和剩余问题清单

核心原理

构建命令通过Gradle任务执行诊断和修复：

诊断命令 
# 构建诊断
./gradlew build 2>&1

# 代码质量检查
./gradlew detekt 2>&1 || echo "detekt not configured"
./gradlew ktlintCheck 2>&1 || echo "ktlint not configured"

# 依赖检查
./gradlew dependencies --configuration runtimeClasspath
./gradlew build --refresh-dependencies

修复流程：

  1. 运行诊断命令并解析错误输出
  2. 按文件和严重程度对错误进行分组
  3. 逐个修复错误，每次修复后重新运行构建
  4. 验证修复效果，直到构建成功
  5. 运行detekt和ktlint检查代码质量

使用方法

触发场景 
/kotlin-build

何时使用：

  • ./gradlew build 命令失败时
  • Kotlin编译器报告错误时
  • ./gradlew detekt 报告违规时
  • Gradle依赖解析失败时
  • 拉取可能导致构建失败的更改后

使用技巧

1
构建错误优先：确保代码能够编译，这是修复的基础
2
一次修复一个：逐步修复每个错误，验证每个变更
3
最小化变更：仅修复问题，不要进行重构
4
检查配置：如果问题持续存在，检查build.gradle配置

常见错误类型 
| Error | 典型修复 |
|-------|----------|
| Unresolved reference: X | 添加import或依赖 |
| Type mismatch | 修复类型转换或赋值 |

| Smart cast impossible | 使用局部val或let |
| Could not resolve dependency | 修复版本或添加仓库
