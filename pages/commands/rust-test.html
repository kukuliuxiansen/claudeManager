Rust 测试命令

遵循测试驱动开发（TDD）模式编写和运行 Rust 测试

类型: Command类

实现效果

Rust 测试命令引导用户执行测试驱动开发（TDD）流程：先编写失败测试（RED），然后实现代码使其通过（GREEN），最后重构并保持测试通过（REFACTOR）。

  • TDD 循环
    RED -> GREEN -> REFACTOR -> REPEAT 的完整测试驱动流程
  • 覆盖率验证
    使用 cargo-llvm-cov 确保 80%+ 的代码覆盖
  • 模式支持
    支持单元测试、参数化测试、异步测试和基于属性的测试

    • 核心原理

    该命令遵循 TDD 的六步流程：

    1. 定义接口：使用 `todo!()` 持存函数签名
    2. 编写测试：创建全面的测试模块（RED）
    3. 运行测试：验证测试因预期原因失败
    4. 实现代码：编写通过测试的最小代码（GREEN）
    5. 重构：改进代码同时保持测试通过
    6. 检查覆盖率：确保达到 80%+ 覆盖率

    使用方法

    触发场景与调用 
    /rust-test

# 自动触发场景：
# - 实现新的 Rust 函数、方法或 trait
# - 为现有 Rust 代码添加测试覆盖
# - 修复 bug（先编写失败测试）
# - 构建关键业务逻辑
# - 学习 Rust 中的 TDD 工作流

    使用技巧

    1
    先写测试
    在任何实现之前编写测试（RED）
    2
    运行每个变更
    每次更改后运行测试
    3
    测试行为而非实现
    验证输出而不是内部逻辑
    4
    包括边缘情况
    测试空值、边界和错误路径

### Step 1: Define Interface
```rust
pub fn validate_registration(request: &RegistrationRequest) -> ValidationResult {
    todo!()
}
```

### Step 2: Write Tests (Red)
```rust
#[test]
fn valid_registration_returns_valid() {
    let request = RegistrationRequest {
        name: "Alice".into(),
        email: "alice@example.com".into(),
        password: "SecureP@ss1".into(),
    };
    assert!(matches!(validate_registration(&request), ValidationResult::Valid));
}
```

### Step 3: Run Tests - Verify Fail
```bash
$ cargo test
running 4 tests
test tests::valid_registration_returns_valid ... FAILED (not yet implemented)
...
Tests fail as expected (todo! panic).
```

### Step 4: Implement Minimal Code (Green)
```rust
pub fn validate_registration(request: &RegistrationRequest) -> ValidationResult {
    let mut errors = Vec::new();
    if request.name.trim().is_empty() {
        errors.push("Name is required".into());
    }
    if !request.email.contains('@') {
        errors.push("Invalid email format".into());
    }
    if request.password.len() < 8 {
        errors.push("Password must be at least 8 characters".into());
    }
    if errors.is_empty() {
        ValidationResult::Valid
    } else {
        ValidationResult::Invalid(errors)
    }
}
```

### Step 5: Run Tests - Verify Pass
```bash
$ cargo test
running 4 tests
test tests::valid_registration_returns_valid ... ok
...
All tests passing!
```
