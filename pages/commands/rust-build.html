Rust 构建命令

诊断并修复 Rust 构建错误和警告

类型: Command类

实现效果

Rust 构建命令通过运行诊断工具识别构建错误、警告和格式问题，然后逐个修复问题并验证修复效果，直到构建成功。

  • 自动诊断
    运行 cargo check、cargo clippy 和 cargo fmt --check 识别所有问题
  • 增量修复
    一次修复一个问题，每次更改后重新运行诊断验证
  • 总结报告
    展示修复的问题数量和剩余问题的摘要

    • 核心原理

    该命令遵循以下修复流程：

    1. 运行诊断：执行 cargo check、cargo clippy、cargo fmt --check 等命令
    2. 解析错误：提取错误代码、文件位置和行号
    3. 增量修复：每次聚焦一个错误，应用修复后重新运行检查
    4. 最终验证：运行 clippy 和 tests 确保所有问题已解决

    使用方法

    触发场景与调用 
    /rust-build

# 自动触发场景：
# - cargo build 或 cargo check 失败
# - cargo clippy 报告警告
# - 借用检查器或生命周期错误阻止编译
# - Cargo 依赖解析失败
# - 拉取导致构建失败的更改后

    使用技巧

    1
    先修复构建错误
    代码必须先编译，然后才能处理其他问题
    2
    一次修复一个
    每个更改后运行 cargo check 验证
    3
    最小化更改
    只需修复问题，不要重构代码
    4
    停止条件
    同一错误尝试 3 次后停止报告
