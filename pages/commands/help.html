帮助命令

访问文档和命令帮助的入口

类型: Command类

实现效果

提供内置命令、技能、插件的快速帮助文档访问：

  • 命令帮助
    所有可用命令的列表和简要说明
  • 快速查找
    通过命令名称或功能描述查找帮助
  • 使用示例
    每个帮助项包含实际使用示例

    • 核心原理 

    
    帮助系统架构
    

    
帮助系统组成:

1. 内置命令 (Built-in Commands)
├─ /help - 帮助命令本身
├─ /docs - 文档阅读
├─ /sessions - 会话管理
├─ /save-session - 保存会话
└─ /resume-session - 恢复会话

2. 项目命令 (Project Commands)
├─ /feature-dev - 功能开发
├─ /go-build - Go构建修复
├─ /go-review - Go审查
├─ /go-test - Go测试
└─ /gradle-build - Gradle修复

3. 评估命令 (Evaluation Commands)
├─ /harness-audit - 工作流审计
├─ /eval - 评估运行
└─ /test-coverage - 覆盖率

4. 仓库命令 (Repo Commands)
├─ /commit - 提交
├─ /commit-push-pr - 提交并推送PR
├─ /clean_gone - 清理已删除文件
└─ /list - 列出项

5. 插件命令 (Plugin Commands)
├─ /hookify - 创建钩子
├─ /instinct-export - 导出本能
└─ /instinct-import - 导入本能

帮助系统读取:
- commands/*.html 中的元数据
- 解析命令标题和描述
- 生成索引供搜索
- 显示相关命令推荐

    使用方法

    触发场景与调用 
    # 查看命令帮助时触发

"/help"
"/help [command]"
格式: 显示命令帮助文档
要求: 运行 /help 查看帮助

示例用法：
1. /help                    # 查看所有命令列表
2. /help feature-dev        # 查看功能开发帮助
3. /help go-build           # 查看Go构建修复帮助
4. /help hooks              # 查看钩子帮助

帮助系统包含：
- 命令名称
- 简短描述
- 使用示例
- 输入/输出格式
- 适用场景
- 相关命令链接

    使用技巧

    1
    未知命令时，用 /help 列出所有可用命令
    2
    输入部分命令名，/help 会提供自动补全建议
    3
    帮助文档包含链接，可直接跳转到详细页面
    4
    每个帮助页面都包含"相关命令"部分
    5
    搜索功能可在帮助中查找特定关键词
    6
    帮助文档由 /docs 命令增强。
