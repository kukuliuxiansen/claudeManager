writing-rules

Hookify规则编写指南，创建项目自定义钩子规则

类型: Rule Management

实现效果

writing-rules Skill 提供Hookify规则编写的完整指南，帮助开发者创建自定义钩子规则监控和拦截特定操作。

  • 四种事件类型

    bash（Bash命令）、file（文件编辑）、stop（完成检查）、prompt（用户输入）四种触发事件

  • 多条件匹配

    支持regex_match、contains、equals、not_contains、starts_with、ends_with六种匹配操作符

  • 两种动作模式

    warn（显示警告但允许继续）和block（阻止操作或停止会话）两种处理动作

  • 动态加载

    规则在每次工具使用时动态读取，修改后立即生效无需重启

    • 核心原理

    规则编写遵循"模式匹配 + 动作执行"原则。定义正则模式匹配特定操作，触发时显示自定义消息或阻止操作。

    规则文件结构:

    • 位置: .claude/hookify.{rule-name}.local.md
    • Frontmatter: name、enabled、event、pattern/conditions
    • 消息体: 触发时显示给Claude的Markdown内容

    字段选项:

    • bash事件: 匹配command字段
    • file事件: 匹配file_path、new_text、old_text、content字段
    • stop事件: 完成检查清单
    • prompt事件: 匹配user_prompt字段

    使用方法

    writing-rules Skill 在需要创建自定义钩子规则时使用：

    规则示例 
    // 最小规则结构
---
name: my-rule
enabled: true
event: bash
pattern: dangerous_command
---

Warning message here

// 多条件规则示例
---
name: warn-env-file-edits
enabled: true
event: file
conditions:
  - field: file_path
    operator: regex_match
    pattern: \.env$
  - field: new_text
    operator: contains
    pattern: API_KEY
---

You're adding an API key to a .env file.
Ensure this file is in .gitignore!

// 常用模式
rm\s+-rf         # 危险删除命令
console\.log\(   # console.log调试代码
\.env$           # 环境变量文件
API_KEY\s*=      # API密钥赋值

// 测试正则
python3 -c "import re; print(re.search(r'pattern', 'test text'))"

    使用技巧

    1

    具体而非宽泛 - pattern: log太宽泛，使用console\.log\(更精确

    2

    转义特殊字符 - .、(、)等正则元字符需用\.、\(、\)转义

    3

    消息要有用 - 解释检测到什么、为何有问题、建议替代方案

    4

    立即测试 - 规则动态加载，修改后下一次工具使用立即生效
