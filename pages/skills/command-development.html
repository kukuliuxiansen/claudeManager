Command Development

命令开发指南，帮助开发新的斜杠命令和可复用工作流

类型: Plugin Development

实现效果

Command Development Skill 是 Claude Code 斜杠命令开发的专业指南。命令是频繁使用的提示词，定义为 Markdown 文件，在交互会话中由 Claude 执行。

  • 可复用性

    定义一次，重复使用。标准化常见工作流程，提高效率

  • YAML 配置

    使用 YAML frontmatter 配置描述、允许工具、模型选择等

  • 动态参数

    支持 $ARGUMENTS、$1、$2 等动态参数，以及 @ 文件引用语法

  • 命令组织

    支持项目命令、个人命令、插件命令，以及命名空间组织

    • 核心原理

    命令是为 Claude 编写的指令，不是给用户的消息。当用户调用 /command-name 时，命令内容成为 Claude 的指令。

    命令位置:

    • 项目命令: .claude/commands/ - 团队共享
    • 个人命令: ~/.claude/commands/ - 全局可用
    • 插件命令: plugin-name/commands/ - 随插件安装

    关键原则:

    • 单一职责：一个命令，一个任务
    • 清晰描述：在 /help 中自解释
    • 显式依赖：使用 allowed-tools 明确工具
    • 文档参数：始终提供 argument-hint

    使用方法

    命令示例与调用 
    # 基础命令结构
.claude/commands/
├── review.md           # /review 命令
├── test.md             # /test 命令
└── deploy.md           # /deploy 命令

# 带 YAML frontmatter 的命令
---
description: Review code for security issues
allowed-tools: Read, Grep, Bash(git:*)
model: sonnet
argument-hint: [pr-number] [priority]
---

Review pull request #$1 with priority level $2.

# 使用动态参数
$ARGUMENTS  # 捕获所有参数
$1, $2, $3  # 位置参数

# 文件引用
Review @$1 for code quality  # 读取文件内容

# Bash 执行
Files changed: !`git diff --name-only`

# 调用命令
/review-pr 123 high
/deploy api staging --force

    使用技巧

    1

    单一职责 - 一个命令只做一件事，保持简单专注

    2

    验证参数 - 检查必需参数，提供默认值和错误提示

    3

    限制 Bash - 使用 Bash(git:*) 而非 Bash(*)，保持安全

    4

    添加注释 - 解释复杂逻辑，提供使用示例
