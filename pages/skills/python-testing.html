Python Testing

pytest测试策略、TDD方法论、Fixtures、Mocking和覆盖率要求

类型: Testing Skill

实现效果

Python Testing Skill 提供pytest测试框架的完整使用指南，包括TDD方法论和最佳实践。

  • TDD工作流

    RED(写失败测试) -> GREEN(最小实现) -> REFACTOR(优化代码)

  • Fixtures系统

    可复用测试夹具，支持多种作用域(function/module/session)

  • 参数化测试

    @pytest.mark.parametrize实现多输入测试，减少代码重复

  • Mocking支持

    unittest.mock集成，模拟外部依赖和函数行为

    • 核心原理

    pytest是Python最流行的测试框架，以简洁的断言语法和强大的fixtures系统著称。

    覆盖率要求:

    • 目标: 80%+代码覆盖率
    • 关键路径: 100%覆盖率要求
    • 测量: pytest --cov生成报告

    Fixture作用域:

    • function: 每个测试运行一次(默认)
    • module: 每个模块运行一次
    • session: 整个测试会话运行一次

    测试组织:

    • tests/unit/ - 单元测试
    • tests/integration/ - 集成测试
    • tests/e2e/ - 端到端测试
    • conftest.py - 共享fixtures

    使用方法

    pytest测试示例和常用模式：

    Fixtures与参数化测试 
    # Fixture示例
import pytest

@pytest.fixture
def database():
    db = Database(":memory:")
    db.create_tables()
    yield db
    db.close()

def test_database_query(database):
    result = database.query("SELECT * FROM users")
    assert len(result) > 0

# 参数化测试
@pytest.mark.parametrize("input,expected", [
    ("hello", "HELLO"),
    ("world", "WORLD"),
    ("PyThOn", "PYTHON"),
])
def test_uppercase(input, expected):
    assert input.upper() == expected

# Mock示例
from unittest.mock import patch

@patch("mypackage.external_api_call")
def test_with_mock(api_call_mock):
    api_call_mock.return_value = {"status": "success"}
    result = my_function()
    api_call_mock.assert_called_once()

# 异常测试
def test_divide_by_zero():
    with pytest.raises(ZeroDivisionError):
        divide(10, 0)

# 运行命令
pytest                              # 运行所有测试
pytest -v                           # 详细输出
pytest --cov=mypackage              # 覆盖率
pytest -m "not slow"                # 跳过慢测试
pytest -x                           # 首次失败后停止

    使用技巧

    1

    测试一件事 - 每个测试只验证单一行为

    2

    使用描述性名称 - test_user_login_with_invalid_credentials_fails

    3

    Mock外部依赖 - 测试不应依赖外部服务

    4

    测试边界条件 - 空输入、None值、边界值

[pytest]
testpaths = tests
python_files = test_*.py
addopts = --strict-markers --cov=mypackage --cov-report=html
markers =
    slow: marks tests as slow
    integration: marks tests as integration tests

# 异步测试 (pytest-asyncio)
@pytest.mark.asyncio
async def test_async_function():
    result = await async_add(2, 3)
    assert result == 5

# API端点测试
@pytest.fixture
def client():
    app = create_app(testing=True)
    return app.test_client()

def test_create_user(client):
    response = client.post("/api/users", json={
        "name": "Alice",
        "email": "alice@example.com"
    })
    assert response.status_code == 201
    assert response.json["name"] == "Alice"
