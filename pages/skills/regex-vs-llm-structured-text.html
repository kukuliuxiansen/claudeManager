Regex vs LLM Structured Text

结构化文本解析决策框架，regex处理95%+场景，LLM仅用于低置信度边缘情况

类型: Pattern Skill

实现效果

Regex vs LLM Skill 提供结构化文本解析的决策框架，regex处理95-98%场景，LLM仅处理低置信度边缘情况。

  • 决策框架

    根据文本一致性(>90%模式匹配)选择regex或LLM

  • 置信度评分

    自动计算提取结果置信度，<0.95阈值触发LLM验证

  • 混合Pipeline

    Regex -> 置信度检查 -> LLM验证(仅低置信项)

  • 成本节省

    生产环境实测节省~95%成本(vs全LLM方案)

    • 核心原理

    结构化文本(测验、表单、发票)解析遵循"先用regex，仅在必要时用LLM"原则。

    决策流程:

    • 文本一致性检查: >90%遵循模式? -> 使用Regex
    • Regex成功率: >95%? -> 完成，无需LLM
    • 低置信项: <95% -> LLM验证边缘情况

    架构模式:

    • Source Text -> Regex Parser (95-98%准确率)
    • -> Text Cleaner (去除噪声)
    • -> Confidence Scorer (标记低置信项)
    • -> High confidence(>=0.95): 直接输出
    • -> Low confidence(<0.95): LLM Validator -> 输出

    生产实测数据(410项):

    • Regex成功率: 98.0%
    • 低置信项: 8 (2.0%)
    • LLM调用: ~5次
    • 成本节省: ~95%

    import re
from dataclasses import dataclass

@dataclass(frozen=True)
class ParsedItem:
    id: str
    text: str
    choices: tuple[str, ...]
    answer: str
    confidence: float = 1.0

def parse_structured_text(content: str) -> list[ParsedItem]:
    """使用regex解析结构化文本"""
    pattern = re.compile(
        r"(?P\d+)\.\s*(?P.+?)\n"
        r"(?P(?:[A-D]\..+?\n)+)"
        r"Answer:\s*(?P[A-D])",
        re.MULTILINE | re.DOTALL,
    )
    items = []
    for match in pattern.finditer(content):
        choices = tuple(
            c.strip() for c in re.findall(r"[A-D]\.\s*(.+)", match.group("choices"))
        )
        items.append(ParsedItem(
            id=match.group("id"),
            text=match.group("text").strip(),
            choices=choices,
            answer=match.group("answer"),
        ))
    return items

def score_confidence(item: ParsedItem) -> float:
    """计算置信度分数"""
    score = 1.0
    if len(item.choices) < 3: score -= 0.3
    if not item.answer: score -= 0.5
    if len(item.text) < 10: score -= 0.2
    return max(0.0, score)

def process_document(content: str, llm_client=None) -> list[ParsedItem]:
    """完整Pipeline: regex -> 置信度检查 -> LLM边缘情况"""
    items = parse_structured_text(content)
    low_confidence = [i for i in items if score_confidence(i) < 0.95]

    if low_confidence and llm_client:
        # 仅对低置信项调用LLM
        for item in low_confidence:
            validated = validate_with_llm(item, content, llm_client)
            items = [validated if i.id == item.id else i for i in items]

    return items

    反模式与最佳实践 
    # 反模式 - 避免
- 全部发给LLM (regex能处理95%+时昂贵且慢)
- 对自由格式、高变异性文本用regex
- 跳过置信度评分，期望regex"刚好工作"
- 在清理/验证步骤中修改解析对象
- 不测试边缘情况(格式错误、缺失字段)

# 最佳实践
- Quiz/exam问题解析
- 表单数据提取
- 发票/收据处理
- 文档结构解析(标题、章节、表格)
- 任何有重复模式且成本重要的结构化文本
