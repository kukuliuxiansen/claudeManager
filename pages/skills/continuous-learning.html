Continuous Learning

从对话中自动提取经验并保存到记忆系统的持续学习技能

类型: Learning Skill

实现效果

Continuous Learning Skill 是自动模式提取的专业技能，作为 Stop hook 在会话结束时运行，自动识别可提取的模式并保存为学习技能。

  • 会话评估

    检查会话是否有足够消息（默认: 10+），评估是否值得提取

  • 模式检测

    识别错误解决、用户纠正、变通方案、调试技术等可提取模式

  • 技能提取

    将有用的模式保存到 ~/.claude/skills/learned/ 作为可复用技能

  • 轻量级 Hook

    仅在会话结束时运行一次，不增加每条消息的延迟

    • 核心原理

    作为 Stop hook 运行，在会话结束时自动评估和提取模式。这是 v1 版本，v2 版本使用更先进的 instinct-based 架构。

    模式类型:

    • error_resolution: 特定错误的解决方法
    • user_corrections: 用户纠正中的模式
    • workarounds: 框架/库问题的变通方案
    • debugging_techniques: 有效的调试方法
    • project_specific: 项目特定约定

    何时使用:

    • 设置自动模式提取
    • 配置 Stop hook 进行会话评估
    • 审查或管理 ~/.claude/skills/learned/ 中的学习技能

    使用方法

    Hook 配置与模式类型 
    # 添加到 ~/.claude/settings.json
{
  "hooks": {
    "Stop": [{
      "matcher": "*",
      "hooks": [{
        "type": "command",
        "command": "~/.claude/skills/continuous-learning/evaluate-session.sh"
      }]
    }]
  }
}

# config.json 配置示例
{
  "min_session_length": 10,
  "extraction_threshold": "medium",
  "auto_approve": false,
  "learned_skills_path": "~/.claude/skills/learned/",
  "patterns_to_detect": [
    "error_resolution",
    "user_corrections",
    "workarounds",
    "debugging_techniques",
    "project_specific"
  ],
  "ignore_patterns": [
    "simple_typos",
    "one_time_fixes",
    "external_api_issues"
  ]
}

    使用技巧

    1

    选择 Stop Hook - 仅在会话结束时运行，轻量且不阻塞，可访问完整会话上下文

    2

    配置阈值 - 调整 min_session_length 避免短会话的无意义提取

    3

    手动触发 - 使用 /learn 命令在会话中手动提取模式

    4

    考虑 v2 - 如需更可靠的观察，考虑升级到 instinct-based v2 架构
