skill-comply

自动化合规性测量，验证技能、规则和代理定义是否被实际遵循

类型: Compliance Skill

实现效果

skill-comply Skill 测量编码代理是否真正遵循技能、规则或代理定义，通过自动生成测试场景并分析合规性。

  • 自动规范生成

    从任何.md文件自动生成预期的行为序列（规范）

  • 多严格度场景

    生成支持、中立、竞争三种提示严格度级别的测试场景

  • LLM行为分类

    使用LLM（非正则表达式）对工具调用进行分类，精确判断合规性

  • 自包含报告

    生成包含规范、提示、合规分数和工具调用时间线的完整报告

    • 核心原理

    合规性测量遵循"提示独立性"原则。测量技能/规则是否在提示没有明确支持时也被遵循。

    支持的目标:

    • Skills (skills/*/SKILL.md): 工作流技能如search-first、TDD指南
    • Rules (rules/common/*.md): 强制规则如testing.md、security.md、git-workflow.md
    • Agent definitions (agents/*.md): 代理是否按预期被调用

    三种提示严格度:

    • Supportive: 提示明确支持遵循规则/技能
    • Neutral: 提示不提及规则/技能
    • Competing: 提示可能与规则/技能竞争注意力

    使用方法

    skill-comply Skill 在验证规则/技能合规性时使用：

    运行命令 
    // 完整运行
uv run python -m scripts.run ~/.claude/rules/common/testing.md

// 试运行（无成本，仅生成规范和场景）
uv run python -m scripts.run --dry-run ~/.claude/skills/search-first/SKILL.md

// 自定义模型
uv run python -m scripts.run --gen-model haiku --model sonnet <path>

// 触发条件
1. 用户运行 `/skill-comply <path>`
2. 用户问 "这个规则真的被遵循了吗？"
3. 添加新规则/技能后验证代理合规性
4. 定期质量维护检查

    使用技巧

    1

    新增规则后验证 - 安装新技能或添加新规则后运行合规性测试

    2

    关注Neutral场景 - 中性提示最能反映实际合规水平

    3

    使用Dry Run - 先试运行检查规范生成是否正确

    4

    低合规步骤改进 - 对合规率低的步骤考虑添加钩子强化

    输出示例
