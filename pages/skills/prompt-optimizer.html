提示词优化

分析原始提示词，识别意图和缺口，匹配ECC组件并输出优化后的提示词

类型: Tool Skill

实现效果

提示词优化Skill 分析原始提示词，匹配ECC生态组件，输出完整优化提示词供用户直接使用。

  • 项目检测

    自动检测项目技术栈(package.json, go.mod等)用于提示词优化

  • 意图分类

    识别任务类型：新功能、Bug修复、重构、研究、测试、审查等

  • 组件匹配

    自动匹配Commands、Skills、Agents和推荐模型

  • 缺口检测

    识别缺失信息，必要时提出澄清问题

    • 核心原理

    提示词优化器采用6阶段分析管道，仅输出分析和优化提示词，不执行任务。

    分析阶段:

    • Phase 0: 项目检测 - CLAUDE.md、技术栈文件
    • Phase 1: 意图检测 - 任务分类
    • Phase 2: 范围评估 - TRIVIAL/LOW/MEDIUM/HIGH/EPIC
    • Phase 3: ECC组件匹配 - Commands/Skills/Agents
    • Phase 4: 缺口检测 - 缺失关键信息
    • Phase 5: 工作流与模型推荐

    范围定义:

    • TRIVIAL: 单文件，<50行
    • LOW: 单组件或模块
    • MEDIUM: 多组件，同领域
    • HIGH: 跨领域，5+文件
    • EPIC: 多会话，多PR，架构变更

# 示例：模糊中文提示词
输入: "帮我写一个用户登录页面"
检测: package.json with Next.js 15, TypeScript

优化后:
使用项目现有技术栈（Next.js 15 + TypeScript + Tailwind CSS）实现用户登录页面。

技术要求：
- 沿用项目现有的组件结构和路由约定
- 认证方式：沿用项目现有认证方案

工作流：
1. /plan 先规划组件结构和认证流程
2. /tdd 测试先行
3. /code-review 审查实现
4. /verify 验证测试通过

    使用技巧

    1

    仅建议不执行 - 此技能只输出优化提示词，不执行实际任务

    2

    澄清优先 - 缺失3+关键信息时先提问再生成提示词

    3

    EPIC用blueprint - 大型任务使用blueprint skill进行多会话规划

    4

    模型推荐 - 根据范围推荐合适的模型(Haiku/Sonnet/Opus)

    EPIC项目示例 
    # 输入: "Migrate our monolith to microservices"

# 优化提示词:
Use the blueprint skill to plan: "Migrate monolith to microservices architecture"

Before executing, answer these questions in the blueprint:
1. Which domain boundaries exist in the current monolith?
2. Which service should be extracted first (lowest coupling)?
3. Communication pattern: REST APIs, gRPC, or event-driven?
4. Database strategy: shared DB or database-per-service?
5. Deployment target: Kubernetes, Docker Compose, or serverless?

The blueprint should produce phases like:
- Phase 1: Identify service boundaries and create domain map
- Phase 2: Set up infrastructure (API gateway, service mesh)
- Phase 3: Extract first service (strangler fig pattern)
- Phase 4: Verify with integration tests, then extract next
- Phase N: Decommission monolith

Each phase = 1 PR, with /verify gates between phases.
Use /save-session between phases. Use /resume-session to continue.

Recommended: Opus 4.6 for blueprint planning, Sonnet 4.6 for execution.
