dmux Workflows

基于 tmux 的多代理编排工具，支持 Claude Code、Codex、OpenCode 等并行会话

类型: Workflow Skill

实现效果

dmux Workflows Skill 是多代理编排的专业模式，使用 tmux 管理 AI 代理面板，支持并行代理会话和跨工具协调。

  • 多代理并行

    按 'n' 创建新面板，按 'm' 合并输出到主会话

  • 跨工具支持

    Claude Code、Codex、OpenCode、Cline、Gemini、Qwen

  • 工作流模式

    Research+Implement、Multi-File Feature、Test+Fix Loop、Cross-Harness

  • Git Worktree 集成

    每个面板使用独立 worktree，避免文件冲突

    • 核心原理

    dmux 是基于 tmux 的编排工具，管理 AI 代理面板:

    • 按 n 创建新面板并输入提示
    • 按 m 合并面板输出到主会话
    • 支持: Claude Code、Codex、OpenCode、Cline、Gemini、Qwen

    工作流模式:

    • Pattern 1: Research + Implement - 研究和实现分两条并行轨道
    • Pattern 2: Multi-File Feature - 跨独立文件并行工作
    • Pattern 3: Test + Fix Loop - 一个面板运行测试，另一个修复
    • Pattern 4: Cross-Harness - 不同 AI 工具处理不同任务
    • Pattern 5: Code Review Pipeline - 并行审查不同视角

    使用方法

    dmux Quick Start 
    # Start dmux session
dmux

# Create agent panes (press 'n' in dmux, then type prompt)
# Pane 1: "Implement the auth middleware in src/auth/"
# Pane 2: "Write tests for the user service"
# Pane 3: "Update API documentation"

# Each pane runs its own agent session
# Press 'm' to merge results back

# Git Worktree Integration (for overlapping files)
git worktree add -b feat/auth ../feature-auth HEAD
git worktree add -b feat/billing ../feature-billing HEAD

# Run agents in separate worktrees
# Pane 1: cd ../feature-auth && claude
# Pane 2: cd ../feature-billing && claude

# Merge branches when done
git merge feat/auth
git merge feat/billing

    使用技巧

    1

    仅独立任务 - 不要并行化相互依赖的任务，确保边界清晰

    2

    策略合并 - 合并前检查面板输出，避免冲突

    3

    Git Worktrees - 文件冲突风险高时，每个面板使用独立 worktree

    4

    资源意识 - 每个面板消耗 API tokens，保持总数在 5-6 以下

    输出示例

    ECC Helper - orchestrate-worktrees.js 
    # ECC helper for external tmux-pane orchestration
node scripts/orchestrate-worktrees.js plan.json --execute

# Example plan.json:
{
  "sessionName": "skill-audit",
  "baseRef": "HEAD",
  "launcherCommand": "codex exec --cwd {worktree_path} --task-file {task_file}",
  "workers": [
    { "name": "docs-a", "task": "Fix skills 1-4 and write handoff notes." },
    { "name": "docs-b", "task": "Fix skills 5-8 and write handoff notes." }
  ]
}

# The helper:
# - Creates one branch-backed git worktree per worker
# - Writes per-worker task.md, handoff.md, status.md files
# - Starts a tmux session with one pane per worker
# - Launches each worker command in its own pane
# - Leaves the main pane free for the orchestrator
