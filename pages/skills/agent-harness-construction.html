Agent Harness构建

设计和优化AI Agent的动作空间、工具定义和观察格式，提高完成率

类型: Agent类

实现效果

Agent输出质量受四个关键约束：动作空间质量、观察质量、恢复质量、上下文预算质量。

  • 动作空间设计
    使用稳定明确的工具名称，保持输入模式优先且窄化，返回确定性输出形状
  • 观察设计
    每个工具响应包含status、summary、next_actions、artifacts四个关键元素
  • 错误恢复契约
    每个错误路径包含根本原因提示、安全重试指令、明确停止条件
  • 上下文预算
    保持系统提示最小化和不变性，按需加载大型指导到技能中
  • 架构模式指导
    ReAct适合探索性任务，Function-calling适合结构化确定性流程，推荐混合模式

    • 核心原理

    粒度规则：根据操作风险级别选择工具粒度。

    • 微工具：用于高风险操作（部署、迁移、权限）
    • 中等工具：用于常见编辑/读取/搜索循环
    • 宏工具：仅当往返开销是主要成本时使用

    使用方法

    观察设计标准格式 
    # 工具响应标准格式
{
  "status": "success|warning|error",
  "summary": "一行结果摘要",
  "next_actions": ["可执行的后续步骤"],
  "artifacts": ["文件路径 / ID"]
}

# 错误恢复契约
{
  "root_cause_hint": "根本原因提示",
  "safe_retry_instruction": "安全重试指令",
  "explicit_stop_condition": "明确停止条件"
}

# 上下文预算原则
1. 保持系统提示最小化和不变性
2. 将大型指导移到按需加载的技能中
3. 优先使用文件引用而非内联长文档
4. 在阶段边界压缩，而非任意token阈值

    使用技巧

    1
    避免具有重叠语义的过多工具，这会导致Agent混淆
    2
    工具输出必须包含恢复提示，不透明输出是最常见的失败模式
    3
    错误输出必须包含下一步，仅返回错误的工具是最差的
    4
    跟踪完成率、每任务重试次数、pass@1和pass@3、每成功任务成本
    5
    使用混合架构：ReAct规划 + 类型化工具执行
    6
    避免上下文过载无关引用，在阶段边界压缩而非任意阈值

    Anti-Patterns 反模式 
    # 常见反模式

1. 工具过多且语义重叠
   - 问题：Agent不知道选择哪个工具
   - 解决：合并相似工具，明确每个工具的职责边界

2. 工具输出不透明，无恢复提示
   - 问题：Agent无法理解失败原因
   - 解决：每个错误包含root_cause_hint和next_actions

3. 仅返回错误，没有下一步
   - 问题：Agent陷入死循环
   - 解决：所有错误路径都提供safe_retry_instruction

4. 上下文过载无关引用
   - 问题：上下文窗口被无关信息占用
   - 解决：按需加载技能，使用文件引用代替内联
