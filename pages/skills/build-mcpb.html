构建MCPB

构建打包的本地MCP服务器，用户无需Node/Python即可安装运行

类型: 打包工具

实现效果

应用MCPB构建技能，创建可分发的本地MCP服务器包。

  • 打包运行时
    将MCP服务器与Node/Python运行时一起打包，用户无需安装任何工具链
  • 清单配置
    manifest.json定义身份、入口点、配置模式、兼容性和用户配置
  • 标准stdio服务器
    服务器代码与本地stdio MCP服务器相同，只是打包方式不同
  • UI组件支持
    MCPB服务器可以像远程MCP应用一样提供UI资源，机制与传输无关

    • 核心原理

    何时使用MCPB

    服务器必须在用户机器上运行 - 读取本地文件、驱动桌面应用、访问localhost服务、OS级API。

    何时不使用

    服务器只访问云API - 应使用远程HTTP服务器，无需承担MCPB打包开销。

    安全责任

    MCPB没有沙箱 - 服务器以完全用户权限运行。必须验证路径、拒绝逃逸ROOT、白名单spawn。

    构建流程

    Node: npm install + esbuild打包 + mcpb pack。Python: pip安装到vendor目录 + mcpb pack。

    使用方法

    MCPB构建流程 
    # Node构建流程
npm install
npx esbuild src/index.ts --bundle --platform=node --outfile=server/index.js
npx @anthropic-ai/mcpb pack

# Python构建流程
pip install -t server/vendor -r requirements.txt
npx @anthropic-ai/mcpb pack

# 测试
npx @modelcontextprotocol/inspector node server/index.js

# 验证和打包
npx @anthropic-ai/mcpb validate
npx @anthropic-ai/mcpb pack

# 签名分发
npx @anthropic-ai/mcpb sign dist/local-files.mcpb

# 安装：将.mcpb文件拖放到Claude Desktop

    使用技巧

    1
    在分发前在没有开发工具链的机器上测试，避免"在我机器上能运行"问题
    2
    原生依赖必须为每个目标平台构建，尽可能避免使用原生依赖
    3
    每个路径都必须验证，每个spawn都必须白名单，因为平台级别没有任何阻止
    4
    使用${__dirname}引用包内相对路径，使用${user_config.key}替换安装时配置

    输出示例

    manifest.json示例 
    {
  "$schema": "https://raw.githubusercontent.com/anthropics/mcpb/main/schemas/mcpb-manifest-v0.4.schema.json",
  "manifest_version": "0.4",
  "name": "local-files",
  "version": "0.1.0",
  "description": "Read, search, and watch files on the local filesystem.",
  "server": {
    "type": "node",
    "entry_point": "server/index.js",
    "mcp_config": {
      "command": "node",
      "args": ["${__dirname}/server/index.js"],
      "env": {
        "ROOT_DIR": "${user_config.rootDir}"
      }
    }
  },
  "user_config": {
    "rootDir": {
      "type": "directory",
      "title": "Root directory",
      "default": "${HOME}/Documents",
      "required": true
    }
  }
}
