Debug Optimize LCP

使用 Chrome DevTools MCP 调试和优化 Largest Contentful Paint

类型: Performance Skill

实现效果

Debug Optimize LCP Skill 是 Largest Contentful Paint 性能调试的专业技能，帮助识别和优化页面加载性能瓶颈。

  • LCP 四子部分分析

    TTFB、Resource Load Delay、Resource Load Duration、Element Render Delay

  • Performance Trace

    使用 performance_start_trace 捕获完整页面加载，包括 LCP 时间

  • LCP Insights 分析

    LCPBreakdown、DocumentLatency、RenderBlocking、LCPDiscovery 深度洞察

  • 优化策略

    消除资源加载延迟、元素渲染延迟、减少资源加载时间和 TTFB

    • 核心原理

    LCP 四子部分分解:

    • TTFB (~40%) - 导航开始到收到 HTML 第一个字节
    • Resource Load Delay (<10%) - TTFB 到浏览器开始加载 LCP 资源
    • Resource Load Duration (~40%) - 下载 LCP 资源的时间
    • Element Render Delay (<10%) - LCP 资源下载完成到元素渲染

    性能标准:

    • Good: 2.5 秒或更少
    • Needs improvement: 2.5-4.0 秒
    • Poor: 大于 4.0 秒

    常见陷阱: 优化一个子部分而不检查其他子部分。如果渲染延迟是真正的瓶颈，更小的图片不会帮助 - 节省的时间只是转移到渲染延迟。

    使用方法

    LCP 调试工作流 
    # Step 1: 记录性能追踪
navigate_page to the target URL
performance_start_trace with reload: true and autoStop: true

# Step 2: 分析 LCP Insights
performance_analyze_insight with insight name and insight set ID:
- LCPBreakdown - 显示四个 LCP 子部分及时间
- DocumentLatency - 影响 TTFB 的服务器响应时间
- RenderBlocking - 阻塞 LCP 元素渲染的资源
- LCPDiscovery - LCP 资源是否早期可发现

# Step 3: 识别 LCP 元素
evaluate_script with "Identify LCP Element" snippet

# Step 4: 检查网络瀑布图
list_network_requests filtered by resourceTypes: ["Image", "Font"]
get_network_request with the LCP resource's request ID

# Step 5: 检查 HTML 常见问题
evaluate_script with "Audit Common Issues" snippet

    使用技巧

    1

    永不懒加载 LCP 图片 - 使用标准 <img> 加 src，添加 fetchpriority="high"

    2

    预加载关键资源 - 添加 <link rel="preload" fetchpriority="high"> 提前加载

    3

    消除延迟子部分 - 资源加载延迟和元素渲染延迟应接近零

    4

    使用模拟测试 - emulate with Fast 3G 和 cpuThrottlingRate: 4 测试慢速场景

    输出示例

    LCP Optimization Strategies 
    # 1. 消除资源加载延迟 (目标: <10%)
- Root Cause: LCP 图片通过 JS/CSS 加载, data-src, loading="lazy"
- Fix: 使用标准 <img> 加 src, 添加 fetchpriority="high"
- Fix: 添加 <link rel="preload" fetchpriority="high">

# 2. 消除元素渲染延迟 (目标: <10%)
- Root Cause: 大样式表, <head> 中同步脚本, 主线程阻塞
- Fix: 内联关键 CSS, 延迟非关键 CSS/JS
- Fix: 使用 SSR 让元素存在于初始 HTML

# 3. 减少资源加载时间 (目标: ~40%)
- Fix: 使用现代格式 (WebP, AVIF) 和响应式图片
- Fix: 从 CDN 提供服务, 设置 Cache-Control
- Fix: 使用 font-display: swap 避免 Web 字体阻塞

# 4. 减少 TTFB (目标: ~40%)
- Fix: 最小化重定向, 优化服务器响应时间
- Fix: 在边缘缓存 HTML (CDN)
- Fix: 确保页面符合 bfcache 条件
