Agent开发

Claude Code插件Agent开发指南，帮助创建自主子进程处理复杂多步骤任务

类型: Agent类

实现效果

Agent是自主子进程，独立处理复杂的多步骤任务。理解Agent结构、触发条件和系统提示设计是创建强大自主能力的关键。

  • Markdown文件格式
    使用YAML前置内容的Markdown文件格式，包含name、description、model、color等字段
  • 触发条件设计
    通过description字段定义触发条件，包含多个example块展示使用场景
  • 系统提示设计
    Markdown主体成为Agent的系统提示，以第二人称编写，定义核心职责和分析流程
  • 模型和颜色定制
    支持inherit/sonnet/opus/haiku模型选择，提供blue/cyan/green/yellow等颜色选项
  • 工具权限控制
    可选的tools字段限制Agent可访问的工具，遵循最小权限原则

    • 核心原理

    Agent vs Command：Agent用于自主工作，Command用于用户发起的操作。Agent通过description字段中的example块实现智能触发。

    • name字段：3-50字符，小写字母、数字、连字符，必须以字母数字开头和结尾
    • description字段：必须包含触发条件和2-4个example块
    • model字段：推荐使用inherit，除非Agent需要特定模型能力
    • 系统提示：以第二人称编写，定义职责、流程、输出格式
    • 工具限制：仅限需要的工具，实现最小权限原则

    使用方法

    Agent文件结构示例 
    ---
name: code-reviewer
description: Use this agent when reviewing code. Examples:
<example>
Context: User has just committed changes
user: "Review my latest commit"
assistant: "I'll use the code-reviewer agent to analyze your changes."
<commentary>
This is a code review task, perfect for the code-reviewer agent.
</commentary>
</example>
model: inherit
color: blue
tools: ["Read", "Grep", "Glob"]
---

You are a code reviewer specializing in quality and security.

**Your Core Responsibilities:**
1. Review code for bugs and security issues
2. Check adherence to coding standards
3. Suggest improvements

**Output Format:**
- Summary of findings
- Critical issues (must fix)
- Suggestions (optional improvements)

    使用技巧

    1
    在description中包含2-4个具体示例，展示主动和被动触发场景
    2
    系统提示使用第二人称"You are..."，定义清晰的职责边界
    3
    限制工具权限到最小需求，使用tools字段实现最小权限原则
    4
    为不同类型的Agent选择不同颜色，便于UI中区分
    5
    系统提示保持在10,000字符以内，包含质量标准和边缘情况处理
    6
    使用validate-agent.sh脚本验证Agent文件结构

    输出示例

    Frontmatter字段总结 
    | Field      | Required | Format              | Example              |
|------------|----------|--------------------|----------------------|
| name       | Yes      | lowercase-hyphens  | code-reviewer        |
| description| Yes      | Text + examples    | Use when... <example>... |
| model      | Yes      | inherit/sonnet/opus/haiku | inherit       |
| color      | Yes      | Color name         | blue                 |
| tools      | No       | Array of tool names| ["Read", "Grep"]     |

# 常见工具集配置
- 只读分析: ["Read", "Grep", "Glob"]
- 代码生成: ["Read", "Write", "Grep"]
- 测试: ["Read", "Bash", "Grep"]
- 完全访问: 省略字段或使用 ["*"]
