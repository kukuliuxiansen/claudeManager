自主循环

自主循环控制模式，从简单的顺序管道到RFC驱动的多代理DAG系统

类型: 自动化

实现效果

应用自主循环模式，实现无需人工干预的自动化开发工作流。

  • 顺序管道模式
    使用claude -p链式调用构建自动化流程，每个步骤独立的上下文窗口避免信息泄露
  • 无限代理循环
    两提示系统编排并行子代理，通过分配唯一迭代号防止重复概念
  • 持续PR循环
    生产级脚本自动创建PR、等待CI、合并，通过SHARED_TASK_NOTES.md跨迭代桥接上下文
  • De-Sloppify模式
    在实现步骤后添加独立清理步骤，移除冗余测试和过度防御性检查

    • 核心原理

    循环模式光谱

    从简单到复杂：顺序管道 -> NanoClaw REPL -> 无限代理循环 -> 持续Claude PR循环 -> Ralphinho DAG编排

    关键设计原则

    1. 每个步骤隔离 - 独立上下文窗口防止信息泄露
    2. 负面指令危险 - 使用独立清理步骤而非约束提示
    3. 退出代码传播 - set -e在失败时停止管道

    上下文桥接

    使用SHARED_TASK_NOTES.md或文件系统状态在独立的claude -p调用之间桥接上下文。

    使用方法

    顺序管道示例 
    #!/bin/bash
# daily-dev.sh - 顺序管道示例

set -e

# Step 1: 实现功能
claude -p "Read the spec in docs/auth-spec.md. Implement OAuth2 login in src/auth/."

# Step 2: De-sloppify（清理）
claude -p "Review all changes. Remove unnecessary tests and defensive checks."

# Step 3: 验证
claude -p "Run build, lint, type check, and tests. Fix failures."

# Step 4: 提交
claude -p "Create conventional commit for staged changes."

    使用技巧

    1
    始终设置退出条件：max-runs、max-cost、max-duration或完成信号
    2
    使用SHARED_TASK_NOTES.md在迭代间桥接上下文，记录进度和下一步
    3
    审查者不应是作者，分离上下文窗口消除自我偏见
    4
    添加独立的de-sloppify清理步骤，而非在实现提示中使用负面约束
