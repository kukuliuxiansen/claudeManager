端到端测试 (E2E Testing)

使用Playwright进行关键用户流程的自动化测试，确保应用端到端质量

类型: Testing Skill

实现效果

E2E Testing Skill 是端到端测试专家，使用Playwright进行关键用户流程的自动化测试，确保应用在真实浏览器环境中的质量。

  • Page Object Model

    封装页面元素和操作，提高测试代码可维护性和可复用性

  • 多浏览器支持

    支持Chromium、Firefox、WebKit以及移动设备模拟器

  • 自动等待机制

    智能等待元素状态，避免竞态条件和脆弱测试

  • CI/CD集成

    与GitHub Actions等CI平台无缝集成，生成HTML和JSON报告

    • 核心原理

    E2E测试模拟真实用户行为，在真实浏览器中验证完整的用户流程。

    Playwright核心特性:

    • 自动等待: 智能等待元素可交互，无需手动sleep
    • 网络拦截: 模拟API响应，测试边界情况
    • 并行执行: 多测试并行运行，加速反馈周期
    • 追踪调试: 自动截图、录像、trace文件

    反模式警示:

    • 避免任意超时：使用waitForResponse替代waitForTimeout
    • 避免脆弱选择器：优先使用data-testid属性
    • 避免测试依赖：每个测试应独立运行

    使用方法

    E2E Testing Skill 在开发关键功能、验证用户流程、集成测试时主动触发：

    触发场景与调用 
    // Page Object Model示例
// pages/ItemsPage.ts
import { Page, Locator } from '@playwright/test'

export class ItemsPage {
  readonly page: Page
  readonly searchInput: Locator
  readonly itemCards: Locator

  constructor(page: Page) {
    this.page = page
    this.searchInput = page.locator('[data-testid="search-input"]')
    this.itemCards = page.locator('[data-testid="item-card"]')
  }

  async goto() {
    await this.page.goto('/items')
    await this.page.waitForLoadState('networkidle')
  }

  async search(query: string) {
    await this.searchInput.fill(query)
    await this.page.waitForResponse(resp =>
      resp.url().includes('/api/search')
    )
  }
}

// 测试文件
// tests/e2e/search.spec.ts
import { test, expect } from '@playwright/test'
import { ItemsPage } from '../pages/ItemsPage'

test('should search by keyword', async ({ page }) => {
  const itemsPage = new ItemsPage(page)
  await itemsPage.goto()
  await itemsPage.search('test')

  const count = await itemsPage.itemCards.count()
  expect(count).toBeGreaterThan(0)
})

    使用技巧

    1

    使用data-testid - 为测试元素添加稳定的测试ID，避免依赖易变的CSS类名

    2

    等待特定条件 - 使用waitForResponse替代waitForTimeout，确保网络请求完成

    3

    隔离测试数据 - 每个测试使用独立的测试数据，避免测试间相互影响

    4

    保留失败证据 - 配置screenshot和video在失败时保留，便于调试

## Playwright配置示例
```typescript
import { defineConfig, devices } from '@playwright/test'

export default defineConfig({
  testDir: './tests/e2e',
  fullyParallel: true,
  retries: process.env.CI ? 2 : 0,
  reporter: [
    ['html', { outputFolder: 'playwright-report' }],
    ['json', { outputFile: 'playwright-results.json' }]
  ],
  use: {
    baseURL: 'http://localhost:3000',
    trace: 'on-first-retry',
    screenshot: 'only-on-failure',
    video: 'retain-on-failure'
  },
  projects: [
    { name: 'chromium', use: { ...devices['Desktop Chrome'] } },
    { name: 'firefox', use: { ...devices['Desktop Firefox'] } },
    { name: 'webkit', use: { ...devices['Desktop Safari'] } }
  ]
})
```
