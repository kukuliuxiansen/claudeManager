Docker Patterns

Docker和Docker Compose最佳实践，用于容器化开发、多服务编排和安全配置

类型: DevOps Skill

实现效果

Docker Patterns Skill 提供容器化开发的完整指南，从本地开发环境到生产部署的最佳实践。

  • 本地开发环境

    使用Docker Compose快速搭建包含数据库、缓存、邮件服务的完整开发栈

  • 多阶段构建

    分离开发与生产Dockerfile，优化镜像大小并提升安全性

  • 网络与存储

    自定义网络隔离服务，命名卷持久化数据，绑定挂载实现热重载

  • 安全加固

    非root用户运行、只读文件系统、能力限制等容器安全最佳实践

    • 核心原理

    Docker容器化的核心是将应用程序及其依赖打包成可移植的镜像，实现"一次构建，到处运行"。

    关键概念:

    • 镜像(Image): 只读模板，包含运行应用所需的所有内容
    • 容器(Container): 镜像的运行实例，相互隔离的进程
    • 卷(Volume): 持久化存储，独立于容器生命周期
    • 网络(Network): 容器间通信的虚拟网络

    Docker Compose优势:

    • 声明式配置管理多容器应用
    • 服务发现与健康检查自动化
    • 开发与生产环境一致性
    • 一键启动/停止所有服务

    使用方法

    Docker Patterns Skill 在设置容器化环境、设计多服务架构、排查网络问题时触发：

    触发场景与示例 
    # 标准Web应用开发栈
# docker-compose.yml
services:
  app:
    build:
      context: .
      target: dev
    ports:
      - "3000:3000"
    volumes:
      - .:/app
      - /app/node_modules
    environment:
      - DATABASE_URL=postgres://postgres@db:5432/app
    depends_on:
      db:
        condition: service_healthy

  db:
    image: postgres:16-alpine
    environment:
      POSTGRES_DB: app
    volumes:
      - pgdata:/var/lib/postgresql/data
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U postgres"]
      interval: 5s

volumes:
  pgdata:

# 常用命令
docker compose up -d          # 后台启动
docker compose logs -f app    # 查看日志
docker compose exec app sh    # 进入容器
docker compose down -v        # 停止并删除卷

    使用技巧

    1

    使用特定标签 - 永远不要使用:latest标签，指定具体版本确保可重复构建

    2

    保护node_modules - 使用匿名卷保护容器内的依赖不被宿主机覆盖

    3

    健康检查 - 配置healthcheck确保依赖服务就绪后再启动应用

    4

    最小化镜像 - 使用多阶段构建，生产镜像只包含运行时必需文件

    Docker Compose配置示例 
    # 多阶段Dockerfile示例
# Stage: dependencies
FROM node:22-alpine AS deps
WORKDIR /app
COPY package*.json ./
RUN npm ci

# Stage: dev (hot reload)
FROM node:22-alpine AS dev
WORKDIR /app
COPY --from=deps /app/node_modules ./node_modules
COPY . .
EXPOSE 3000
CMD ["npm", "run", "dev"]

# Stage: production
FROM node:22-alpine AS production
WORKDIR /app
RUN addgroup -g 1001 -S app && adduser -S app -u 1001
USER app
COPY --from=build /app/dist ./dist
COPY --from=build /app/node_modules ./node_modules
ENV NODE_ENV=production
EXPOSE 3000
HEALTHCHECK --interval=30s CMD wget -qO- http://localhost:3000/health || exit 1
CMD ["node", "dist/server.js"]
