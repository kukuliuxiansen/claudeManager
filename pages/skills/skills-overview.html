核心原理

Skills 是什么？

Skills 是 Claude Code 的扩展能力模块，通过 Skill 工具调用。每个 Skill 是一个专门的提示词模板，封装了特定领域的最佳实践和工作流程。

工作机制:

  • 触发条件: 根据上下文自动触发或通过 /skill-name 手动调用
  • 提示词注入: Skill 的内容被注入到对话中，作为额外的指导
  • 执行能力: 按照 Skill 定义的工作流执行相应任务
  • 输出格式: 标准 JSON 格式，包含状态、数据、消息等字段

分类依据:

  • 领域: 构建测试、安全合规、Agent自动化等
  • 来源: everything-claude-code 前缀来自主项目，其他来自子模块
  • 触发方式: 自动触发 (TRIGGER when) 或手动调用

使用方法

Skills 调用方式 
// 方式1: 手动调用 (推荐)
/skill-name [arguments]

// 示例
/tdd-workflow user-authentication
/commit
/review-pr 123

// 方式2: 通过 Skill 工具调用
Skill tool: { skill: "tdd-workflow", args: "feature-name" }

// 方式3: 自动触发 (部分 Skills 有 TRIGGER 条件)
// 当代码导入 anthropic SDK 时，claude-api Skill 自动触发
// 当用户说 "commit" 时，commit Skill 自动触发

// 获取 Skill 详情
/help [skill-name]

使用技巧

1

按需调用 - 不要滥用 Skills，只在需要特定能力时调用

2

组合使用 - 多个 Skills 可以组合，如 tdd-workflow + code-review

3

查看详情 - 用 /help [skill-name] 查看具体用法

4

自动触发 - 部分 Skills 会根据上下文自动激活
