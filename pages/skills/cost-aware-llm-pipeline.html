Cost-Aware LLM Pipeline

LLM API 成本优化模式，模型路由、预算追踪、重试逻辑和提示缓存

类型: Backend Pattern

实现效果

Cost-Aware LLM Pipeline Skill 是 LLM API 成本控制的专业模式，组合模型路由、预算追踪、重试逻辑和提示缓存。

  • 任务复杂度路由

    简单任务用 Haiku (3-4x 便宜)，复杂任务用 Sonnet

  • 不可变成本追踪

    frozen dataclass 追踪累计花费，每次调用返回新追踪器

  • 窄重试逻辑

    仅瞬态错误重试，认证/验证错误立即失败

  • 提示缓存

    长系统提示使用 cache_control，避免每次请求重复发送

    • 核心原理

    从最便宜的模型开始，仅在复杂度阈值被触发时路由到昂贵模型。设置明确的预算限制，在处理批次前确保早期失败。

    定价参考 (2025-2026):

    • Haiku 4.5: $0.80/$4.00 (Input/Output per 1M tokens) - 1x
    • Sonnet 4.6: $3.00/$15.00 - ~4x
    • Opus 4.5: $15.00/$75.00 - ~19x

    路由阈值:

    • text_length >= 10,000 chars → Sonnet
    • item_count >= 30 items → Sonnet
    • 否则 → Haiku

    使用方法

    Python 实现示例 
    # 1. 模型路由
MODEL_SONNET = "claude-sonnet-4-6"
MODEL_HAIKU = "claude-haiku-4-5-20251001"
_SONNET_TEXT_THRESHOLD = 10_000  # chars
_SONNET_ITEM_THRESHOLD = 30     # items

def select_model(text_length: int, item_count: int) -> str:
    if text_length >= _SONNET_TEXT_THRESHOLD or item_count >= _SONNET_ITEM_THRESHOLD:
        return MODEL_SONNET  # Complex task
    return MODEL_HAIKU  # Simple task (3-4x cheaper)

# 2. 不可变成本追踪
@dataclass(frozen=True, slots=True)
class CostTracker:
    budget_limit: float = 1.00
    records: tuple[CostRecord, ...] = ()

    def add(self, record: CostRecord) -> "CostTracker":
        return CostTracker(budget_limit=self.budget_limit, records=(*self.records, record))

    @property
    def over_budget(self) -> bool:
        return sum(r.cost_usd for r in self.records) > self.budget_limit

# 3. 窄重试逻辑
_RETRYABLE_ERRORS = (APIConnectionError, RateLimitError, InternalServerError)
_MAX_RETRIES = 3

def call_with_retry(func, *, max_retries: int = _MAX_RETRIES):
    for attempt in range(max_retries):
        try: return func()
        except _RETRYABLE_ERRORS:
            if attempt == max_retries - 1: raise
            time.sleep(2 ** attempt)  # Exponential backoff

    使用技巧

    1

    最便宜模型优先 - 从 Haiku 开始，仅在阈值触发时路由到昂贵模型

    2

    明确预算限制 - 处理批次前设置预算，早期失败而非超支

    3

    提示缓存 - 系统提示超过 1024 tokens 时使用，节省成本和延迟

    4

    记录模型选择 - 日志记录选择决策，便于基于真实数据调优阈值
