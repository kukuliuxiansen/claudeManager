Context Budget

分析上下文窗口消耗，识别臃肿组件，提供优化建议

类型: Performance Skill

实现效果

Context Budget Skill 是上下文窗口审计的专业技能，扫描所有组件目录并估算 Token 消耗，提供可操作的优化建议。

  • 全面审计

    扫描 agents、skills、rules、MCP servers、CLAUDE.md 等所有组件

  • Token 估算

    words x 1.3 用于散文，chars / 4 用于代码，MCP 每工具约 500 tokens

  • 问题检测

    识别臃肿描述、重度代理、冗余组件、MCP 过度订阅等问题

  • 优化建议

    按 Token 节省排序的 Top 3 优化建议，显示潜在节省百分比

    • 核心原理

    上下文窗口是有限资源，每个加载的组件都会消耗 Token。MCP 工具是最大的开销来源，每个工具 schema 约消耗 500 tokens。

    组件分类:

    • Always needed: CLAUDE.md 引用、命令支持、项目类型匹配 - 保留
    • Sometimes needed: 领域特定但未引用 - 考虑按需激活
    • Rarely needed: 无引用、内容重叠 - 移除或懒加载

    何时使用:

    • 会话性能变慢或输出质量下降
    • 最近添加了大量 skills、agents 或 MCP servers
    • 计划添加更多组件，需要确认空间

    使用方法

    触发场景与调用 
    # 基础审计
/context-budget
# 输出: 16 agents (12,400 tokens), 28 skills (6,200),
#       87 MCP tools (43,500), 2 CLAUDE.md (1,200)
# Flags: 3 heavy agents, 14 MCP servers (3 CLI-replaceable)
# Top saving: remove 3 MCP servers → -27,500 tokens (47%)

# 详细模式
/context-budget --verbose
# 输出: 完整报告 + 每文件分解，显示最重文件详情

# 扩展前检查
User: I want to add 5 more MCP servers, do I have room?
# 输出: 当前开销 33% → 添加 5 servers (~50 tools)
#       将增加 ~25,000 tokens → 推至 45%
# 建议: 先移除 2 个 CLI 可替换 servers

    使用技巧

    1

    MCP 是最大杠杆 - 每个工具 schema 约 500 tokens，30 工具的 server 超过所有 skills 总和

    2

    Agent 描述始终加载 - 即使从不调用，description 字段也在每次 Task 工具上下文中

    3

    详细模式调试 - 用于定位具体文件开销，不用于日常审计

    4

    变更后审计 - 添加任何 agent、skill 或 MCP server 后运行，及早发现问题
