coding-standards

通用编码规范和最佳实践，适用于 TypeScript、JavaScript、React 和 Node.js

类型: Quality Skill

实现效果

coding-standards Skill 提供通用编码规范，确保代码质量和可维护性。涵盖命名规范、不可变性、错误处理、React 最佳实践、API 设计等核心原则。

  • Readability First

    代码被阅读次数远多于编写次数，优先保证可读性

  • KISS Principle

    保持简单，避免过度工程化，易懂优于聪明代码

  • DRY Pattern

    不重复自己，提取公共逻辑，避免复制粘贴编程

  • YAGNI Approach

    不会需要它，避免预构建功能，只在需要时添加复杂性

    • 核心原理

    编码规范的核心是不可变性模式（CRITICAL）。始终创建新对象，绝不直接修改现有对象。这防止隐藏副作用，使调试更容易，支持安全并发。

    命名规范:

    • 变量: 描述性名称，如 marketSearchQuery、isUserAuthenticated
    • 函数: 动词-名词模式，如 fetchMarketData、calculateSimilarity
    • 布尔值: 使用 is/has/can 前缀，如 isValidEmail

    错误处理:

    • 在每一层显式处理错误
    • UI 层提供用户友好消息
    • 服务端记录详细错误上下文
    • 绝不静默吞掉错误

    输入验证:

    • 在系统边界验证所有用户输入
    • 使用 schema 验证（如 Zod）
    • 快速失败，清晰错误消息
    • 不信任外部数据（API 响应、文件内容）

    使用方法

    coding-standards Skill 在启动新项目、代码审查、重构或设置 linting 规则时触发：

    TypeScript 编码规范示例 
    // === 不可变性模式 (CRITICAL) ===

// ✅ GOOD: 使用 spread operator 创建新对象
const updatedUser = {
  ...user,
  name: 'New Name'
}
const updatedArray = [...items, newItem]

// ❌ BAD: 直接修改
user.name = 'New Name'  // 绝不这样做
items.push(newItem)     // 绝不这样做

// === 变量命名 ===

// ✅ GOOD: 描述性名称
const marketSearchQuery = 'election'
const isUserAuthenticated = true
const totalRevenue = 1000

// ❌ BAD: 不清晰名称
const q = 'election'
const flag = true
const x = 1000

// === 错误处理 ===

// ✅ GOOD: 全面错误处理
async function fetchData(url: string) {
  try {
    const response = await fetch(url)
    if (!response.ok) {
      throw new Error(`HTTP ${response.status}`)
    }
    return await response.json()
  } catch (error) {
    console.error('Fetch failed:', error)
    throw new Error('Failed to fetch data')
  }
}

// === Async 并行执行 ===

// ✅ GOOD: Promise.all 并行
const [users, markets, stats] = await Promise.all([
  fetchUsers(),
  fetchMarkets(),
  fetchStats()
])

// ❌ BAD: 不必要的顺序执行
const users = await fetchUsers()
const markets = await fetchMarkets()
const stats = await fetchStats()

    使用技巧

    1

    函数大小 - 保持函数小于 50 行，超过时拆分

    2

    嵌套深度 - 不超过 4 层，使用 early return 减少

    3

    命名常量 - 避免魔法数字，使用 MAX_RETRIES 等

    4

    注释 WHY - 解释为什么而非做什么
