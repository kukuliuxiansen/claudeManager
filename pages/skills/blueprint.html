蓝图

将单行目标转换为多会话、多代理工程项目的分步构建计划

类型: 规划工具

实现效果

应用蓝图技能，生成可被全新代理冷启动执行的构建计划。

  • 5阶段管道
    研究->设计->起草->审查->注册，每个步骤都有明确的输入输出和验证标准
  • 冷启动执行
    每个步骤包含自包含的上下文简报，无需先验上下文即可执行
  • 对抗性审查门
    每个计划由最强模型子代理审查，检查完整性和反模式检测
  • 并行步骤检测
    依赖图识别无共享文件或输出依赖的步骤，支持并行执行

    • 核心原理

    何时使用

    将大型功能分解为多个PR、规划跨会话的重构、协调并行工作流、任何上下文丢失会导致返工的任务。

    何时不使用

    单个PR可完成的任务、少于3次工具调用、用户说"直接做"。

    步骤分解

    目标分解为单PR大小的步骤（3-12个典型），分配依赖边、并行/串行顺序、模型层级和回滚策略。

    计划变异协议

    步骤可以拆分、插入、跳过、重新排序或放弃，带有正式协议和审计跟踪。

    使用方法

    蓝图调用示例 
    # 基本用法
/blueprint myapp "migrate database to PostgreSQL"

# 生成 plans/myapp-migrate-database-to-postgresql.md
# 包含步骤如：
# - Step 1: Add PostgreSQL driver and connection config
# - Step 2: Create migration scripts for each table
# - Step 3: Update repository layer to use new driver
# - Step 4: Add integration tests against PostgreSQL
# - Step 5: Remove old database code and config

# 多代理项目
/blueprint chatbot "extract LLM providers into a plugin system"

# 生成带有并行步骤的计划
# 如"implement Anthropic plugin"和"implement OpenAI plugin"
# 在插件接口步骤完成后并行执行

    使用技巧

    1
    确保git和gh可用，以获得完整的分支/PR/CI工作流计划
    2
    每个步骤都包含自包含的上下文简报，确保冷启动可执行
    3
    依赖图自动识别可并行执行的步骤，优化执行效率
    4
    计划变异协议支持步骤的拆分、插入、跳过和重新排序
