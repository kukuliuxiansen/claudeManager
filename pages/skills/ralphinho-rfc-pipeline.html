Ralphinho RFC Pipeline

RFC驱动多代理DAG执行模式，包含质量门、合并队列和工作单元编排

类型: Workflow Skill

实现效果

Ralphinho RFC Pipeline 提供RFC驱动的多代理DAG执行模式，适合大型功能分解为独立可验证工作单元。

  • DAG分解

    将RFC分解为有向无环图(DAG)的工作单元，明确依赖关系

  • 质量门

    每个工作单元必须通过测试、审查和验证才能进入合并队列

  • 合并队列

    自动管理单元合并顺序，确保依赖完整性

  • 恢复机制

    工作单元停滞时自动驱逐、快照、缩小范围重试

    • 核心原理

    Ralphinho Pipeline将大型RFC分解为可并行执行的工作单元，通过DAG管理依赖关系。

    Pipeline阶段:

    • 1. RFC intake: 接收RFC文档
    • 2. DAG decomposition: 分解为工作单元DAG
    • 3. Unit assignment: 分配给代理
    • 4. Unit implementation: 实现工作单元
    • 5. Unit validation: 验证工作单元
    • 6. Merge queue: 合并队列管理
    • 7. Final verification: 最终系统验证

    复杂度层级:

    • Tier 1: 隔离文件编辑，确定性测试
    • Tier 2: 多文件行为变更，中等集成风险
    • Tier 3: Schema/Auth/性能/安全变更

    使用方法

    当功能过大无法单次代理完成时使用此Skill：

    工作单元规范与合并规则 
    # 工作单元规范模板
Each work unit should include:
- id: 单元唯一标识
- depends_on: 依赖的其他单元ID
- scope: 工作范围描述
- acceptance_tests: 验收测试
- risk_level: 风险等级 (Tier 1/2/3)
- rollback_plan: 回滚计划

# 单元质量Pipeline
1. research        # 研究阶段
2. implementation plan  # 实现计划
3. implementation  # 实现
4. tests          # 测试
5. review         # 审查
6. merge-ready report  # 合并就绪报告

# 合并队列规则
- Never merge a unit with unresolved dependency failures.
- Always rebase unit branches on latest integration branch.
- Re-run integration tests after each queued merge.

# 恢复机制
If a unit stalls:
- evict from active queue
- snapshot findings
- regenerate narrowed unit scope
- retry with updated constraints

# 输出产物
- RFC execution log
- unit scorecards
- dependency graph snapshot
- integration risk summary

    使用技巧

    1

    明确依赖 - 工作单元的depends_on必须准确，避免循环依赖

    2

    分层风险 - Tier 3变更需要更严格的审查和验证

    3

    验收测试先行 - 每个单元必须有明确的验收标准

    4

    保持回滚计划 - 每个单元都应有可执行的回滚方案

    工作单元示例 
    # 工作单元定义示例
Unit: auth-service-extraction
- id: unit-003
- depends_on: [unit-001, unit-002]
- scope: Extract authentication service from monolith
- acceptance_tests:
    - All auth endpoints functional
    - Token validation working
    - Integration tests passing
- risk_level: Tier 3
- rollback_plan: Revert to monolith auth module
