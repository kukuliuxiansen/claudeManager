Chrome DevTools

通过MCP使用Chrome DevTools进行调试、故障排查和浏览器自动化

类型: 调试工具

实现效果

应用Chrome DevTools技能，实现高效的浏览器调试和自动化。

  • 浏览器生命周期管理
    首次工具调用时自动启动浏览器，使用持久化Chrome配置文件
  • 页面选择与交互
    使用list_pages查看可用页面，select_page切换上下文，通过uid与元素交互
  • 快照与自动化
    take_snapshot获取页面结构和元素uid，适合自动化和交互场景
  • 并行执行支持
    可以并行发送多个工具调用，但需保持正确顺序：navigate -> wait -> snapshot -> interact

    • 核心原理

    工作流模式

    与页面交互前：导航(navigate_page) -> 等待(wait_for) -> 快照(take_snapshot) -> 交互(使用uid)。

    工具选择

    自动化/交互使用take_snapshot（文本化、更快），视觉检查使用take_screenshot。

    高效数据获取

    使用filePath参数保存大型输出，使用分页和过滤最小化数据量，设置includeSnapshot: false减少不必要数据。

    使用方法

    工作流程 
    # 与页面交互前的标准流程
1. 导航: navigate_page 或 new_page
2. 等待: wait_for 确保内容加载完成
3. 快照: take_snapshot 理解页面结构
4. 交互: 使用快照中的元素uid进行click、fill等操作

# 高效数据获取
- 使用filePath参数保存大型输出（截图、快照、追踪）
- 使用分页(pageIdx, pageSize)和过滤(types)最小化数据
- 在输入操作上设置includeSnapshot: false除非需要更新页面状态

# 工具选择建议
- 自动化/交互: take_snapshot（文本化、更快、更适合自动化）
- 视觉检查: take_screenshot（当用户需要看到视觉状态时）
- 额外细节: evaluate_script获取accessibility tree中没有的数据

    使用技巧

    1
    如果找不到元素，获取新的快照 - 元素可能已被移除或页面已更改
    2
    可以并行发送多个工具调用，但保持正确顺序：navigate -> wait -> snapshot -> interact
    3
    对于大型输出使用filePath参数直接保存到文件，避免内存占用
    4
    如果chrome-devtools-mcp不足，引导用户使用Chrome DevTools UI界面
