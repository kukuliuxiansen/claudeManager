插件设置

使用.local.md文件存储插件配置和状态，支持YAML frontmatter结构化配置

类型: Configuration Skill

实现效果

插件设置Skill 提供 .claude/plugin-name.local.md 文件模式，用于存储项目级插件配置和状态。

  • YAML Frontmatter

    结构化配置使用YAML格式，支持字符串、布尔、数字、列表类型

  • Markdown主体

    配置文件可包含Markdown内容作为额外上下文或提示词

  • 快速退出模式

    Hook可快速检查文件存在和enabled字段，未配置时跳过

  • 多组件读取

    Hook、Command、Agent都可读取同一设置文件

    • 核心原理

    设置文件使用YAML frontmatter + Markdown结构，允许插件存储项目特定的配置。

    文件结构:

    • 位置: .claude/plugin-name.local.md 在项目根目录
    • 结构: YAML frontmatter + Markdown body
    • 生命周期: 用户管理，不提交git，应加入.gitignore

    解析技术:

    • 提取frontmatter: sed提取两个---之间的内容
    • 读取字段: grep + sed提取特定字段值
    • 读取body: awk提取第二个---之后的内容

    使用方法

    插件设置用于存储项目特定的配置和状态：

    设置文件示例与解析 
    # .claude/my-plugin.local.md
---
enabled: true
strict_mode: false
max_retries: 3
notification_level: info
coordinator_session: team-leader
dependencies: ["Task 3.4"]
---

# Plugin Configuration

This plugin is configured for standard validation mode.
Contact @team-lead with questions.

# Bash解析示例
STATE_FILE=".claude/my-plugin.local.md"

# 快速退出检查
if [[ ! -f "$STATE_FILE" ]]; then
  exit 0  # 未配置，跳过
fi

# 提取frontmatter
FRONTMATTER=$(sed -n '/^---$/,/^---$/{ /^---$/d; p; }' "$STATE_FILE")

# 读取字段
ENABLED=$(echo "$FRONTMATTER" | grep '^enabled:' | sed 's/enabled: *//')

# 检查是否启用
if [[ "$ENABLED" != "true" ]]; then
  exit 0  # 已禁用
fi

# 读取body
BODY=$(awk '/^---$/{i++; next} i>=2' "$STATE_FILE")

    使用技巧

    1

    添加gitignore - 将.claude/*.local.md加入.gitignore，避免提交敏感配置

    2

    提供默认值 - 文件不存在时使用合理默认值，而非报错

    3

    验证设置值 - 检查数值范围、枚举值，无效时使用默认值

    4

    文档重启需求 - 设置变更需要重启Claude Code才能生效
