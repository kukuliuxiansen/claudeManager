MCP集成

Model Context Protocol集成指南，将外部服务能力作为工具暴露给Claude Code

类型: Integration Skill

实现效果

MCP集成Skill 提供 Model Context Protocol 的完整集成指南，让插件能够连接外部服务并将其能力作为工具暴露给Claude Code。

  • 多种传输类型

    支持stdio、SSE、HTTP、WebSocket四种MCP服务器类型

  • OAuth认证

    SSE服务器自动处理OAuth流程，无需额外配置

  • 便携路径引用

    使用${CLAUDE_PLUGIN_ROOT}环境变量确保跨平台兼容

  • 工具命名空间

    自动前缀mcp__plugin_确保工具名称唯一且可识别

    • 核心原理

    Model Context Protocol (MCP) 是Claude Code与外部服务通信的标准协议，通过结构化工具访问实现服务能力暴露。

    MCP服务器类型:

    • stdio: 本地进程，通过stdin/stdout通信，适合自定义服务器
    • SSE: Server-Sent Events，适合云服务，支持OAuth
    • HTTP: REST API，适合Token认证的API后端
    • WebSocket: 实时双向通信，适合流式数据

    工具命名格式:

    mcp__plugin_<plugin-name>_<server-name>__<tool-name>

    使用方法

    MCP集成在需要连接外部服务、暴露工具能力时使用：

    MCP配置示例 
    // .mcp.json - 推荐的独立配置文件
{
  "database-tools": {
    "command": "${CLAUDE_PLUGIN_ROOT}/servers/db-server",
    "args": ["--config", "${CLAUDE_PLUGIN_ROOT}/config.json"],
    "env": {
      "DB_URL": "${DB_URL}"
    }
  },
  "asana": {
    "type": "sse",
    "url": "https://mcp.asana.com/sse"
  },
  "api-service": {
    "type": "http",
    "url": "https://api.example.com/mcp",
    "headers": {
      "Authorization": "Bearer ${API_TOKEN}"
    }
  }
}

// 在命令中使用MCP工具
---
allowed-tools: [
  "mcp__plugin_asana_asana__create_task",
  "mcp__plugin_asana_asana__search_tasks"
]
---

// 命令步骤
1. 收集任务详情
2. 调用mcp__plugin_asana_asana__create_task
3. 确认创建结果

    使用技巧

    1

    使用HTTPS - 生产环境永远使用HTTPS/WSS，避免不安全连接

    2

    环境变量注入 - 使用${VAR}语法从宿主环境注入敏感信息

    3

    预授权特定工具 - 只预授权必要的MCP工具，避免使用通配符

    4

    文档化依赖 - 在README中列出所有必需的环境变量

    MCP服务器类型参考 
    # MCP Server Types Quick Reference

| Type | Transport | Best For | Auth |
|------|-----------|----------|------|
| stdio | Process | Local tools, custom servers | Env vars |
| SSE | HTTP | Hosted services, cloud APIs | OAuth |
| HTTP | REST | API backends, token auth | Tokens |
| ws | WebSocket | Real-time, streaming | Tokens |

# Configuration Checklist
- [ ] Server type specified (stdio/SSE/HTTP/ws)
- [ ] Type-specific fields complete (command or url)
- [ ] Authentication configured
- [ ] Environment variables documented
- [ ] HTTPS/WSS used (not HTTP/WS)
- [ ] ${CLAUDE_PLUGIN_ROOT} used for paths

# Debug Commands
claude --debug           # Enable debug logging
/mcp                     # View all MCP servers and tools

# Common Issues
- Server not connecting: Check URL, verify server running
- Tools not available: Run /mcp, restart Claude Code
- Authentication failing: Clear cached tokens, re-authenticate
