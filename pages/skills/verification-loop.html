verification-loop

验证循环，确保代码修改的正确性和一致性

类型: Testing Skill

实现效果

verification-loop Skill 是Claude Code会话的综合验证系统，通过六阶段验证确保代码修改的质量。

  • 六阶段验证

    构建验证、类型检查、Lint检查、测试套件、安全扫描、Diff审查六个阶段

  • 覆盖率检查

    运行测试并检查覆盖率，目标最低80%覆盖率

  • 安全检查

    扫描硬编码密钥、API密钥、console.log等敏感代码模式

  • 验证报告

    生成包含各阶段结果、问题和改进建议的完整验证报告

    • 核心原理

    验证循环遵循"构建优先 + 安全扫描"原则。构建失败立即停止，安全扫描检查常见敏感代码模式。

    六阶段流程:

    • Phase 1: Build - 检查项目构建，失败立即停止
    • Phase 2: Type Check - TypeScript/Python类型检查
    • Phase 3: Lint - 代码风格检查
    • Phase 4: Test Suite - 运行测试并检查覆盖率（目标80%）
    • Phase 5: Security - 扫描密钥和敏感代码
    • Phase 6: Diff Review - 审查变更文件

    触发条件:

    • 完成功能或重要代码修改后
    • 创建PR之前
    • 希望确保质量门通过时
    • 重构之后

    使用方法

    verification-loop Skill 在需要验证代码修改时使用：

    验证命令 
    // Phase 1: Build Verification
npm run build 2>&1 | tail -20
# OR
pnpm build 2>&1 | tail -20

// Phase 2: Type Check
npx tsc --noEmit 2>&1 | head -30    # TypeScript
pyright . 2>&1 | head -30           # Python

// Phase 3: Lint Check
npm run lint 2>&1 | head -30        # JS/TS
ruff check . 2>&1 | head -30        # Python

// Phase 4: Test Suite
npm run test -- --coverage 2>&1 | tail -50

// Phase 5: Security Scan
grep -rn "sk-" --include="*.ts" --include="*.js" .
grep -rn "api_key" --include="*.ts" --include="*.js" .
grep -rn "console.log" --include="*.ts" --include="*.tsx" src/

// Phase 6: Diff Review
git diff --stat
git diff HEAD~1 --name-only

// 持续模式：每15分钟或重大变更后
/verify

    使用技巧

    1

    构建失败立即停止 - Phase 1失败不继续后续阶段，先修复构建

    2

    关注覆盖率 - 目标80%最低覆盖率，低于目标需补充测试

    3

    审查变更文件 - Diff Review检查非预期变更和潜在边界情况

    4

    持续模式 - 长会话每15分钟运行验证，或在完成函数/组件后
