troubleshooting

Chrome DevTools MCP故障排查，诊断和解决连接问题

类型: Debugging Skill

实现效果

troubleshooting Skill 在Chrome DevTools MCP工具失败时提供逐步诊断流程，帮助用户解决连接和目标问题。

  • 配置文件发现

    自动搜索并读取.mcp.json、settings.json、launch.json等配置文件

  • 常见错误分流

    针对DevToolsActivePort、Target closed、ProtocolError等常见错误提供精确诊断流程

  • 已知问题参考

    链接到GitHub troubleshooting文档，覆盖沙盒限制、WSL要求等高级场景

  • 诊断命令执行

    运行npx、DEBUG模式等命令测试服务器状态并捕获详细日志

    • 核心原理

    故障排查遵循"分流→诊断→修复"原则。首先识别常见错误模式，然后查阅已知问题，最后制定正确配置。

    常见错误模式:

    • DevToolsActivePort: autoConnect找不到Chrome调试端口文件
    • Target closed: 目标页面关闭或导航
    • ProtocolError: 网络启用超时或套接字意外关闭
    • Tool not found: 使用--slim模式但调用非导航工具

    沙盒限制:

    • macOS Seatbelt: Claude Desktop受系统沙盒限制
    • WSL: 需要特殊配置访问Windows Chrome
    • Linux容器: 可能需要--browserUrl替代autoConnect

    使用方法

    troubleshooting Skill 在list_pages、new_page、navigate_page失败时自动触发：

    诊断流程 
    // Step 1: 查找并读取配置
搜索文件: .mcp.json, gemini-extension.json,
         .claude/settings.json, .vscode/launch.json

// Step 2: 分流常见错误
DevToolsActivePort错误:
1. 确认正确Chrome版本正在运行
2. 打开 chrome://inspect/#remote-debugging
3. 启用"Enable remote debugging"
4. 调用 list_pages 验证连接

// Step 3: 制定配置
沙盒环境使用:
--browser-url=http://127.0.0.1:9222

// Step 4: 运行诊断命令
npx chrome-devtools-mcp@latest --help
DEBUG=* npx chrome-devtools-mcp@latest --logFile=/tmp/cdm-test.log

// Step 5: 检查GitHub问题
gh issue list --repo ChromeDevTools/chrome-devtools-mcp \
  --search "error snippet" --state all

    使用技巧

    1

    先确认Chrome运行 - autoConnect需要Chrome 144+正在运行并启用远程调试

    2

    沙盒用browserUrl - Claude Desktop等沙盒环境使用--browserUrl替代autoConnect

    3

    检查标志拼写 - 如autoBronnect等拼写错误会导致创建空配置文件

    4

    增加启动超时 - Windows Codex环境增加startup_timeout_ms到20000

    输出示例

    Configuration Fix 
    // 错误: Could not find DevToolsActivePort

// 诊断步骤:
1. Chrome Canary is running? ✓
2. chrome://inspect/#remote-debugging enabled? ✓
3. list_pages still fails? ✗

// 解决方案: 沙盒环境使用browserUrl
{
  "mcpServers": {
    "chrome-devtools": {
      "command": "npx",
      "args": [
        "-y",
        "chrome-devtools-mcp@latest",
        "--browser-url=http://127.0.0.1:9222"
      ]
    }
  }
}

// 启动Chrome调试模式
/Applications/Google\ Chrome.app/Contents/MacOS/Google\ Chrome \
  --remote-debugging-port=9222
