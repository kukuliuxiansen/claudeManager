search-first

搜索优先原则，在进行实现前先搜索现有代码和最佳实践

类型: Research Skill

实现效果

search-first Skill 是"先搜索后编码"工作流的系统化工具。在编写任何新代码之前，自动搜索现有工具、库和模式，避免重复造轮子，提升开发效率。

  • 并行多源搜索

    同时搜索 npm/PyPI、MCP 服务器、Claude Code Skills 和 GitHub，确保不遗漏任何可用资源

  • 决策矩阵

    根据搜索结果自动推荐 Adopt、Extend、Compose 或 Build 四种行动方案

  • Researcher Agent 集成

    复杂任务自动启动研究员代理进行深度搜索和结构化比较

  • 零自定义代码目标

    优先使用久经考验的开源方案，减少维护负担和潜在 Bug

    • 核心原理

    搜索优先遵循"确定性收集 + 智能决策"原则。先穷尽搜索所有可能的资源，再根据评分矩阵做出最优决策。

    决策矩阵:

    • Adopt: 精确匹配、维护良好、MIT/Apache 许可 → 直接安装使用
    • Extend: 部分匹配、基础良好 → 安装 + 编写薄包装层
    • Compose: 多个弱匹配 → 组合 2-3 个小包
    • Build: 无合适方案 → 自定义开发（但已通过研究获得最佳实践）

    评分维度:

    • 功能完整性 - 是否满足需求
    • 维护状态 - 最近更新时间、Issue 响应
    • 社区活跃度 - Star 数、下载量
    • 文档质量 - 是否有清晰的 API 文档
    • 许可证兼容性 - MIT/Apache 优先

    使用方法

    search-first Skill 在开始新功能、添加依赖或创建工具时自动触发：

    触发场景与调用 
    // 触发条件
1. 开始新功能开发
2. 添加依赖或集成
3. 创建新的工具或抽象
4. 用户说"添加 X 功能"

// 快速模式（内联）
0. 仓库中是否已存在？ → 用 rg 搜索相关模块/测试
1. 是否是常见问题？ → 搜索 npm/PyPI
2. 是否有 MCP 可用？ → 检查 ~/.claude/settings.json
3. 是否有 Skill 可用？ → 检查 ~/.claude/skills/
4. 是否有 GitHub 实现？ → 搜索开源项目

// 完整模式（Agent）
Task(subagent_type="general-purpose", prompt="
  Research existing tools for: [DESCRIPTION]
  Language/framework: [LANG]
  Constraints: [ANY]

  Search: npm/PyPI, MCP servers, Skills, GitHub
  Return: Structured comparison with recommendation
")

// 示例：添加死链检测
Need: Check markdown files for broken links
Search: npm "markdown dead link checker"
Found: textlint-rule-no-dead-link (score: 9/10)
Action: ADOPT — npm install textlint-rule-no-dead-link
Result: Zero custom code, battle-tested solution

    使用技巧

    1

    仓库优先 - 先用 rg 搜索项目内部，避免引入不必要的外部依赖

    2

    MCP 优先 - MCP 服务器提供的功能通常比 npm 包更稳定

    3

    避免过度包装 - 如果需要大量包装，考虑是否有更好的选择

    4

    检查许可证 - 优先选择 MIT、Apache-2.0 等宽松许可证

## Query: Add HTTP client with retry support

| Source | Package | Score | Verdict |

### Recommendation: ADOPT

**For Node.js:**
```bash
npm install got
```
- Built-in retry support
- Production-proven
- 15M+ weekly downloads

**For Python:**
```bash
pip install httpx
```
- Native async support
- HTTP/2 ready
- Active maintenance

### Implementation
```javascript
import got from 'got';

const client = got.extend({
  retry: { limit: 3 },
  timeout: { request: 10000 }
});
```
