skill-development

Claude Code插件技能开发指南，创建和维护高质量技能

类型: Plugin Development

实现效果

skill-development Skill 提供完整的技能开发工作流程，帮助开发者创建高质量、可维护的Claude Code插件技能。

  • 渐进式披露设计

    三层加载系统：元数据始终加载、SKILL.md触发时加载、资源按需加载，优化上下文使用

  • 资源组织结构

    scripts/存放可执行代码、references/存放参考文档、assets/存放输出资源文件

  • 触发描述优化

    第三人称格式描述，包含具体触发短语，确保技能在正确时机被加载

  • 验证检查清单

    完整的质量检查清单，覆盖结构、描述、内容、渐进式披露和测试要求

    • 核心原理

    技能开发遵循"渐进式披露 + 祈使语气"原则。通过三层加载系统管理上下文，使用第三人称和祈使语气确保内容一致性。

    技能解剖:

    • SKILL.md: 必需文件，包含YAML frontmatter（name + description）和Markdown指令
    • scripts/: 可执行代码，用于确定性任务或重复性操作
    • references/: 参考文档，按需加载到上下文
    • assets/: 输出资源文件（模板、图标、字体）

    渐进式披露:

    • 元数据 (name + description): 始终在上下文中 (~100词)
    • SKILL.md主体: 技能触发时加载 (1500-2000词理想)
    • 捆绑资源: 按需加载 (无限制)

    使用方法

    skill-development Skill 在创建或改进插件技能时使用：

    创建流程 
    // 创建技能目录结构
mkdir -p plugin-name/skills/skill-name/{references,examples,scripts}
touch plugin-name/skills/skill-name/SKILL.md

// SKILL.md frontmatter格式
---
name: Skill Name
description: This skill should be used when the user asks to "phrase1", "phrase2"...
version: 0.1.0
---

// 写作风格要求
- 使用祈使语气（"创建配置文件"而非"你应该创建"）
- 描述使用第三人称（"This skill should be used..."）
- 保持SKILL.md精简（1500-2000词）
- 详细内容移至references/

// 验证检查清单
1. SKILL.md存在且包含有效YAML frontmatter
2. 描述包含具体触发短语
3. 正文使用祈使语气
4. 所有引用文件存在
5. 测试技能在预期触发时加载

    使用技巧

    1

    保持SKILL.md精简 - 目标1500-2000词，详细内容移至references/

    2

    具体触发短语 - 描述中包含用户会说的具体短语，确保正确触发

    3

    避免重复内容 - 信息只存在于一个位置，SKILL.md或references二选一

    4

    参考示例技能 - 学习plugin-dev中的hook-development、agent-development等优秀示例
