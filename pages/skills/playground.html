Playground

创建交互式HTML工具，让用户通过控件配置、实时预览并生成提示词

类型: Tool Skill

实现效果

Playground Skill 创建自包含的HTML文件，包含交互控件、实时预览和可复制的提示词输出。

  • 单文件自包含

    所有CSS和JS内联，无外部依赖，确保离线可用

  • 实时预览

    每次控件变更立即更新预览，无需"应用"按钮

  • 自然语言输出

    提示词输出使用自然语言，只提及非默认选项

  • 预设与默认值

    3-5个命名预设，首次加载即有良好外观

    • 核心原理

    Playground是用户通过控件探索配置空间、可视化结果、然后复制生成的提示词的工具。

    模板类型:

    • design-playground: 视觉设计决策(组件、布局、颜色)
    • data-explorer: 数据和查询构建(SQL、API、正则)
    • concept-map: 学习和探索(概念图、知识缺口)
    • diff-review: 代码审查(git diff、PR)
    • code-map: 代码库架构(组件关系、数据流)

    状态管理模式:

    单一state对象，所有控件写入state，所有渲染从state读取

    使用方法

    当用户请求交互式playground、explorer或可视化工具时触发：

    状态管理与提示输出 
    // 状态管理模式
const state = { /* 所有可配置值 */ };

function updateAll() {
  renderPreview(); // 更新视觉效果
  updatePrompt();  // 重建提示文本
}
// 每个控件变更时调用updateAll()

// 提示输出模式
function updatePrompt() {
  const parts = [];

  // 只提及非默认值
  if (state.borderRadius !== DEFAULTS.borderRadius) {
    parts.push(`border-radius of ${state.borderRadius}px`);
  }

  // 使用定性语言配合数字
  if (state.shadowBlur > 16) parts.push('a pronounced shadow');
  else if (state.shadowBlur > 0) parts.push('a subtle shadow');

  prompt.textContent = `Update the card to use ${parts.join(', ')}.`;
}

// 完成后打开浏览器
// open playground.html

    使用技巧

    1

    避免值堆砌 - 提示输出应使用自然语言描述，而非原始值列表

    2

    分组控件 - 按关注点分组，将高级选项放入可折叠区域

    3

    提供上下文 - 提示词应包含足够信息，无需查看playground即可操作

    4

    深色主题 - 使用系统字体UI、等宽字体代码，保持简洁

  <style>

    body { font-family: system-ui; background: #1a1a2e; color: #fff; }

    <div class="preview">
      <!-- 实时预览 -->
    </div>
  </div>
  <div class="prompt">
    <p id="promptText"></p>

  </div>
  <script>
    // 内联所有JS
    const state = { borderRadius: 8, shadowSize: 'medium' };
    function updateAll() { /* ... */ }
  </script>
</body>
</html>
