Agent评估

编程Agent横向对比工具，在自定义任务上比较Claude Code、Aider、Codex等Agent的通过率、成本、时间和一致性

类型: Agent类

实现效果

轻量级CLI工具，用于在可重现任务上对比编程Agent。系统化"哪个编程Agent最好？"的比较。

  • YAML任务定义
    声明式定义任务，包含做什么、触摸哪些文件、如何判断成功
  • Git Worktree隔离
    每个Agent运行获得独立的git worktree，无需Docker即可实现可重现隔离
  • 多维度指标
    收集通过率、成本、时间、一致性等多项指标进行综合评估
  • 多种Judge类型
    支持代码测试、模式匹配、LLM评审等多种验证方式
  • 比较报告生成
    生成表格格式的对比报告，直观展示各Agent的性能差异

    • 核心原理

    评估流程：每个Agent运行在独立的git worktree中，避免相互干扰或破坏基础仓库。

    • 通过率：Agent是否生成了通过Judge的代码
    • 成本：每个任务的API花费（如果可用）
    • 时间：完成所需的秒数
    • 一致性：重复运行的通过率（如3/3 = 100%）

    使用方法

    触发场景与调用 
    # 1. 创建任务定义文件
mkdir tasks
# 编写 tasks/add-retry-logic.yaml

# 2. 运行Agent对比
agent-eval run --task tasks/add-retry-logic.yaml \
  --agent claude-code --agent aider --runs 3

# 3. 生成比较报告
agent-eval report --format table

# 输出示例:
# Task: add-retry-logic (3 runs each)
# | Agent        | Pass Rate | Cost   | Time   | Consistency |
# |--------------|-----------|--------|--------|-------------|
# | claude-code  | 3/3       | $0.12  | 45s    | 100%        |
# | aider        | 2/3       | $0.08  | 38s    |  67%        |

    使用技巧

    1
    从3-5个代表真实工作负载的任务开始，不要用玩具示例
    2
    每个Agent至少运行3次试验以捕获方差，Agent是非确定性的
    3
    在任务YAML中固定commit，以便结果在数天/数周内可重现
    4
    每个任务至少包含一个确定性Judge（测试、构建），LLM Judge会增加噪声
    5
    同时跟踪成本和通过率，95%通过率但成本10倍的Agent可能不是正确选择
    6
    对任务定义进行版本控制，它们是测试固定装置，应视为代码对待

    输出示例

    YAML任务定义示例 
    name: add-retry-logic
description: Add exponential backoff retry to the HTTP client
repo: ./my-project
files:
  - src/http_client.py
prompt: |
  Add retry logic with exponential backoff to all HTTP requests.
  Max 3 retries. Initial delay 1s, max delay 30s.
judge:
  - type: pytest
    command: pytest tests/test_http_client.py -v
  - type: grep
    pattern: "exponential_backoff|retry"
    files: src/http_client.py
commit: "abc1234"  # 固定到特定commit以确保可重现性
