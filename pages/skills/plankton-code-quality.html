Plankton代码质量

写入时代码质量强制执行，通过Hook自动格式化、检查并修复代码问题

类型: Quality Skill

实现效果

Plankton代码质量Skill 提供写入时的代码质量强制执行，通过PostToolUse Hook自动格式化、检查并修复代码问题。

  • 三阶段架构

    自动格式化 -> 收集违规 -> 委托修复，静默解决40-50%问题

  • 多语言支持

    Python、TypeScript、Shell、YAML、JSON、TOML、Markdown、Dockerfile

  • 分层模型路由

    Haiku处理简单格式、Sonnet处理逻辑、Opus处理类型系统

  • 配置保护

    防止LLM修改linter配置绕过规则，三层防护确保一致性

    • 核心原理

    Plankton通过Hook在每次文件编辑时自动执行质量检查，确保代码质量而不是在提交时才发现问题。

    三阶段架构:

    • Phase 1: 自动格式化 - 运行formatter静默修复40-50%问题
    • Phase 2: 收集违规 - 运行linter收集JSON格式违规
    • Phase 3: 委托修复 - 生成子进程修复剩余问题

    配置保护机制:

    • PreToolUse Hook阻止编辑linter配置
    • Stop Hook通过git diff检测配置变更
    • 保护.ruff.toml、biome.json等关键文件

    使用方法

    Plankton在文件编辑时自动触发，无需手动调用：

    设置与配置 
    # 快速开始
git clone https://github.com/alexfazio/plankton.git
cd plankton

# 安装核心依赖
brew install jaq ruff uv

# 安装Python linters
uv sync --all-extras

# 启动Claude Code - Hook自动激活
claude

# 配置文件 .claude/hooks/config.json
{
  "languages": {
    "python": true,
    "typescript": { "enabled": true, "biome_nursery": "warn" },
    "shell": true
  },
  "phases": {
    "auto_format": true,
    "subprocess_delegation": true
  },
  "subprocess": {
    "tiers": {
      "haiku":  { "timeout": 120, "max_turns": 10 },
      "sonnet": { "timeout": 300, "max_turns": 10 },
      "opus":   { "timeout": 600, "max_turns": 15 }
    }
  }
}

# 环境变量覆盖
HOOK_SKIP_SUBPROCESS=1      # 跳过Phase 3
HOOK_DEBUG_MODEL=1          # 打印模型选择日志

    使用技巧

    1

    禁用不需要的语言 - 在config.json中禁用不使用的语言以加速Hook

    2

    设置volume_threshold - 违规数超过阈值自动升级到更高模型层

    3

    与ECC配合使用 - ECC提供agents和rules，Plankton提供质量执行

    4

    避免Hook冲突 - 使用Plankton时禁用ECC的Prettier Hook
