security-review

安全审查技能，确保代码遵循安全最佳实践并识别潜在漏洞

类型: Security Skill

实现效果

security-review Skill 是安全编码的综合检查工具，覆盖OWASP Top 10关键领域，帮助开发者避免常见安全漏洞。

  • 密钥管理验证

    检测硬编码密钥，确保所有敏感信息使用环境变量或密钥管理器

  • 输入验证指南

    提供Zod模式验证示例，确保所有用户输入经过严格校验后再处理

  • 认证授权模式

    JWT安全存储方案（httpOnly cookie）、授权检查模式、Supabase RLS配置

  • 部署前安全检查清单

    完整的17项检查清单，覆盖XSS、CSRF、SQL注入、依赖安全等关键领域

    • 核心原理

    安全审查遵循"防御纵深 + 验证优先"原则。在系统边界进行严格验证，在每层都实施安全措施。

    十大安全领域:

    • 密钥管理: 绝不硬编码，使用环境变量或密钥管理器
    • 输入验证: 使用Zod等模式验证，白名单优于黑名单
    • SQL注入: 始终使用参数化查询，避免字符串拼接
    • 认证授权: httpOnly cookie存储token，实施角色检查
    • XSS防护: 使用DOMPurify清理HTML，配置CSP
    • CSRF保护: 使用CSRF token，SameSite=Strict
    • 速率限制: 所有API端点启用，敏感操作更严格
    • 数据暴露: 日志不记录敏感信息，错误消息通用化
    • 区块链安全: 验证钱包签名，验证交易详情
    • 依赖安全: 定期npm audit，提交lock文件

    使用方法

    security-review Skill 在涉及安全敏感操作时自动触发：

    触发场景与检查项 
    // 触发条件
1. 实现认证或授权功能
2. 处理用户输入或文件上传
3. 创建新的API端点
4. 处理密钥或凭证
5. 实现支付功能
6. 存储或传输敏感数据

// 快速检查清单
Secrets:
- [ ] 无硬编码密钥，全部使用环境变量
- [ ] .env.local 在 .gitignore 中
- [ ] 生产密钥在托管平台配置

Input:
- [ ] 所有用户输入使用模式验证
- [ ] 文件上传限制大小和类型
- [ ] 错误消息不泄露敏感信息

Auth:
- [ ] Token 存储在 httpOnly cookie
- [ ] 敏感操作前检查授权
- [ ] Supabase 启用行级安全

    使用技巧

    1

    环境变量验证 - 启动时验证必需的密钥是否存在，失败快速报错

    2

    白名单优于黑名单 - 验证输入时使用允许列表而非禁止列表

    3

    最小权限原则 - 只授予完成任务所需的最小权限

    4

    定期安全审计 - 每次部署前运行完整的安全检查清单
