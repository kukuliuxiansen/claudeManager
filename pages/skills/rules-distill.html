rules-distill

技能到规则的提炼工具，从多个技能中提取通用原则并生成规则文件

类型: Rule Management

实现效果

rules-distill Skill 是规则提炼的专业工具，能够扫描所有已安装的技能，自动提取跨技能的通用原则，并将其提炼为可执行的规则文件。

  • 自动技能扫描

    确定性收集所有技能文件，确保不会遗漏任何潜在的规则来源

  • 智能原则提取

    使用 LLM 判断提取跨技能的通用原则，确保规则的可操作性

  • 规则判决系统

    支持 Append、Revise、New Section、New File 等多种规则更新方式

  • 用户审核确认

    所有规则变更都需要用户明确批准，确保不会自动修改规则文件

    • 核心原理

    规则提炼遵循"确定性收集 + LLM 判断"原则。脚本保证穷尽性扫描，LLM 保证上下文理解。确保提取的原则符合以下标准：

    • 出现在 2+ 技能: 单技能特有的原则应保留在该技能中
    • 可执行的行为变更: 可以写成"做 X"或"不做 Y"
    • 清晰的违规风险: 忽略该原则会导致什么问题
    • 不与现有规则重复: 检查是否存在相同概念的不同表述

    设计原则:

    • What, not How - 提取原则(规则领域)，代码示例留在技能中
    • Link back - 草稿文本包含 `See skill: [name]` 引用
    • Anti-abstraction safeguard - 三层过滤器防止过度抽象

    使用方法

    规则提炼工具用于定期维护规则，或在新技能安装后验证规则完整性：

    触发场景与调用 
    // 触发条件
1. 定期规则维护（每月或安装新技能后）
2. skill-stocktake 揭示应成为规则的模式
3. 规则相对于使用的技能感觉不完整

// 运行命令
/rules-distill

// 工作流阶段
Phase 1: Inventory (确定性收集)
  - bash ~/.claude/skills/rules-distill/scripts/scan-skills.sh
  - bash ~/.claude/skills/rules-distill/scripts/scan-rules.sh

Phase 2: Cross-read, Match & Verdict (LLM 判断)
  - 批量处理技能（每批约20个）
  - 提取候选原则并判决

Phase 3: User Review & Execution
  - 展示摘要表格
  - 用户批准、修改或跳过

// 判决类型
| Verdict | Meaning |
|---------|---------|
| Append | 添加到现有规则的现有部分 |
| Revise | 修正不准确或不足的内容 |
| New Section | 在现有规则文件添加新部分 |
| New File | 创建新规则文件 |
| Already Covered | 已在规则中覆盖 |
| Too Specific | 应保留在技能级别 |

    使用技巧

    1

    定期运行 - 建议每月运行一次，保持规则与技能同步

    2

    审慎批准 - 仔细审查每个候选原则的违规风险说明

    3

    检查证据 - 确认候选原则确实出现在多个技能中

    4

    避免重复 - 检查是否与现有规则表达相同概念
