security-scan

使用AgentShield扫描Claude Code配置文件的安全漏洞和配置错误

类型: Security Skill

实现效果

security-scan Skill 使用AgentShield对Claude Code配置进行全面安全审计，识别潜在的注入风险和配置错误。

  • 多文件扫描

    扫描CLAUDE.md、settings.json、MCP配置、hooks和agent定义文件

  • 严重程度分级

    Critical/High/Medium/Info四级分类，A-F评分系统快速评估安全状态

  • 自动修复

    对可安全修复的问题提供自动修复功能，如替换硬编码密钥为环境变量

  • 多种输出格式

    支持Terminal、JSON、Markdown、HTML输出，便于CI/CD集成

    • 核心原理

    安全扫描遵循"全面覆盖 + 精确诊断"原则。扫描所有配置文件，根据风险等级提供建议。

    扫描范围:

    • CLAUDE.md: 硬编码密钥、自动运行指令、提示注入模式
    • settings.json: 过于宽松的允许列表、缺失的拒绝列表、危险绕过标志
    • mcp.json: 风险MCP服务器、硬编码环境密钥、npx供应链风险
    • hooks/: 通过插值的命令注入、数据泄露、静默错误抑制
    • agents/*.md: 不受限制的工具访问、提示注入面、缺失模型规范

    严重程度分级:

    • Critical: 立即修复 - 硬编码API密钥、Bash(*)权限、命令注入
    • High: 生产前修复 - 自动运行指令、缺失拒绝列表、不必要的Bash访问
    • Medium: 建议修复 - 静默错误抑制、缺失PreToolUse钩子
    • Info: 提醒 - 缺失MCP描述、良好实践的标志

    使用方法

    security-scan Skill 在配置Claude Code项目或定期安全检查时使用：

    扫描命令 
    // 安装AgentShield
npm install -g ecc-agentshield

// 基本扫描
npx ecc-agentshield scan

// 指定路径
npx ecc-agentshield scan --path /path/to/.claude

// 最小严重程度过滤
npx ecc-agentshield scan --min-severity medium

// 输出格式
npx ecc-agentshield scan --format json    # CI/CD集成
npx ecc-agentshield scan --format markdown # 文档
npx ecc-agentshield scan --format html    # 完整报告

// 自动修复
npx ecc-agentshield scan --fix

// Opus深度分析（需要API密钥）
export ANTHROPIC_API_KEY=your-key
npx ecc-agentshield scan --opus --stream

// 初始化安全配置
npx ecc-agentshield init

    使用技巧

    1

    CI/CD集成 - 在GitHub Actions中使用AgentShield自动扫描配置变更

    2

    定期扫描 - 安装新MCP或修改配置后运行安全扫描

    3

    修复优先级 - 先修复Critical和High级别问题，再处理Medium

    4

    理解风险 - 不是所有Info级别提示都需要处理，评估实际影响
