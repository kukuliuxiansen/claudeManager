插件结构

Claude Code插件标准化目录结构，支持自动组件发现和便携路径引用

类型: Architecture Skill

实现效果

插件结构Skill 定义Claude Code插件的标准目录布局，支持自动发现和组件加载。

  • 自动发现机制

    扫描commands/、agents/、skills/目录自动加载组件

  • 清单驱动配置

    .claude-plugin/plugin.json定义元数据和自定义路径

  • 便携路径引用

    ${CLAUDE_PLUGIN_ROOT}环境变量确保跨平台兼容

  • 组件化组织

    Commands、Agents、Skills、Hooks、MCP各司其职

    • 核心原理

    Claude Code插件遵循约定优于配置原则，标准目录结构自动触发组件发现。

    关键规则:

    • 清单位置: plugin.json必须在.claude-plugin/目录
    • 组件位置: 所有组件目录在插件根级别，不在.claude-plugin/内
    • 可选组件: 只创建插件实际使用的目录
    • 命名约定: 所有目录和文件使用kebab-case

    组件类型:

    • Commands: commands/目录，.md文件，集成为斜杠命令
    • Agents: agents/目录，.md文件，手动或自动调用
    • Skills: skills/目录，SKILL.md文件，自主激活
    • Hooks: hooks/hooks.json，响应Claude Code事件

    使用方法

    创建插件时遵循标准目录结构：

    目录结构与清单配置 
    # 标准插件目录结构
plugin-name/
├── .claude-plugin/
│   └── plugin.json          # 必需：插件清单
├── commands/                 # 斜杠命令 (.md文件)
├── agents/                   # 子代理定义 (.md文件)
├── skills/                   # Agent技能 (子目录)
│   └── skill-name/
│       └── SKILL.md         # 每个技能必需
├── hooks/
│   └── hooks.json           # 事件处理器配置
├── .mcp.json                # MCP服务器定义
└── scripts/                 # 辅助脚本和工具

# .claude-plugin/plugin.json
{
  "name": "plugin-name",
  "version": "1.0.0",
  "description": "Brief explanation",
  "commands": "./custom-commands",  # 自定义路径
  "agents": ["./agents", "./specialized-agents"]
}

# Hook中使用${CLAUDE_PLUGIN_ROOT}
{
  "PreToolUse": [{
    "matcher": "Write|Edit",
    "hooks": [{
      "type": "command",
      "command": "bash ${CLAUDE_PLUGIN_ROOT}/hooks/scripts/validate.sh"
    }]
  }]
}

    使用技巧

    1

    保持清单简洁 - 只在必要时指定自定义路径，依赖自动发现

    2

    始终使用变量 - 永远使用${CLAUDE_PLUGIN_ROOT}，避免硬编码路径

    3

    逻辑分组 - 将相关组件放在一起，scripts/使用子目录

    4

    包含README - 插件根目录和组件目录都应有文档
