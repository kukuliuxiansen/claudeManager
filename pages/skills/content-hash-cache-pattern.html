Content Hash Cache Pattern

基于内容哈希的文件缓存模式，自动失效，路径无关

类型: Backend Pattern

实现效果

Content Hash Cache Pattern 是一种高效的文件缓存策略，使用 SHA-256 内容哈希作为缓存键，实现路径无关、自动失效的缓存机制。

  • 路径无关

    文件重命名/移动后仍是缓存命中，基于内容而非路径

  • 自动失效

    内容变化时自动失效缓存，无需手动管理过期

  • O(1) 查找

    使用 {hash}.json 文件命名，无需索引文件即可快速查找

  • SRP 分离

    处理函数保持纯净，缓存作为独立服务层包装

    • 核心原理

    使用文件内容的 SHA-256 哈希作为缓存键，而非文件路径。这意味着文件重命名或移动后仍然是缓存命中，而内容变化时自动失效。

    关键设计决策:

    • SHA-256 内容哈希 - 路径无关，内容变化自动失效
    • {hash}.json 文件命名 - O(1) 查找，无需索引文件
    • 服务层包装 - 处理函数保持纯净，SRP 原则
    • 分块哈希 - 大文件分块处理，避免内存溢出

    何时使用:

    • PDF 解析、OCR、文本提取等高成本处理
    • 需要 --cache/--no-cache CLI 选项的工具
    • 相同文件跨批次重复处理的场景

    使用方法

    Python 实现示例 
    import hashlib
from pathlib import Path
from dataclasses import dataclass
import json

# 1. 计算文件哈希
_HASH_CHUNK_SIZE = 65536  # 64KB chunks

def compute_file_hash(path: Path) -> str:
    """SHA-256 of file contents (chunked for large files)."""
    sha256 = hashlib.sha256()
    with open(path, "rb") as f:
        while chunk := f.read(_HASH_CHUNK_SIZE):
            sha256.update(chunk)
    return sha256.hexdigest()

# 2. 缓存条目 (frozen dataclass)
@dataclass(frozen=True, slots=True)
class CacheEntry:
    file_hash: str
    source_path: str
    document: ExtractedDocument

# 3. 服务层包装 (SRP)
def extract_with_cache(
    file_path: Path,
    *,
    cache_enabled: bool = True,
    cache_dir: Path = Path(".cache"),
) -> ExtractedDocument:
    if not cache_enabled:
        return extract_text(file_path)  # 纯函数

    file_hash = compute_file_hash(file_path)

    # 检查缓存
    cached = read_cache(cache_dir, file_hash)
    if cached is not None:
        return cached.document

    # 缓存未命中 -> 提取 -> 存储
    doc = extract_text(file_path)
    write_cache(cache_dir, CacheEntry(
        file_hash=file_hash,
        source_path=str(file_path),
        document=doc
    ))
    return doc

    使用技巧

    1

    哈希内容 - 使用内容而非路径作为键，路径会变，内容不会

    2

    分块处理 - 大文件分块哈希，避免内存溢出

    3

    保持纯净 - 处理函数不应知道缓存的存在

    4

    优雅降级 - 损坏的缓存条目视为未命中，不崩溃
