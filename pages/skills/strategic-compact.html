strategic-compact

策略性压缩，在逻辑边界手动压缩上下文而非依赖自动压缩

类型: Context Management

实现效果

strategic-compact Skill 在逻辑边界建议手动压缩上下文，避免自动压缩在任务中途打断导致重要上下文丢失。

  • 工具调用追踪

    在PreToolUse钩子中追踪Edit/Write工具调用次数，达到阈值时建议压缩

  • 决策指南表

    提供阶段转换压缩决策表，明确何时压缩、何时不压缩及原因

  • 持久化说明

    明确压缩后什么保留（CLAUDE.md、TodoWrite、Memory文件）什么丢失（中间推理、已读文件）

  • 令牌优化模式

    触发表懒加载、上下文组合感知、重复指令检测等高级优化技巧

    • 核心原理

    策略压缩遵循"逻辑边界优先"原则。自动压缩在任意点触发，策略压缩在任务边界执行。

    何时激活:

    • 长会话接近上下文限制（200K+ tokens）
    • 多阶段任务（研究→计划→实现→测试）
    • 同一会话切换不相关任务
    • 完成主要里程碑后开始新工作
    • 响应变慢或不够连贯（上下文压力）

    压缩决策表:

    • 研究→计划: 压缩 - 研究上下文臃肿，计划是精炼输出
    • 计划→实现: 压缩 - 计划在TodoWrite/文件中，释放上下文给代码
    • 实现中途: 不压缩 - 丢失变量名、文件路径、部分状态代价高
    • 调试→新功能: 压缩 - 调试追踪污染不相关工作的上下文

    使用方法

    strategic-compact Skill 在长会话中管理上下文时使用：

    Hook配置与压缩命令 
    // 添加钩子到 ~/.claude/settings.json
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Edit",
        "hooks": [{ "type": "command", "command": "node ~/.claude/skills/strategic-compact/suggest-compact.js" }]
      },
      {
        "matcher": "Write",
        "hooks": [{ "type": "command", "command": "node ~/.claude/skills/strategic-compact/suggest-compact.js" }]
      }
    ]
  }
}

// 环境变量配置
COMPACT_THRESHOLD=50  // 首次建议前的工具调用次数（默认50）

// 手动压缩命令
/compact                      // 基础压缩
/compact Focus on implementing auth middleware next  // 带摘要压缩

    使用技巧

    1

    计划后压缩 - 计划确定并保存到TodoWrite后压缩，开始全新

    2

    调试后压缩 - 清除错误解决上下文后再继续其他工作

    3

    实现中途不压缩 - 保留相关变更的变量名、文件路径和部分状态

    4

    压缩前写入 - 将重要上下文保存到文件或内存，再执行压缩
