eval-harness

评估驱动开发(EDD)框架，提供 pass@k 指标、能力评估、回归测试等完整的评估体系

类型: Testing Skill

实现效果

eval-harness 是评估驱动开发(EDD)的专业框架，将评估视为"AI开发的单元测试"，在实现前定义预期行为，跟踪每次变更的回归情况。

  • 能力评估 (Capability Evals)

    测试 Claude 是否能完成之前无法完成的功能

  • 回归评估 (Regression Evals)

    确保变更不会破坏现有功能

  • 多种评分器

    代码评分器、模型评分器、人工评分器

  • pass@k 指标

    衡量"k次尝试中至少一次成功"的可靠性指标

    • 核心原理

    评估驱动开发的核心是先定义，后实现。在编写任何代码之前，先定义成功的标准，然后持续运行评估来验证。

    pass@k 指标体系:

    • pass@1: 首次尝试成功率
    • pass@3: 3次尝试内成功率 (目标 > 90%)
    • pass^3: 连续3次全部成功 (用于关键路径)

    评估流程:

    • Define - 在编码前定义评估标准
    • Implement - 编写代码通过评估
    • Evaluate - 运行评估并记录结果
    • Report - 生成评估报告

    使用方法

    eval-harness 适用于设置评估驱动开发、定义通过/失败标准、衡量 Agent 可靠性等场景。

    触发场景与调用 
    // 主动触发条件
1. 设置评估驱动开发 (EDD)
2. 定义 pass/fail 标准
3. 衡量 Agent 可靠性 (pass@k)
4. 创建回归测试套件
5. 跨模型版本基准测试

// 评估类型示例
[CAPABILITY EVAL: feature-name]
Task: Description of what Claude should accomplish
Success Criteria:
  - [ ] Criterion 1
  - [ ] Criterion 2
Expected Output: Description of expected result

// 评分器类型
1. Code-Based Grader - 确定性检查
   grep -q "export function handleAuth" src/auth.ts && echo "PASS"

2. Model-Based Grader - LLM评估
   评估代码变更是否解决问题、结构是否良好

3. Human Grader - 人工审核
   [HUMAN REVIEW REQUIRED]
   Change: Description
   Risk Level: LOW/MEDIUM/HIGH

    使用技巧

    1

    先定义后编码 - 强制清晰思考成功标准

    2

    频繁运行评估 - 尽早发现回归问题

    3

    跟踪 pass@k - 监控可靠性趋势

    4

    优先代码评分器 - 确定性优于概率性

    5

    安全需人工审核 - 永远不要完全自动化安全检查

    6

    保持评估快速 - 慢评估不会被运行
