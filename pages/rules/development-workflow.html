开发工作流规则

定义完整开发流程：研究复用、规划、TDD、代码审查、提交推送

类型: 通用规则

实现效果

开发工作流规则定义了功能实现的完整管道，确保代码质量和开发效率。

  • 研究复用优先

    实现前先搜索现有方案和库

  • 规划先行

    使用 planner agent 创建实现计划

  • TDD 开发

    使用 tdd-guide agent 进行测试驱动开发

  • 代码审查

    编写代码后立即使用 code-reviewer

    • 功能实现工作流

    0

    研究复用（强制）

    GitHub搜索 → 库文档 → Exa搜索 → 包注册表 → 寻找可适配实现

    1

    规划先行

    使用 planner agent，生成 PRD、架构、系统设计、技术文档、任务列表

    2

    TDD 方法

    使用 tdd-guide agent：RED → GREEN → IMPROVE，验证 80%+ 覆盖率

    3

    代码审查

    使用 code-reviewer agent，处理 CRITICAL 和 HIGH 问题

    4

    提交推送

    详细提交消息，遵循 conventional commits 格式

    规划规则

    除非明确要求，否则不要进入规划模式或创建实现计划。

    用户分享需求时 
    1. 先分析需求
2. 等待用户确认下一步
3. 只有用户明确要求时才创建计划

# 原因
创建详细实现计划会浪费时间，可能修改错误的代码库

    使用技巧

    1

    先搜索再实现 - GitHub 和包注册表优先

    2

    TDD 严格遵循 - 先写测试，再写实现

    3

    立即审查 - 编写代码后立刻 code-reviewer

    4

    等待确认 - 用户分享需求后等待确认
