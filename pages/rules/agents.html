Agent 编排规则

定义代理使用策略、触发时机和并行执行方法

类型: 通用规则

实现效果

Agent 编排规则定义了如何有效使用各种 AI 代理来加速开发流程，提高代码质量和安全性。

  • 自动触发策略

    无需用户提示，自动在适当时机触发相应代理

  • 并行执行

    独立任务并行执行，最大化效率

  • 多视角分析

    复杂问题使用分角色子代理进行全面分析

    • 核心原理

    可用代理列表（位于 ~/.claude/agents/）：

    Agent 用途 使用时机
    planner实现规划复杂功能、重构
    architect系统设计架构决策
    tdd-guide测试驱动开发新功能、Bug修复
    code-reviewer代码审查编写代码后
    security-reviewer安全分析提交前
    build-error-resolver修复构建错误构建失败时
    e2e-runnerE2E测试关键用户流程
    refactor-cleaner死代码清理代码维护
    doc-updater文档更新更新文档

    使用方法

    立即代理使用（无需用户提示）：

    自动触发场景 
    1. 复杂功能请求 → 使用 planner agent
2. 代码刚编写/修改 → 使用 code-reviewer agent
3. Bug修复或新功能 → 使用 tdd-guide agent
4. 架构决策 → 使用 architect agent

    并行任务执行（独立操作）：

    并行执行示例 
    # GOOD: 并行执行
Launch 3 agents in parallel:
1. Agent 1: Security analysis of auth module
2. Agent 2: Performance review of cache system
3. Agent 3: Type checking of utilities

# BAD: 不必要的串行
First agent 1, then agent 2, then agent 3

    使用技巧

    1

    识别时机 - 复杂任务自动触发相应代理

    2

    并行优先 - 独立任务同时执行，节省时间

    3

    多视角分析 - 复杂问题使用分角色子代理

    4

    代码审查 - 编写代码后立即使用 code-reviewer

    多视角分析角色

    复杂问题时使用分角色子代理进行全面分析：

    • 事实审查者 - 验证事实准确性
    • 资深工程师 - 技术深度分析
    • 安全专家 - 安全漏洞检查
    • 一致性审查者 - 代码风格和模式一致性
    • 冗余检查者 - 重复代码和逻辑识别
