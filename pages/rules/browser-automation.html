浏览器自动化规则

定义浏览器会话管理、连接策略和 MCP 配置方法

类型: 通用规则

核心规则

不要重启已经运行的 Chrome DevTools 或浏览器。连接到现有会话，不要重启。

实现效果

浏览器自动化规则确保正确的会话管理，避免不必要的重启和状态丢失。

  • 会话保持

    连接到现有浏览器会话，避免重启

  • 状态验证

    页面状态不清晰时先验证再操作

  • 隔离上下文

    需要隔离环境时使用 --isolated 标志

    • 核心原理

    会话管理策略：

    场景 操作
    浏览器已运行 连接到现有会话
    用户说"不要关闭" 保留所有现有进程
    页面状态不清晰 操作前验证
    需要隔离上下文 使用 --isolated 标志

    使用方法

    MCP 配置（会话开始时静态）：

    隔离浏览器会话 
    claude --mcp '{"browser": {"cmd": "npx", "args": ["-y", "@modelcontextprotocol/server-browser"]}}' -p "your task" --allowedTools "MCP"

    使用技巧

    1

    连接优先 - 浏览器已运行时连接而非重启

    2

    状态验证 - 页面状态不清晰时先验证

    3

    尊重用户 - 遵守"不要关闭"指令

    4

    隔离会话 - 需要时使用 --isolated 标志
