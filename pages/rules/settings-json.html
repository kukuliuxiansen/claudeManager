settings.json 配置

项目级设置文件，定义权限模式、自动批准工具和Hooks配置

类型: 项目规则

实现效果

settings.json 是项目级设置文件，控制 Claude Code 在项目中的行为。

  • 权限模式 - 定义自动批准或需要确认的工具
  • Hooks 配置 - 设置工具执行前后的钩子
  • MCP 服务器 - 配置 MCP 服务器连接
  • 模型设置 - 指定默认模型和参数

文件位置

settings.json 文件应放置在项目 .claude 目录：

位置
[项目根目录]/.claude/settings.json

# 示例
/Users/weidian/myproject/.claude/settings.json

典型配置结构

settings.json 示例
{
  "permissions": {
    "allow": [
      "Read(**)",
      "Bash(npm run *)",
      "Bash(git status)"
    ],
    "deny": [
      "Bash(rm -rf *)"
    ]
  },
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Bash",
        "hooks": ["echo 'Running bash command'"]
      }
    ],
    "PostToolUse": [
      {
        "matcher": "Write",
        "hooks": ["npm run lint"]
      }
    ]
  },
  "mcpServers": {
    "browser": {
      "command": "npx",
      "args": ["-y", "@anthropic-ai/mcp-server-browser"]
    }
  }
}

使用技巧

1

权限配置 - 定义允许和拒绝的工具

2

3

MCP 服务器 - 连接外部工具服务

4

谨慎配置 - 避免 dangerous 权限
