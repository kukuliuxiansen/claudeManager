性能优化规则

定义模型选择策略、上下文窗口管理、Extended Thinking 和构建调试方法

类型: 通用规则

实现效果

性能优化规则帮助选择正确的模型和管理上下文，最大化效率和控制成本。

  • 模型选择策略

    根据任务复杂度选择 Haiku、Sonnet 或 Opus

  • 上下文窗口管理

    避免在上下文窗口最后 20% 处理复杂任务

  • Extended Thinking

    默认启用深度思考，复杂任务使用 Plan Mode

    • 核心原理

    模型选择策略：

    Haiku 4.5

    90% Sonnet 能力，3x 成本节省

    • 轻量代理频繁调用
    • 结对编程和代码生成
    • 多代理系统中的工作者代理

    Sonnet 4.6

    最佳编码模型

    • 主要开发工作
    • 编排多代理工作流
    • 复杂编码任务

    Opus 4.5

    最深推理

    • 复杂架构决策
    • 最大推理要求
    • 研究和分析任务

    使用方法

    上下文窗口管理：

    上下文敏感度指南 
    # 避免在上下文窗口最后 20% 处理：
- 大规模重构
- 跨多个文件的功能实现
- 调试复杂交互

# 较低上下文敏感度任务：
- 单文件编辑
- 独立工具创建
- 文档更新
- 简单 Bug 修复

    Extended Thinking 控制：

    Extended Thinking 配置 
    # 切换
Option+T (macOS) / Alt+T (Windows/Linux)

# 配置
设置 ~/.claude/settings.json 中的 alwaysThinkingEnabled

# 预算上限
export MAX_THINKING_TOKENS=10000

# 详细模式
Ctrl+O 查看思考输出

    使用技巧

    1

    Haiku 优先 - 轻量任务用 Haiku 节省成本

    2

    上下文监控 - 避免在窗口末尾处理复杂任务

    3

    Plan Mode - 复杂任务启用规划模式

    4

    构建调试 - 构建失败用 build-error-resolver
