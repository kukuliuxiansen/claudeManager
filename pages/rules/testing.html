测试要求规则

定义最低覆盖率、测试类型、TDD 工作流和失败排查策略

类型: 通用规则
80%
最低测试覆盖率要求

实现效果

测试要求规则确保代码质量和可靠性，通过测试驱动开发保证功能正确性。

  • 单元测试

    测试独立函数、工具和组件

  • 集成测试

    测试 API 端点和数据库操作

  • E2E 测试

    测试关键用户流程（框架按语言选择）

    • TDD 工作流

    强制性工作流：

    1

    RED - 先写测试

    编写测试，运行后应该失败

    2

    GREEN - 写最小实现

    编写最小代码使测试通过

    3

    IMPROVE - 重构

    改进代码质量，保持测试通过

    4

    验证覆盖率

    确保达到 80%+ 覆盖率

    测试失败排查

    排查步骤 
    1. 使用 tdd-guide agent
2. 检查测试隔离
3. 验证 mock 是否正确
4. 修复实现，而非测试（除非测试本身错误）

    使用技巧

    1

    测试先行 - 新功能使用 tdd-guide

    2

    覆盖率监控 - 始终保持 80%+ 覆盖率

    3

    三种测试 - 单元、集成、E2E 都必需

    4

    修复实现 - 通常问题在实现而非测试
