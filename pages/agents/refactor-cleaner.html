重构清理代理

检测并清理死代码、重复代码和未使用依赖

类型: Agent类

实现效果

重构清理代理专注于代码洁癖和可持续维护，自动检测并安全地清理项目中的冗余代码。

  • 死代码检测

    识别未使用的导出、函数、变量和文件

  • 重复代码消除

    识别并合并重复的组件和工具函数

  • 依赖清理

    移除未使用的npm包和导入

  • 安全重构

    确保每次改动后测试通过且无回归

    • 核心原理

    重构清理代理基于"持续清洁"原则，使用专业工具进行静态分析，并遵循严格的验证流程：

    • 工具驱动检测 - 使用 knip、depcheck、ts-prune 等工具识别冗余代码
    • 引用验证 - 通过 grep 确保代码确实未被引用（包括动态导入）
    • 风险分级 - 将清理项分为 SAFE（安全）、CAREFUL（谨慎）、RISKY（高风险）三个等级
    • 渐进式清理 - 每次只清理一个类别，运行测试，提交后再继续
    • 最小化破坏 - 只移除真正的死代码，保留可能用于未来需求的代码

    使用方法

    触发时机与命令 
    // 触发时机
1. 代码审查时发现大量重复代码
2. 项目依赖体积过大
3. 代码库中存在可疑的未使用文件
4. 重构前的准备工作

// 推荐命令（按风险等级）
npx knip                                    # 未使用的文件、导出、依赖
npx depcheck                                # 未使用的 npm 依赖
npx ts-prune                                # 未使用的 TypeScript 导出
npx eslint . --report-unused-disable-directives  # 未使用的 eslint 指令

    使用技巧

    1

    从 SAFE 开始 - 先清理未使用的开发依赖，风险最低

    2

    一次一类 - 每次只清理一个类别（依赖→导出→文件）

    3

    测试驱动 - 每批删除后立即运行测试和构建

    4

    提交每批 - 每批清理后提交，便于回滚

    5

    动态导入 - 严格检查通过字符串动态引用的代码
