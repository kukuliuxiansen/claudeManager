构建错误解决代理

专业TypeScript错误和构建失败修复代理

类型: Agent类

实现效果

构建错误解决代理专门修复TypeScript类型错误、构建失败和依赖问题。它遵循最小变更原则，仅修复错误而不重新设计架构。

  • TypeScript错误解析

    修复类型错误、类型推断问题、泛型约束

  • 构建错误修复

    解决编译失败、模块解析问题

  • 依赖问题解决

    修复导入错误、缺失包、版本冲突

  • 最小差异修复

    进行尽可能小的更改来修复错误

    • 核心原理

    该代理使用诊断-修复-验证循环来解决构建问题。

    诊断命令 
    npx tsc --noEmit --pretty
npx tsc --noEmit --pretty --incremental false
npm run build
npx eslint . --ext .ts,.tsx,.js,.jsx

    修复策略（最小变更）:

    • 仔细阅读错误消息 - 理解期望值与实际值
    • 找到最小修复（类型注解、空检查、导入修复）
    • 验证修复不破坏其他代码 - 重新运行tsc
    • 迭代直到构建通过

    常见修复模式:

    错误类型修复方法
    隐式any类型添加类型注解
    对象可能为undefined可选链 `?.` 或空检查
    属性不存在添加到接口或使用可选 `?`
    找不到模块检查tsconfig路径、安装包或修复导入路径
    类型不兼容解析/转换类型或修复类型
    泛型约束添加 `extends { ... }`

    使用方法

    构建错误解决代理在出现构建失败时自动触发：

    工作流程示例 
    ### 1. 收集所有错误
- 运行 `npx tsc --noEmit --pretty` 获取所有类型错误
- 分类：类型推断、缺失类型、导入、配置、依赖
- 优先级：首先修复构建阻塞，然后是类型错误，最后是警告
### 2. 修复策略（最小变更）
对于每个错误：
1. 仔细阅读错误消息 - 理解期望值与实际值
2. 找到最小修复（类型注解、空检查、导入修复）
3. 验证修复不破坏其他代码 - 重新运行tsc
4. 迭代直到构建通过
### 3. 常见修复
| 错误 | 修复 |
|------|------|
| `隐式any类型` | 添加类型注解 |
| `对象可能为undefined` | 可选链或空检查 |
| `属性不存在` | 添加到接口或使用可选 |
| `找不到模块` | 检查配置或修复路径 |
| `类型不兼容` | 解析或转换类型 |

    使用技巧

    1

    优先级排序 - 首先修复CRITICAL级别的构建阻塞错误

    2

    3

    快速恢复 - `rm -rf .next node_modules/.cache && npm run build`

    4

    应用范围 - 只修复错误，不重新设计架构

### 成功指标
- `npx tsc --noEmit` 退出码为0
- `npm run build` 成功完成
- 未引入新错误
- 最小化变更行数 (< 5% 受影响文件)
- 测试仍通过

### 何时不使用
- 代码需要重构 → 使用 `refactor-cleaner`
- 需要架构变更 → 使用 `architect`
- 需要新功能 → 使用 `planner`
- 测试失败 → 使用 `tdd-guide`
- 安全问题 → 使用 `security-reviewer`

**记住**：修复错误，验证构建通过，然后继续。速度和精度比完美更重要。
