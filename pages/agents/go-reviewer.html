Go审查代理

专业审查Go代码质量、安全性和并发模式的智能代理

类型: Agent类

实现效果

专注于Go代码审查的智能代理，系统化检查安全漏洞、错误处理、并发问题和代码质量。

  • SQL注入检测
    识别database/sql查询中的字符串拼接风险
  • 命令注入检测
    识别os/exec中未经验证的用户输入
  • 错误处理审查
    识别被丢弃的错误、缺少错误包装和panic滥用
  • 并发问题检测
    识别Goroutine泄漏、未缓冲通道死锁和缺少WaitGroup
  • 上下文管理审查
    验证context.Context作为函数第一参数和取消机制
  • 性能优化建议
    识别字符串拼接循环、缺少slice预分配和N+1查询

    • 核心原理

    Go审查代理基于严重程度分级体系审查代码，优先识别SQL注入、命令注入和错误处理等CRITICAL安全问题，然后关注并发模式、上下文管理和代码质量等HIGH级别问题。代理使用go vet、staticcheck和golangci-lint作为辅助诊断工具，结合代码模式识别提供精准诊断。

    使用方法

    触发场景与调用 
    # 触发场景
- 代码提交前的安全审查
- Pull Request的质量检查
- Go项目定期代码审计

# 在Claude Code中使用
claude -p "审查Go代码安全性和质量" --allowedTools "demo"

    使用技巧

    1
    使用参数化查询避免SQL注入
    2
    错误必须包装：fmt.Errorf("context: %w", err)
    3
    Goroutine必须有context.Context取消机制
    4
    使用defer mu.Unlock()而非手动lock/unlock
    5
    context.Context必须作为函数第一个参数
    6
    错误消息格式：小写，无标点

    输出示例

    Go Code Review Output

[CRITICAL] SQL注入风险
文件: internal/repository/users.go:42
问题: `fmt.Sprintf("SELECT * FROM users WHERE id = %d", id)` 使用字符串拼接
修复: 使用参数化查询 `db.QueryContext(ctx, "SELECT * FROM users WHERE id = ?", id)`

[CRITICAL] 错误被丢弃
文件: internal/handler/auth.go:18
问题: `_ = json.NewDecoder(r.Body).Decode(&req)` 忽略错误
修复: `if err := json.NewDecoder(r.Body).Decode(&req); err != nil { ... }`

=== HIGH 并发问题 ===
[HIGH] Goroutine泄漏风险
文件: internal/worker/processor.go:55
问题: 启动Goroutine无context.Context取消机制
修复: 添加 `ctx, cancel := context.WithCancel(ctx)` 并在Goroutine中检查ctx.Done()

[HIGH] 缺少WaitGroup
文件: internal/handler/batch.go:32
问题: 多个Goroutine并行执行无协调
修复: 使用sync.WaitGroup等待所有Goroutine完成

=== MEDIUM 性能优化 ===
[MEDIUM] 循环中字符串拼接
文件: internal/utils/formatter.go:28
问题: 使用`result += part`在循环中
修复: 使用`strings.Builder`或 Pre-allocation

[MEDIUM] N+1查询模式
文件: internal/repository/posts.go:45
问题: 为每个post单独查询评论
修复: 使用JOIN或批量查询
