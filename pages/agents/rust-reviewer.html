Rust审查代理

专业 Rust 代码审查，涵盖安全性、所有权模式和性能优化

类型: Agent类

实现效果

Rust审查代理执行全面的 Rust 代码审查，重点关注安全性、所有权模式、并发正确性和代码质量。

  • 安全审查

    检测不安全代码、注入、路径遍历和硬编码密钥

  • 所有权分析

    检查不必要的克隆、String vs &str 使用、Cow 模式

  • 并发审查

    检查 Send/Sync 边界、死锁模式和未绑定通道

  • 代码质量

    检查函数大小、嵌套深度、Clippy 警告和文档

    • 核心原理

    Rust审查代理基于三类风险等级进行审查，优先处理可能引起运行时错误和安全问题的代码：

    • CRITICAL - 安全相关 - 未检查的 unwrap()、不安全代码、注入漏洞、硬编码密钥
    • HIGH - 所有权和并发 - 不必要的克隆、缺少 Send/Sync 边界、死锁模式
    • MEDIUM - 代码质量和性能 - 大函数、重复分配、Clippy 警告、缺少文档

    审查标准遵循"顶级 Rust 代码库"的质量要求，参考 rust-patterns skill 以获取详细代码模式。

### CRITICAL - 安全问题 (1)

#### 问题 1: 未检查的 unwrap() 在生产路径
**文件**: src/service/auth.rs:45
**代码**: `let config = fs::read_to_string("config.json").unwrap();`
**风险**: 应用在配置文件不存在时会 panic
**修复**: 使用 match 或 ? 运算符
```rust
// 修复前
let config = fs::read_to_string("config.json").unwrap();

// 修复后
let config = match fs::read_to_string("config.json") {
    Ok(c) => c,
    Err(e) => {
        log::error!("Failed to read config: {}", e);
        return Err(AuthError::ConfigMissing);
    }
};
```

### HIGH - 所有权/并发 (2)

#### 问题 2: 不必要的 HashMap 克隆
**文件**: src/handler/user.rs:78
**代码**: `let users = db.fetch_all().map(|v| v.clone()).collect();`
**风险**: 全量克隆可能非常昂贵
**修复**: 使用迭代器 adaptors 而非 collect() + clone()
```rust
// 修复前
let users = db.fetch_all().map(|v| v.clone()).collect();

// 修复后
let users: Vec<_> = db.fetch_all().collect();
```

### MEDIUM - 代码质量 (2)

#### 问题 3: Clippy 警告未处理
**文件**: src/lib.rs:12
**警告**: `/pub use std::collections::HashMap;` - HashMap 默认已导出
**建议**: 移除冗余的 pub use 指令

#### 问题 4: 公共结构缺少文档
**文件**: src/model/user.rs:5
**代码**: `pub struct User { id: u64, name: String }`
**建议**: 添加 #[derive(Debug, Clone)] 和文档注释

/// 表示系统用户

}
