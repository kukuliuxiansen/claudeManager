安全审查代理

主动检测安全漏洞、密钥泄露和 OWASP Top 10 问题

类型: Agent类

实现效果

安全审查代理在处理用户输入、认证、API 端点或敏感数据时主动触发，执行全面的安全审计。

  • 漏洞检测

    识别 OWASP Top 10 和常见安全漏洞

  • 密钥检测

    发现硬编码的 API 密钥、密码和认证令牌

  • 输入验证

    确保所有用户输入经过正确清理和验证

  • 权限验证

    检查认证和访问控制是否正确实施

    • 核心原理

    安全审查代理遵循安全优先原则，在关键操作后立即进行审查：

    • 被动触发 - 在处理安全相关代码时自动触发审查
    • OWASP Top 10 覆盖 - 审查所有 10 个类别涵盖的漏洞模式
    • 工具辅助检测 - 使用 npm audit、eslint-security 插件等
    • 纵深防御 - 检查多个安全层：输入验证、认证、权限、日志
    • 修复建议 - 提供具体的代码修复示例而不是仅报告问题

## 发现漏洞

### [CRITICAL] 硬编码 API 密钥
**文件**: src/api/client.ts:42
**问题**: API 密钥直接暴露在源代码中
**修复方案**:
```typescript
// 错误
const apiKey = "sk-abc123"

// 正确
const apiKey = process.env.OPENAI_API_KEY
if (!apiKey) {
  throw new Error("API key not configured")
}
```

### [CRITICAL] SQL 注入风险
**文件**: src/db/query.ts:28
**问题**: 字符串拼接构建 SQL 查询
**修复方案**:
```typescript
// 错误
const query = `SELECT * FROM users WHERE id = ${userId}`

// 正确
const query = `SELECT * FROM users WHERE id = $1`
const result = await db.query(query, [userId])
```

### [HIGH] XSS 漏洞
**文件**: src/components/Comment.tsx:15
**问题**: 用户输入未转义直接渲染
**修复方案**:
```tsx
// 错误
<div>{userComment}</div>

// 正确
<div>{DOMPurify.sanitize(userComment)}</div>
// 或使用 textContent
```
