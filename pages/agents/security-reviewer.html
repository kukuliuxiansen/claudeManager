security-reviewer

专业安全审计技能，检测漏洞、密钥泄露和 OWASP Top 10 问题

类型: Security Agent

实现效果

security-reviewer Agent 是专业的安全审计代理，主动在处理用户输入、认证、API端点或敏感数据后立即进行安全审查。识别 OWASP Top 10 漏洞、硬编码密钥、注入攻击等安全风险。

  • 漏洞检测

    识别 OWASP Top 10 和常见安全漏洞

  • 密钥检测

    发现硬编码的 API 密钥、密码、令牌

  • 输入验证

    确保所有用户输入正确清理和验证

  • 权限验证

    检查认证和访问控制是否正确实施

    • OWASP Top 10 检查

    安全审查覆盖 OWASP Top 10 所有类别：

    A01
    注入攻击
    A02
    失效认证
    A03
    敏感数据泄露
    A04
    XML外部实体
    A05
    失效访问控制
    A06
    安全配置错误
    A07

    A10

    使用方法

    security-reviewer Agent 在处理安全相关代码时主动触发：

    触发场景与检测模式 
    // 主动触发条件
1. 新 API 端点 - 检查认证和输入验证
2. 认证代码变更 - 验证密码哈希和会话
3. 用户输入处理 - 检查注入和XSS
4. 数据库查询 - 验证参数化查询
5. 文件上传 - 检查路径遍历
6. 支付代码 - 验证安全和合规

// 关键检测模式
| 模式 | 严重级别 | 修复方法 |
|-----|---------|---------|
| 硬编码密钥 | CRITICAL | 使用 process.env |
| Shell命令用户输入 | CRITICAL | 使用安全API |
| 字符串拼接SQL | CRITICAL | 参数化查询 |
| innerHTML用户输入 | HIGH | textContent/DOMPurify |
| fetch用户URL | HIGH | 白名单域名 |
| 明文密码比较 | CRITICAL | bcrypt.compare() |

// 安全审查命令
npm audit --audit-level=high
npx eslint . --plugin security

    安全原则

    1

    纵深防御 - 多层安全防护，不只依赖单一措施

    2

    最小权限 - 只授予必需的最小权限

    3

    安全失败 - 错误不应暴露敏感数据

    4

    不信输入 - 验证和清理所有外部输入

    5

    定期更新 - 保持依赖最新版本

    输出示例

    Security Review Report 
    # 安全审查报告

## 发现漏洞

### [CRITICAL] 硬编码 API 密钥
文件: src/api/client.ts:42
问题: API 密钥直接暴露在源代码中
修复: 移至环境变量
  const apiKey = "sk-abc123"        // 错误
  const apiKey = process.env.API_KEY // 正确

### [CRITICAL] SQL 注入风险
文件: src/db/query.ts:28
问题: 字符串拼接构建 SQL 查询
修复: 使用参数化查询
  const query = `SELECT * FROM users WHERE id = ${userId}`  // 错误
  const query = `SELECT * FROM users WHERE id = $1`         // 正确

### [HIGH] XSS 漏洞
文件: src/components/Comment.tsx:15
问题: 用户输入未转义直接渲染
修复: 使用 textContent 或 DOMPurify
  <div>{userComment}</div>           // 错误
  <div>{DOMPurify.sanitize(userComment)}</div> // 正确

## 审查摘要
| 严重级别 | 数量 | 状态 |
|---------|------|------|
| CRITICAL | 2    | block |
| HIGH     | 1    | warn |
| MEDIUM   | 0    | pass |
| LOW      | 0    | pass |

结论: BLOCK - 2 个 CRITICAL 漏洞必须立即修复

    紧急响应

    发现 CRITICAL 漏洞时的处理流程：

    • 1. 记录报告 - 详细记录漏洞详情
    • 2. 立即通知 - 告警项目负责人
    • 3. 提供修复 - 给出安全代码示例
    • 4. 验证修复 - 确认修复方案有效
    • 5. 密钥轮换 - 如果凭证暴露，立即轮换
