静默失败检测代理

识别并报告未记录和未通知用户的错误处理代码

类型: Agent类

实现效果

静默失败检测代理专注于发现代码中被隐藏的错误，这些错误可能导致调试困难和用户体验问题。

  • 空 Catch 块检测

    识别不记录也不处理错误的 catch 块

  • 错误静默检测

    发现仅记录但继续执行的错误处理代码

  • 空回退检测

    识别不通知用户的回退逻辑和 mock 数据

  • 宽泛异常捕获检测

    发现可能隐藏意外错误的通用 catch 块

    • 核心原理

    静默失败检测代理基于五个不可协商的原则：

    • 静默失败不可接受 - 任何错误都必须被记录和通知
    • 用户应获得可操作反馈 - 每个错误消息必须告诉用户发生了什么以及如何解决
    • 回退必须显式和有理由 - 不经过用户授权的回退是隐藏问题
    • Catch 块必须具体 - 只捕获预期的错误类型
    • Mock 实现仅在测试中 - 生产代码不应回退到 mock

    使用方法

    审查流程与触发时机 
    // 触发时机
1. PR 审查时 - 检查所有错误处理代码
2. 生产环境问题分析 - 查找可能的静默失败点
3. 代码迁移审查 - 确保错误处理符合新环境标准

// 审查流程
1. 找到所有 try-catch 块和错误回调
2. 检查每个 catch 块是否:
   - 记录了足够上下文
   - 提供了用户可操作的反馈
   - 只捕获了预期的错误类型
   - 没有使用空 catch 块
3. 找到所有 fallback 逻辑
4. 验证是否通知了用户 fallback 发生

// 常见静默失败模式
| 模式 | 描述 | 等级 |
|-----|------|------|
| 空 catch | catch {} 内为空 | CRITICAL |
| 仅日志继续 | log.error(e); continue | HIGH |
| 返回默认值 | catch { return null } | HIGH |
| 可选链隐藏 | a?.b?.c?.d | MEDIUM |
| 重试无通知 | while(retry--) { ... } | MEDIUM |

    使用技巧

    1

    每行 Ø 问题 - 即使是最小的 catch 也必须有理由

    2

    具体优于宽泛 - 捕获特定错误而非通用 Exception

    3

    回退需显式 - fallback 必须通知用户或记录原因

    4

    上下文重要 - 日志必须包含操作、ID 和相关状态

    5

    测试文件例外 - mock 只在测试中使用，不许进入生产

## 发现的问题

### [CRITICAL] 空 catch 块 - src/api/client.ts:145
**位置**: `src/api/client.ts:145-148`
**代码**:
```typescript
try {
  await api.send(data);
} catch (e) {
  // empty - 错误被完全静默
}
```
**问题**:
- 错误被完全静默，调用者不知道失败
- 调试时无法追踪问题起源
- 用户不知道操作失败

**修复方案**:
```typescript
try {
  await api.send(data);
} catch (e) {
  logError(`Failed to send API request: ${e.message}`, { userId, clientId });
  throw new UserVisibleError("网络请求失败，请检查网络连接");
}
```

### [HIGH] 仅记录但继续 - src/service/cache.ts:89
**位置**: `src/service/cache.ts:89`
**代码**:
```typescript
try {
  const cached = cache.get(key);
} catch (e) {
  console.error("Cache read error"); // 只是记录，函数继续
  return null; // 隙默回退到 null
}
```
**问题**:
- 缓存失败被隐藏
- 函数悄悄回退到数据库
- 用户遇到意外延迟

**修复方案**:
```typescript
try {
  const cached = cache.get(key);
} catch (e) {
  logError(`Cache read failed for ${key}: ${e.message}`, { userId });
  metrics.increment("cache.fallback", 1);
  return null;
}
```

### [HIGH] 宽泛异常捕获 - src/handler/processor.ts:32
**位置**: `src/handler/processor.ts:32-35`
**代码**:
```typescript
try {
  processItem(item);
} catch (e) {
  // 捕获所有异常，可能隐藏对象错误
}
```
**隐藏的意外错误**:
- RangeError - 数组越界
- TypeError - 类型错误
- SyntaxError - JSON 解析失败
- ReferenceError - 未定义变量

**修复方案**:
```typescript
try {
  processItem(item);
} catch (e) {
  if (e instanceof ProcessorError) {
    logError(`Processing failed for item ${item.id}: ${e.message}`);
  } else {
    // 重新抛出非预期错误
    throw e;
  }
}
```

### [MEDIUM] 可选链隐藏错误 - src/utils/parser.ts:22
**位置**: `src/utils/parser.ts:22`
**代码**:
```typescript
const value = data?.nested?.value?.toString();
```
**问题**:
- 任何中间属性为 null/undefined 都被静默跳过
- 调用者无法知道数据是否完整
- 缺少错误通知

**修复方案**:
```typescript
if (!data || !data.nested || !data.nested.value) {
  throw new ValidationError("Missing required field: data.nested.value");
}
const value = data.nested.value.toString();
```
