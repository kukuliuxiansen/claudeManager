TDD 指导代理

引导 Red-Green-Refactor 循环，确保 80%+ 测试覆盖率

类型: Agent类

实现效果

TDD 指导代理是测试优先开发方法的引导者，确保代码在测试覆盖率达到 80%+ 的情况下编写。它在接到描述预期行为的需求时触发。

  • 测试先于代码

    强制执行测试驱动的开发流程

  • 红-绿-重构循环

    引导通过 RED-GREEN-REFACTOR 周期

  • 覆盖率保证

    确保 80%+ 行、函数、分支和语句覆盖率

  • 边缘情况检测

    在实现前识别并测试边缘情况

    • 核心原理

    TDD 指导代理基于三个核心原则：

    • 测试先于实现 - 永远不要写未经测试的代码
    • 微小、可验证的步伐 - 每次更改小到可以立即验证
    • 测试是规格 - 测试定义了代码应该做什么

    流程是严格顺序的：RED（测试失败）→ 绿色（通过测试的最小程序）→ REFACTOR（安全改进）。跳过任何步骤都会破坏 TDD 的价值。

    使用方法

    触发时机与 TDD 流程 
    // 触发时机
1. 用户描述预期功能或行为时
2. 修复 bug 时（写失败测试先行）
3. 添加新功能模块
4. 重构时需要增加测试覆盖

// TDD 周期
### 1. 首先写测试 (RED)
编写描述预期行为的失败测试
### 2. 运行测试 -- 验证失败
```bash
npm test
```
### 3. 编写最小程序实现 (GREEN)
仅编写足够通过测试的代码
### 4. 运行测试 -- 验证通过
### 5. 重构 (REFACTOR)
删除重复、改进命名、优化 -- 测试必须保持通过
### 6. 验证覆盖率
```bash
npm run test:coverage
```

// 必须测试的类型
| 类型 | 测试内容 | 何时 |
|-----|---------|------|
| **单元** | 单独的函数 | 始终 |
| **集成** | API 端点、数据库操作 | 始终 |
| **E2E** | 关键用户流 (Playwright) | 关键路径 |

// 覆盖率目标
- 行覆盖率: 80%+
- 函数覆盖率: 80%+
- 分支覆盖率: 80%+
- 语句覆盖率: 80%+

    使用技巧

    1

    最小程序实现 - GREEN 阶段只写必要的代码，不要优化

    2

    失败由测试驱动 - 只有当测试失败时才编写测试

    3

    测试独立 - 每个测试测试一件事，不依赖其他测试

    4

    快反馈循环 - ilitation 快速 IDE/run 命令用于即时反馈

    5

    测试即文档 - 测试应该清楚地说明代码的预期行为

## 阶段 1: RED - 编写失败测试

### 用户需求
"实现一个函数 toTitleCase(string)，将字符串转换为首字母大写格式"

### 编写的测试
```javascript
// tests/toTitleCase.test.js
describe('toTitleCase', () => {
  it('converts lowercase string to title case', () => {
    expect(toTitleCase('hello world')).toBe('Hello World');
  });

  it('handles empty string', () => {
    expect(toTitleCase('')).toBe('');
  });

  it('handles single word', () => {
    expect(toTitleCase('hello')).toBe('Hello');
  });
});
```

### 运行测试
```bash
$ npm test
FAIL ./tests/toTitleCase.test.js
  toTitleCase
    ✕ converts lowercase string to title case (5ms)
    ✕ handles empty string
    ✕ handles single word

Test Suites: 1 failed, 3 tests failed
```
**状态**: RED - 测试失败，符合预期 ✅

## 阶段 2: GREEN - 最小程序实现

### 编写的实现
```javascript
// src/utils.js
export function toTitleCase(str) {
  return str.split(' ').map(word =>
    word.charAt(0).toUpperCase() + word.slice(1)
  ).join(' ');
}
```

### 运行测试
```bash
$ npm test
PASS ./tests/toTitleCase.test.js
  toTitleCase
    ✓ converts lowercase string to title case
    ✓ handles empty string
    ✓ handles single word

Test Suites: 1 passed, 3 tests passed
```
**状态**: GREEN - 测试通过 ✅
