Java审查代理

专注于Java代码审查的专业AI代理

类型: Agent类

实现效果

针对Java代码进行专业审查，重点关注安全性、架构和代码质量。

  • 安全性审查
    检测SQL注入、命令注入、路径遍历、硬编码密钥等安全问题
  • Spring Boot架构
    检查控制器逻辑、事务注解、实体暴露等架构问题
  • JPA/数据库检查
    检测N+1查询、EAGER获取、缺少Modifying注解等问题
  • 异常处理审查
    检查未捕获异常、空catch块、错误HTTP状态码等

    • 核心原理

    采用优先级驱动的审查策略，按照严重程度分类问题：

    • CRITICAL：安全漏洞和严重错误处理问题
    • HIGH：架构设计缺陷和JPA使用不当
    • MEDIUM：代码风格和并发安全性问题

    审查前自动运行诊断命令，确保代码符合项目规范。

    使用方法

    触发方式 
    调用场景：Java代码审查请求

触发命令：
git diff -- '*.java'                 # 查看变更
mvn verify -q                         # 运行构建检查
./gradlew check                       # Gradle等效
./mvnw checkstyle:check               # 代码风格
./mvnw spotbugs:check                 # 静态分析
./mvnw test                           # 单元测试

    使用技巧

    1
    always先检查pom.xml/build.gradle确定构建工具和Spring版本
    2
    发现任何CRITICAL安全问题立即停止并转交给security-reviewer
    3
    重点关注控制器层，确保业务逻辑已委托到服务层
    4
    检查事务注解是否在服务层而非法控制器层
    5
    验证Optional使用，确保不会直接调用.get()导致异常
    6
    检查实体是否直接暴露在API响应中，应使用DTO或记录投影
