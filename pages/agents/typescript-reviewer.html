TypeScript 审查代理

专业 TypeScript/JavaScript 代码审查，涵盖类型安全、异步错误处理和最佳实践

类型: Agent类

实现效果

TypeScript 审查代理执行全面的 TypeScript/JavaScript 审查，重点关注类型安全、异步正确性、错误处理和 React/Node.js 特定问题。

  • 安全审查

    检测注入、XSS、硬编码密钥、路径遍历

  • 类型安全审查

    检测 any、非空断言滥用、类型断言滥用

  • 异步正确性

    检测未处理的 Promise、顺序 await、异步 forEach

  • 错误处理审查

    检测静默错误、缺失 try/catch、非 Error 对象抛出

    • 核心原理

    TypeScript 审查代理按严重级别审查代码，优先处理可能导致运行时错误和安全漏洞的代码：

    • CRITICAL - 安全 - 注入（eval/JSON.parse）、XSS（innerHTML）、硬编码密钥
    • HIGH - 类型安全 - any 类型、非空断言滥用 (!)、不合理的类型断言
    • HIGH - 异步 - 未处理的 Promise、异步 forEach、顺序 await 依赖
    • HIGH - 错误处理 - 空 catch、JSON.parse 无 try/catch、抛出非 Error 对象
    • MEDIUM - React/Node.js 特定 - 缺少依赖数组、状态突变、服务器/客户端边界泄漏

    审查标准遵循"顶级 TypeScript 商店或维护良好的开源项目"的质量要求。

// 审查命令
npm run typecheck --if-present       # TypeScript 类型检查
tsc --noEmit -p <config>             # TypeScript 编译器
eslint . --ext .ts,.tsx,.js,.jsx     # ESLint 检查
prettier --check .                    # 代码格式检查
npm audit                             # 依赖漏洞检查
vitest run                           # 测试运行 (Vitest)
jest --ci                            # 测试运行 (Jest)

// 通过标准
- **批准**: 无 CRITICAL 或 HIGH 问题
- **警告**: 仅有 MEDIUM 问题（可谨慎合并）
- **阻止**: 发现 CRITICAL 或 HIGH 问题

    使用技巧

    1

    any 是危险信号 - 立即用精确类型替代

    2

    避免非空断言 - 用 if (value) 或适当的默认值

    3

    async forEach 警告 - 使用 for...of 或 Promise.all

    4

    JSON.parse 安全 - 总是用 try/catch 包装

    5

    React 依赖数组 - 使用 exhaustive-deps lint 规则

## 发现问题

### [CRITICAL] 安全风险 - 非常量表达式求值
**文件**: src/utils/calculator.ts:42
**代码**:
```typescript
const result = new Function('return ' + userInput)();
```
**问题**: 使用 Function 构造器执行用户控制的代码字符串
**修复**: 使用安全的数学表达式解析器
```typescript
// 推荐使用专用库
import { evaluate } from 'mathjs';
const result = evaluate(userInput);
```

### [CRITICAL] XSS 漏洞 - innerHTML 用户输入
**文件**: src/components/Comment.tsx:15
**代码**:
```typescript
return <div dangerouslySetInnerHTML={{ __html: userComment }} />
```
**问题**: 用户控制的字符串直接设置为 innerHTML
**修复**: 使用 textContent 或 DOMPurify
```typescript
// 选项 1: React 自动转义
return <div>{userComment}</div>

// 选项 2: 使用 DOMPurify
return <div>{DOMPurify.sanitize(userComment)}</div>
```

### [HIGH] 非空断言滥用
**文件**: src/service/user.ts:78
**代码**:
```typescript
const value = someObject.value!; // 确定存在？验证过吗？
```
**问题**: 非空断言跳过类型检查
**修复**:
```typescript
if (!someObject.value) {
  throw new Error('Value is required');
}
const value = someObject.value;
```

### [HIGH] 未处理的 Promise rejection
**文件**: src/handler/request.ts:34
**代码**:
```typescript
async handleRequest() {
  const data = fetchData(); // 缺少 await！
}
```
**问题**: 异步函数调用未 await
**修复**:
```typescript
async handleRequest() {
  const data = await fetchData();
}
```

### [HIGH] JSON.parse 无 try/catch
**文件**: src/utils/parser.ts:22
**代码**:
```typescript
const config = JSON.parse(fs.readFileSync('config.json', 'utf8'));
```
**问题**: 无效 JSON 字符串会导致无捕获的 SyntaxError
**修复**:
```typescript
let config;
try {
  config = JSON.parse(fs.readFileSync('config.json', 'utf8'));
} catch (e) {
  logError('Failed to parse config.json', { error: e.message });
  throw new Error('Configuration invalid');
}
```

### [MEDIUM] 缺少依赖数组
**文件**: src/components/Chart.tsx:45
**代码**:
```typescript
useEffect(() => { updateChart(data); }, []);
```
**问题**: 依赖数组不完整
**修复**:
```typescript
useEffect(() => { updateChart(data); }, [data]);
```
