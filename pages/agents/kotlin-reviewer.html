Kotlin审查代理

专注于Kotlin代码审查的专业AI代理

类型: Agent类

实现效果

针对Kotlin代码进行专业审查，重点关注Android/KMP最佳实践和代码质量。

  • 架构审查
    检查模块边界、数据层泄漏、ViewModel业务逻辑等架构问题
  • 协程与Flow审查
    检测协程误用、Flow反模式、生命周期bug等问题
  • Compose性能审查
    识别组合性能问题和重新组合陷阱
  • Kotlin惯用法审查
    检查!!使用、var/val选择、Java风格模式等问题

    • 核心原理

    采用优先级驱动的审查策略，按照严重程度分类问题：

    • CRITICAL：架构违规和安全问题
    • HIGH：协程/Flow误用和Compose性能问题
    • MEDIUM：Kotlin惯用法和Android特定问题
    • LOW：Gradle和构建配置问题

    审查前自动运行诊断命令，确保代码符合项目规范。

    使用方法

    触发方式 
    调用场景：Kotlin代码审查请求

触发命令：
git diff --staged                    # 查看暂存变更
git diff                             # 查看未暂存变更
git log --oneline -5                 # 查看最近提交
git grep "import android" domain/    # 检查域模块框架依赖
./gradlew build                      # 运行构建检查

    使用技巧

    1
    先检查domain模块是否错误导入了Android/Ktor/Room框架
    2
    检查GlobalScope使用，应使用structured scopes如viewModelScope
    3
    验证CancellationException是否被正确处理，不应捕获并忽略
    4
    检查StateFlow中的可变集合，应使用copy()创建新状态
    5
    Compose中lambda应在remember中稳定化，避免每次重组创建新对象
    6
    Android组件权限配置需检查 exported、intent filters和deep links

    输出示例

    Review Output 
    [CRITICAL] 域模块导入Android框架
文件: domain/src/main/kotlin/com/app/domain/UserUseCase.kt:3
问题: `import android.content.Context` - 域模块必须是纯Kotlin无框架依赖
修复: 将Context依赖逻辑移到data或平台层，通过仓库接口传递数据

[HIGH] StateFlow包含可变列表
文件: presentation/src/main/kotlin/com/app/ui/ListViewModel.kt:25
问题: `_state.value.items.add(newItem)` 变更StateFlow内的列表 - Compose无法检测变更
修复: 使用 `_state.update { it.copy(items = it.items + newItem) }`

审查摘要:
| 严重程度 | 数量 | 状态 |
|----------|------|------|
| CRITICAL | 0    | 通过 |
| HIGH     | 1    | 阻止 |
| MEDIUM   | 2    | 信息 |
| LOW      | 0    | 注释 |

结论: 阻止合并 - 必须在合并前修复HIGH问题
