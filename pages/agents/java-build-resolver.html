Java构建解决代理

专业诊断和修复Java构建错误的AI代理

类型: Agent类

实现效果

专注于Java项目构建错误诊断和修复的专业代理，支持Maven和Gradle构建工具。

  • 编译错误修复
    诊断并修复Java编译错误，包括缺少导入、类型不匹配、未初始化变量等
  • 依赖冲突解决
    分析依赖树，解决版本不兼容和传递性依赖冲突问题
  • 注解处理器错误
    处理Lombok、MapStruct、Spring等注解处理器配置问题
  • 代码质量检查
    修复Checkstyle和SpotBugs违规，确保代码符合规范

    • 核心原理

    采用系统化的诊断流程，通过多阶段验证确保构建错误被根除：

    • 诊断阶段：运行构建命令解析错误消息
    • 上下文分析：阅读受影响文件理解代码场景
    • 精确修复：应用最少 necessary 的修复方案
    • 验证阶段：重新运行构建确认修复效果
    • 回归测试：运行测试确保修复未引入新问题

    使用方法

    触发方式 
    调用场景：Java Maven/Gradle项目构建失败时

自动触发条件：
- Maven: ./mvnw compile 或 mvn compile 失败
- Gradle: ./gradlew build 失败

触发命令：
- Maven项目: ./mvnw compile -q 2>&1
- Gradle项目: ./gradlew build 2>&1

    使用技巧

    1
    始终先运行构建命令解析错误，不要假设问题原因
    2
    修复后必须重新运行构建验证，确保问题已解决
    3
    优先检查pom.xml和build.gradle配置，确认构建工具类型
    4
    检查Java版本兼容性，确保编译器设置正确
    5
    Lombok等注解处理器需配置为annotationProcessorPaths而非普通依赖
    6
    使用dependency:tree分析传递性依赖冲突
