code-reviewer

专业代码审查技能，主动检查代码质量、安全性和可维护性

类型: Review Agent

实现效果

code-reviewer Agent 是专业的代码审查代理，主动在编写或修改代码后立即检查代码质量、安全漏洞和最佳实践。使用置信度过滤机制，只报告真正重要的高置信度问题。

  • 主动触发审查

    编写或修改代码后自动启动审查，无需用户手动请求

  • 安全漏洞检测

    识别 OWASP Top 10 漏洞，如硬编码密钥、SQL注入、XSS等

  • 代码质量检查

    检测大函数、深层嵌套、缺失错误处理、死代码等问题

  • 置信度过滤

    只报告 >80% 确信度的真实问题，避免噪音干扰

    • 核心原理

    审查流程遵循全面扫描 + 优先级分级原则。从 CRITICAL 到 LOW 分级处理问题，确保关键安全和架构问题优先解决。

    审查流程:

    • 1. 收集上下文 - 运行 git diff 查看所有变更
    • 2. 理解范围 - 识别变更文件、功能关联
    • 3. 读取周边代码 - 不孤立审查，理解完整上下文
    • 4. 应用检查清单 - 从 CRITICAL 到 LOW 逐级检查
    • 5. 报告发现 - 使用标准格式输出问题

    严重级别定义:

    级别说明处理优先级
    CRITICAL安全漏洞、数据泄露风险必须立即修复
    HIGH代码质量、架构问题合并前必须解决
    MEDIUM性能优化、改进建议建议处理
    LOW代码风格、命名规范可选改进

    使用方法

    code-reviewer Agent 在编写或修改代码后主动触发：

    触发场景与调用 
    // 主动触发条件
1. 编写新代码后 - 自动启动审查
2. 修改现有代码 - 检查变更影响
3. 准备 PR 前 - 确保代码质量
4. 合并请求时 - 验证安全标准

// 审查输出示例
[CRITICAL] 硬编码 API 密钥
文件: src/api/client.ts:42
问题: API 密钥 "sk-abc..." 暴露在源代码中
修复: 移至环境变量 process.env.API_KEY

[HIGH] 大函数需要拆分
文件: src/utils/parser.ts:89
问题: 函数 processData() 超过 80 行
修复: 拆分为 validateInput() 和 transformData()

// 审查摘要
| 严重级别 | 数量 | 状态 |
|---------|------|------|
| CRITICAL | 0    | pass |
| HIGH     | 2    | warn |
| MEDIUM   | 3    | info |
| LOW      | 1    | note |

结论: WARNING - 2 个 HIGH 问题需要在合并前解决

    使用技巧

    1

    置信度优先 - 只关注 >80% 确信度的真实问题

    2

    合并同类 - 相似问题合并报告，避免重复噪音

    3

    上下文审查 - 不孤立审查，理解完整文件上下文

    4

    项目规范 - 遵循 CLAUDE.md 定义的项目特定规范

    审查清单

    关键检查项 
    // 安全检查 (CRITICAL)
- 硬编码凭证 (API密钥、密码、令牌)
- SQL注入 (字符串拼接查询)
- XSS漏洞 (未转义用户输入)
- 路径遍历 (未验证文件路径)
- 认证绕过 (缺失权限检查)

// 代码质量 (HIGH)
- 大函数 (>50行) - 拆分为小函数
- 大文件 (>800行) - 提取模块
- 深层嵌套 (>4层) - 使用早返回
- 缺失错误处理 - 空catch块
- 变更模式 - 使用不可变操作

// React/Next.js (HIGH)
- 缺失依赖数组 - useEffect依赖不完整
- 渲染中setState - 导致无限循环
- 列表缺失key - 使用数组索引作为key
- 遗留闭包 - 事件处理器捕获旧状态

// Node.js/后端 (HIGH)
- 未验证输入 - 请求体直接使用
- 缺失限流 - 公共端点无 throttling
- N+1查询 - 循环中获取关联数据
- 错误泄露 - 向客户端发送内部错误
